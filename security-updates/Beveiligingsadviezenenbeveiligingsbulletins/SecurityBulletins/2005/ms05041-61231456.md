Informatie over de beveiligingslekken
-------------------------------------

<span></span>
#### Beveiligingslek in Remote Desktop Protocol - CAN-2005-1218:

Dit beveiligingslek heeft betrekking op een [denial of service](http://go.microsoft.com/fwlink/?linkid=21142x), waardoor een aanvaller een speciaal ontworpen Remote Desktop Protocol-bericht (RDP-bericht) kan verzenden naar een systeem waarin dit probleem optreedt. Een aanvaller kan ervoor zorgen dat dit systeem niet meer reageert.

#### Beperkende factoren voor het beveiligingslek in Remote Desktop Protocol - CAN-2005-1218:

-   Met aanbevolen procedures voor firewalls en standaardfirewallconfiguraties kunt u netwerken beveiligen tegen aanvallen die van buiten het bedrijfsnetwerk komen. Het wordt aanbevolen op de systemen die met internet zijn verbonden zo min mogelijk poorten bloot te stellen aan deze aanvallen.
-   Het Remote Desktop Protocol (RDP) is op geen enkel besturingssysteem standaard ingeschakeld. Op Windows XP en Windows Server 2003 kan de functie Hulp op afstand het RDP-protocol inschakelen. Op Windows XP Media Center Edition is het RDP-protocol ingeschakeld als er een Media Center Extender is geïnstalleerd. Ga naar de volgende [website](http://msdn.microsoft.com/library/default.asp?url=/library/en-us/mcxwp/html/windows_media_center_extender_and_windows_media_connect__gruu.asp) voor meer informatie over Media Center Extenders.

    Op Small Business Server 2000 en Windows Small Business Server 2003 is RDP standaard ingeschakeld. Op Windows Small Business Server 2003 en eerdere versies zijn de communicatiepoorten voor het RDP-protocol echter geblokkeerd voor internet. RDP is alleen op het lokale netwerk beschikbaar tenzij Terminal Services of Externe Internet-werkplek zijn ingeschakeld met behulp van de wizard E-mail en Internet-verbinding.

-   Als Extern bureaublad handmatig wordt ingeschakeld worden afhankelijk van de besturingssysteemversie de volgende wijzigingen van kracht in Windows Firewall:
    -   Op Windows XP Service Pack 2-systemen waarop Windows Firewall is ingeschakeld, wordt door het activeren van de functie Extern bureaublad automatisch de uitzondering hiervoor ingeschakeld in de firewall. Deze uitzondering is van toepassing op **alle computers, inclusief computer op internet**. Als u Extern bureaublad uitschakelt, wordt de firewalluitzondering automatisch uitgeschakeld.

        Op Windows XP Service Pack 1, Windows Server 2003 en Windows Server 2003 Service Pack 1 wordt door het activeren van de functie Extern bureaublad de firewalluitzondering niet automatisch ingeschakeld. Bij deze besturingssystemen wordt er een dialoogvenster weergegeven waarin u deze uitzondering handmatig moet inschakelen. Er staat een vermelding voor Extern bureaublad in de lijst met firewalluitzonderingen die een gebruiker handmatig dient in te schakelen. Het uitschakelen van de functie Extern bureaublad verandert er niets aan de uitzonderingsstatus in de firewall. Hoewel er op dit systeem niet langer via Extern bureaublad gebruik kan worden gemaakt van dit beveiligingslek, kan dit nog wel gebeuren via Hulp op afstand en Terminal Services, als deze functies beschikbaar zijn.

#### Tijdelijke oplossingen voor het beveiligingslek in Remote Desktop Protocol - CAN-2005-1218:

Microsoft heeft een aantal methoden getest om het probleem te omzeilen. Het probleem wordt er niet door verholpen, maar bekende aanvalsvectoren kunnen ermee worden geblokkeerd. Wanneer de functionaliteit wordt beperkt door de oplossing, wordt dat aangegeven in het volgende gedeelte.

-   **Schakel Terminal Services, Extern bureaublad, Hulp op afstand, en Externe Internet-werkplek (in Windows Small Business Server 2003) uit als u deze functies niet langer nodig hebt.**

    Als u deze services niet langer nodig hebt, is het een goed idee deze uit te schakelen. Hierdoor bent u beter beveiligd tegen beveiligingslekken.

    Ga naar de volgende [website](http://www.microsoft.com/resources/documentation/windows/xp/all/proddocs/en-us/rdesktop_disable.mspx) voor informatie over het handmatig uitschakelen van Extern bureaublad.

    Lees het volgende [Microsoft Knowledge Base-artikel](http://support.microsoft.com/kb/306300) voor informatie over het uitschakelen van Extern bureaublad met behulp van Groepsbeleid.

    Ga naar de volgende [website](http://www.microsoft.com/technet/prodtechnol/winxppro/maintain/intmgmt/20_xprem.mspx) voor informatie over Hulp op afstand en het handmatig en via Groepsbeleid uitschakelen van deze functie.

    Ga naar de volgende [website](http://www.microsoft.com/technet/security/secnews/articles/sec_sbs2003_network.mspx) voor informatie over de functies Terminal Services en Externe Internet-werkplek in Windows Small Business Server 2003.

-   **TCP-poort 3389 blokkeren op de bedrijfsfirewall:**

    Deze poort wordt gebruikt om een verbinding tot stand te brengen met het onderdeel waarin dit probleem optreedt. Door deze poort op de firewall te blokkeren, zorgt u ervoor dat op systemen achter deze firewall geen aanvallen kunnen worden uitgevoerd. Zo kunt u netwerken beveiligen tegen aanvallen die van buiten het bedrijfsnetwerk komen. Het blokkeren van de getroffen poorten op de firewall is de beste verdediging tegen internetaanvallen. Systemen kunnen echter nog steeds van binnen het bedrijf worden aangevallen. Op Windows XP en Windows Server 2003 kan de Windows Firewall helpen bij de beveiliging van individuele computers. Windows Firewall staat standaard geen verbindingen toe via deze poort, behalve in Windows XP Service Pack 2 als de functie Extern bureaublad is ingeschakeld. Ga naar de volgende [website](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/serverhelp/8b5e3b52-b77b-4d98-a058-b8083f393583.mspx) voor informatie over hoe u de uitzondering voor de functie Extern bureaublad kunt uitschakelen in Windows Firewall. Als u de uitzondering voor de functie Extern bureaublad niet kunt uitschakelen in Windows Firewall, kunt u wellicht de gevolgen van het beveiligingslek beperken door de standaardwaarde in te stellen op de optie voor het lokale netwerk in plaats van op de optie voor alle computers (inclusief computers op internet). Hierdoor verlaagt u de kans op aanvallen via internet.

    **Opmerking** In Windows Small Business Server 2003 zit een functie genaamd Externe Internet-werkplek. Deze functie gebruikt TCP-poort 4125 voor het verwerken van RDP-verbindingen. Als u deze functie gebruikt, dient u te controleren dat behalve poort 3389 ook deze poort is geblokkeerd voor internet.

    **Opmerking** U hebt wellicht andere poorten ingesteld voor de getroffen onderdelen. Als u dat hebt gedaan, moet u ook deze poorten blokkeren.

-   **Beveilig externe bureaubladverbindingen via IPsec-beleid.**

    Specifieke configuraties zijn afhankelijk van de individuele omgeving. Ga naar de volgende [website](http://www.microsoft.com/windowsserver2003/technologies/networking/ipsec/default.mspx) voor meer informatie over Internet Protocol Security (IPsec). Zie de Microsoft Knowledge Base-artikelen [313190](http://support.microsoft.com/kb/313190) en [813878](http://support.microsoft.com/kb/813878) voor uitgebreide informatie over IPSec en het toepassen van filters.

-   **Beveilig externe bureaubladverbindingen via een VPN-verbinding.**

    Specifieke configuraties zijn afhankelijk van de individuele omgeving. Ga naar de volgende [website](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/serverhelp/a08da8ea-a616-4422-bbd7-9cb8de066b29.mspx) voor meer informatie over virtuele particuliere netwerken.

#### Veelgestelde vragen over het beveiligingslek in Remote Desktop Protocol - CAN-2005-1218:

**Wat is de omvang van het probleem?**

Dit beveiligingslek heeft betrekking op een [denial of service](http://go.microsoft.com/fwlink/?linkid=21142). Een aanvaller die dit lek weet te misbruiken, kan ervoor zorgen dat het systeem waarin dit probleem optreedt opnieuw wordt opgestart. Gedurende die tijd kan de server geen aanvragen beantwoorden. Het beveiligingslek betreft een denial-of-service-probleem. Een aanvaller kan geen code uitvoeren of zichzelf gebruikersrechten toekennen, maar kan er wel voor zorgen dat het systeem waarin dit probleem optreedt geen aanvragen meer accepteert.

**Waardoor wordt het beveiligingslek veroorzaakt?**
Het proces waarmee door Remote Desktop Protocol gegevens worden gecontroleerd.

**Wat is Remote Desktop Protocol (RDP)?**
Met Remote Desktop Protocol (RDP) kunnen gebruikers een virtuele sessie op hun computer opzetten. Op deze manier hebben gebruikers via een andere computer toegang tot al hun gegevens en toepassingen op hun eigen computer. Ga naar de volgende [website](http://msdn.microsoft.com/library/en-us/termserv/termserv/remote_desktop_protocol.asp) voor meer informatie over RDP.

**Door welke Microsoft-producten wordt RDP gebruikt?**
RDP wordt als onderliggend protocol gebruikt door Windows-functies voor externe bureaubladsessies. Deze functies zijn:

-   Terminal Services in Windows 2000 en Windows Server 2003 maakt gebruik van RDP. Bezoek de volgende [website](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/technologies/featured/termserv/default.mspx) voor meer informatie over Terminal Services en RDP.
-   De functie [Extern bureaublad](http://www.microsoft.com/windowsxp/pro/using/howto/gomobile/remotedesktop/default.asp) in Windows XP maakt gebruik van RDP.
-   Bij een aanvraag voor hulp op afstand in Windows XP en Windows Server 2003 is RDP tijdens de aanvraag en nog een korte tijd na het verlopen van de aanvraag ingeschakeld. Ga naar de volgende [website](http://www.microsoft.com/technet/prodtechnol/winxppro/maintain/intmgmt/20_xprem.mspx) voor informatie over Hulp op afstand en het handmatig en via Groepsbeleid uitschakelen van deze functie.
-   Media Center Extenders op Windows XP Media Center Edition 2005-systemen maken gebruik van RDP. Ga naar de volgende [website](http://msdn.microsoft.com/library/default.asp?url=/library/en-us/mcxwp/html/windows_media_center_extender_and_windows_media_connect__gruu.asp) voor meer informatie over Media Center Extenders. Ga naar de volgende [website](http://msdn.microsoft.com/library/default.asp?url=/library/en-us/medctrsdk/htm/mediacenterextenders.asp) voor gedetailleerde technische informatie over Media Center Extenders. De Media Center Extenders kunnen behalve de standaard TCP-poort 3389 ook nog andere poorten gebruiken.
-   Externe Internet-werkplek in Windows Small Business Server 2003 maakt gebruik van RDP. Bekijk de volgende [webcast](http://support.microsoft.com/kb/833983) op Technet voor informatie over deze functie. Ga naar de volgende [website](http://www.microsoft.com/technet/security/secnews/articles/sec_sbs2003_network.mspx) voor meer informatie over het uitschakelen van de functie Externe Internet-werkplek in Windows Small Business Server 2003. De functie Externe Internet-werkplek gebruikt voor RDP behalve de standaard TCP-poort 3389 ook nog andere poorten.

**Is RDP standaard ingeschakeld Windows?**
Nee. Het Remote Desktop Protocol (RDP) is op geen enkel besturingssysteem standaard ingeschakeld. Op sommige versies van Windows XP Media Center Edition is RDP ingeschakeld als er een Media Center Extender is geïnstalleerd. Ga naar de volgende [website](http://msdn.microsoft.com/library/default.asp?url=/library/en-us/mcxwp/html/windows_media_center_extender_and_windows_media_connect__gruu.asp) voor meer informatie over Media Center Extenders. Ga naar de volgende [website](http://msdn.microsoft.com/library/default.asp?url=/library/en-us/medctrsdk/htm/mediacenterextenders.asp) voor gedetailleerde technische informatie over Media Center Extenders.

RDP is standaard ingeschakeld in Small Business Server 2000 en Windows Small Business Server 2003. Op Windows Small Business Server 2003 en vroeger zijn de communicatiepoorten voor RDP geblokkeerd voor internet en is RDP alleen maar beschikbaar op het lokale netwerk, tenzij Terminal Services of Externe Internet-werkplek is ingeschakeld met behulp van de wizard E-mail en Internet-verbinding configureren.

De functie Extern bureaublad is niet opgenomen in Windows XP Home Edition. Hulp op afstand maakt wel deel uit van dit besturingssysteem. Bij een aanvraag voor hulp op afstand is RDP tijdens de aanvraag en nog een korte tijd na het verlopen van de aanvraag ingeschakeld. Windows XP Home-systemen kunnen kwetsbaar voor dit probleem zijn als ze via Windows Firewall de uitzondering voor Extern bureaublad toestaan.

**Met welk doel kan een aanvaller het beveiligingslek misbruiken?**
Een aanvaller die dit lek weet te misbruiken, kan ervoor zorgen dat het systeem waarin dit probleem optreedt niet meer reageert.

**Wie kan dit beveiligingslek misbruiken?**
Elke anonieme gebruiker die ervoor kan zorgen dat een systeem waarin dit probleem optreedt een speciaal samengesteld bericht ontvangt, kan proberen dit beveiligingslek te misbruiken.

**Hoe kan een aanvaller dit beveiligingslek misbruiken?**
Een aanvaller kan het beveiligingslek misbruiken door een speciaal ontworpen bericht te maken en te verzenden naar een systeem waarin dit probleem optreedt. Nadat het bericht is ontvangen, kan er op het systeem waarvoor dit probleem geldt een fout optreden die leidt tot een denial of service.

**Voor welke systemen vormt dit beveiligingslek het grootste risico?**
Terminalservers met Windows 2000 Server en Windows Server 2003 lopen het grootste risico. De terminalservers zijn vatbaar voor dit beveiligingslek, als ze handmatig zijn geconfigureerd door een beheerder. Servers met Windows Small Business Server 2003 lopen risico als de beheerder met behulp van de wizard E-mail en Internet-verbinding configureren Terminal Services of verbindingen voor Externe Internet-werkplek van de server naar internet heeft ingeschakeld Systemen met Windows XP en Windows Server 2003 lopen risico als Extern bureaublad of Hulp op afstand handmatig is ingeschakeld. Windows 2000 Professional biedt geen ondersteuning voor het RDP-protocol en is daarom niet vatbaar voor dit probleem.

**Kan het beveiligingslek via internet worden misbruikt?**
Ja. Een aanvaller kan proberen via internet misbruik te maken van het beveiligingslek. Met aanbevolen procedures voor firewalls en standaardfirewallconfiguraties kunt u een betere beveiliging realiseren tegen aanvallen die vanaf internet komen. Microsoft biedt informatie over de beveiliging van uw pc. Eindgebruikers kunnen naar de volgende website gaan: [Beveilig uw pc](http://go.microsoft.com/fwlink/?linkid=21169). IT-professionals kunnen naar de volgende website gaan: [Security Guidance Center](http://go.microsoft.com/fwlink/?linkid=21171).

**Wat doet de update?**
Met de update verwijdert u het beveiligingslek door aanpassing van de manier waarop RDP de lengte van een bericht valideert voordat dit bericht wordt gelezen.

**Was dit beveiligingslek openbaar gemaakt op het moment dat dit beveiligingsbulletin werd uitgegeven?**

    4.  Klik in de lijst met bestanden met de rechtermuisknop op een bestandsnaam uit de betreffende bestandsinformatietabel en klik op **Eigenschappen**.
        **Opmerking** Afhankelijk van de versie van het geïnstalleerde besturingssysteem of de geïnstalleerde programma's zijn sommige bestanden uit de bestandsinformatietabel mogelijk niet geïnstalleerd.
    5.  Bepaal op het tabblad **Versie** de versie van het bestand dat op uw computer is geïnstalleerd door de versie te vergelijken met de versie in de betreffende bestandsinformatietabel.
        **Opmerking** Andere kenmerken dan de bestandsversie kunnen veranderen tijdens de installatie. Vergelijking van andere bestandskenmerken met de gegevens in de bestandsinformatietabel vormt geen ondersteunde methode om te controleren of de update is toegepast. Bovendien kunnen in bepaalde gevallen de namen van bestanden tijdens installatie worden gewijzigd. Als de bestands- of versiegegevens niet aanwezig zijn, gebruikt u een van de andere beschikbare methoden om te controleren of de update is geïnstalleerd.

-   **Verificatie van registersleutel**

    Aan de hand van de volgende registersleutel kunt u ook controleren welke bestanden met deze beveiligingsupdate zijn geïnstalleerd:

    HKEY\_LOCAL\_MACHINE\\SOFTWARE\\Microsoft\\Updates\\Windows 2000\\SP5\\KB899591\\Filelist

    **Opmerking** Deze registersleutel bevat mogelijk geen volledige lijst met geïnstalleerde bestanden. Bovendien is deze registersleutel mogelijk niet correct als een beheerder of een OEM beveiligingsupdate 899591 in de bronbestanden van de Windows-installatie integreert.

**Systems Management Server:**

Microsoft Systems Management Server (SMS) is een configureerbare bedrijfsoplossing voor het beheer van updates. Met SMS kunnen beheerders bepalen of beveiligingsupdates nodig zijn voor Windows-systemen, en deze updates in de gehele organisatie gecontroleerd implementeren met minimaal ongemak voor de eindgebruikers. Ga naar de website [SMS 2003 Security Patch Management](http://go.microsoft.com/fwlink/?linkid=22939) voor meer informatie over hoe beheerders met SMS 2003 beveiligingsupdates kunnen implementeren. SMS 2.0-gebruikers kunnen ook [Software Updates Service Feature Pack](http://go.microsoft.com/fwlink/?linkid=33340) gebruiken voor het implementeren van beveiligingsupdates. Ga naar de [SMS-website](http://go.microsoft.com/fwlink/?linkid=21158) voor informatie over SMS.

**Opmerking** SMS maakt gebruik van de Microsoft Baseline Security Analyzer, de Microsoft Office Detection Tool en de Enterprise Update Scan Tool om brede ondersteuning te kunnen bieden voor het zoeken en uitvoeren van beveiligingsupdates. Bepaalde software-updates worden mogelijk niet opgemerkt door deze hulpprogramma's. Beheerders kunnen in deze gevallen de inventarisatiefuncties van SMS gebruiken om de updates op bepaalde systemen uit te voeren. Ga naar de volgende [website](http://go.microsoft.com/fwlink/?linkid=33341) voor meer informatie over deze procedure. Voor bepaalde beveiligingsupdates zijn beheerdersrechten vereist na het opnieuw opstarten van het systeem. Beheerders kunnen voor het installeren van deze updates de Elevated Rights Deployment Tool gebruiken (die deel uitmaakt van het [SMS 2003 Administration Feature Pack](http://go.microsoft.com/fwlink/?linkid=33387) en het [SMS 2.0 Administration Feature Pack](http://go.microsoft.com/fwlink/?linkid=21161)).
