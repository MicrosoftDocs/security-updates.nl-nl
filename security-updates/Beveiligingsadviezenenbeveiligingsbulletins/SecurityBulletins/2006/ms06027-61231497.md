Samenvatting
------------

<span></span>
**Samenvatting:**

Met deze update wordt een pas ontdekt, openbaar beveiligingslek opgelost. Het beveiligingslek wordt omschreven in de sectie "Informatie over de beveiligingslekken" van dit bulletin.

Een verkeerd ingedeeld object in Word zorgt voor een beveiligingslek waardoor externe code kan worden uitgevoerd. Een aanvaller kan misbruik maken van het lek door een speciaal ontworpen Word-bestand te maken dat het uitvoeren van externe code kan veroorzaken.

Als er een gebruiker met beheerdersrechten is aangemeld, kan een aanvaller die misbruik weet te maken van dit beveiligingslek, volledige controle krijgen over het client-werkstation waarin dit probleem optreedt. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige gebruikersrechten maken. Gebruikers met accounts waarvoor minder gebruikersrechten op het systeem zijn ingesteld, lopen minder risico dan gebruikers die met beheerdersrechten werken.

Klanten wordt aangeraden de update onmiddellijk toe te passen.

#### Beveiligingslek in Microsoft Word door verkeerd ingedeeld object - CVE-2006-2492

Een verkeerd ingedeeld object in Word zorgt voor een beveiligingslek waardoor externe code kan worden uitgevoerd. Een aanvaller kan misbruik maken van het lek door een speciaal ontworpen Word-bestand te maken dat het uitvoeren van externe code kan veroorzaken.

Als een gebruiker met beheerdersrechten is aangemeld, kan een aanvaller die misbruik weet te maken van dit beveiligingslek volledige controle krijgen over een systeem waarin dit probleem optreedt. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige gebruikersrechten maken. Gebruikers met accounts waarvoor minder gebruikersrechten op het systeem zijn ingesteld, lopen minder risico dan gebruikers die met beheerdersrechten werken.

#### Beperkende Factoren voor beveiligingslek in Microsoft Word door verkeerd ingedeeld object - CVE-2006-2492:

-   Een aanvaller die erin slaagt misbruik te maken van dit beveiligingslek, kan dezelfde rechten over het systeem krijgen als de lokale gebruiker. Gebruikers met accounts waarvoor minder gebruikersrechten op het systeem zijn ingesteld, lopen minder risico dan gebruikers die met beheerdersrechten werken.
-   Het beveiligingslek kan niet automatisch via e-mail worden misbruikt. Aanvallen via e-mail lukken alleen als een gebruiker een bijlage opent die in een e-mailbericht is verzonden.
-   In Office XP en Office 2003 kan dit beveiligingslek niet automatisch worden misbruikt via een aanval vanaf het web. De aanvaller dient te beschikken over een website waarop zich een Office-bestand bevindt waarmee dit beveiligingslek wordt misbruikt. Een aanvaller kan een gebruiker er niet toe dwingen om naar de schadelijke website te gaan. De aanvaller moet een gebruiker er dus van overtuigen een bezoek te brengen aan de website. In de meeste gevallen doet de aanvaller dat door de gebruiker ertoe te bewegen op een koppeling te klikken waarmee de gebruiker naar de website van de aanvaller gaat.

**Opmerking** Hoewel dit beveiligingslek niet automatisch kan worden misbruikt wanneer een website wordt bezocht vanuit Word 2000, zal in Office 2000 de gebruiker voor het openen van een document niet worden gevraagd om de opdracht **Openen**, **Opslaan** of **Annuleren** uit te voeren.

#### Tijdelijke oplossingen voor het mogelijk uitvoeren van externe code in Microsoft Word-code met een verkeerd ingedeeld object - CVE-2006-2492:

Microsoft heeft een aantal methoden getest om het probleem te omzeilen. Het probleem wordt er niet door verholpen, maar bekende aanvalsvectoren kunnen ermee worden geblokkeerd. Wanneer de functionaliteit wordt beperkt door de oplossing, wordt dat aangegeven in het volgende gedeelte.

-   **Gebruik Microsoft Word altijd in de veilige modus**

    Als Word in de veilige modus wordt gebruikt, beschermt u het systeem tegen pogingen om dit beveiligingslek te misbruiken.

    **Stap 1 - De Outlook-functie voor het gebruiken van Word als e-maileditor uitschakelen**

    **Voor ondernemingen**

    **Handmatig (gebruikersinteractie)**
    Medewerkers in een onderneming schakelen aan de hand van de volgende stappen de Outlook-functie uit zodat Word niet als e-maileditor wordt gebruikt:

    1.  Start het systeem opnieuw op.
    2.  Open Outlook.
    3.  Klik op het menu **Extra**, klik op **Opties** en vervolgens op het tabblad **E-mailindeling**.
    4.  Schakel het selectievakje Microsoft Word gebruiken voor het bewerken van e-mailberichten uit**.**
    5.  Schakel het selectievakje Microsoft Word gebruiken om e-mailberichten in RTF-indeling te lezen uit.
    6.  Sluit Outlook.
    7.  Start het systeem opnieuw op.

    Raadpleeg [Word uitschakelen als uw e-maileditor voor meer informatie over het uitschakelen van Word als uw e-mail editor.](http://office.microsoft.com/en-us/assistance/hp052428491033.aspx)

    **Gevolgen van de oplossing:** Gebruikers kunnen hun e-mailberichten niet meer in Word of in de RTF-indeling lezen.

    **Groepsbeleid gebruiken**

    Domeinadministrators kunnen met groepsbeleid Word als e-maileditor voor gebruikers uitschakelen. U hoeft de computer niet opnieuw op te starten om deze beveiliging in te schakelen.

    Zie [Beheerderssjabloonbestand gebruiken bij groepsbeleid met het register](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/technologies/management/gp/admtgp.mspx) en [Wijzigingen in register distribueren](http://msdn.microsoft.com/library/default.asp?url=/library/en-us/dnw2kmag01/html/distributingregistrychanges.asp) voor informatie over het gebruik van registersleutels in een groepsbeleid.

    **Voor ondernemingen**

    Als Word in de veilige modus wordt gebruikt, beschermt u het systeem tegen pogingen om dit beveiligingslek te misbruiken.

    Alle versies van Word hebben een functie voor het herstellen van de toepassing waardoor Word in de veilige modus kan worden uitgevoerd. In de veilige modus wordt de functionaliteit uitgeschakeld en wordt voorkomen dat kwetsbare code kan worden misbruikt. U vindt de volledige set beperkingen op: [http://office.microsoft.com/en-us/assistance/HP030823931033.aspx](http://office.microsoft.com/en-us/assistance/hp030823931033.aspx).

#### Veelgestelde vragen over het beveiligingslek in Microsoft Word door verkeerd ingedeeld object - CVE-2006-2492:

**Wat is de omvang van het probleem?**  
Dit beveiligingslek heeft betrekking op de uitvoering van externe code. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan op afstand volledige controle krijgen over een kwetsbaar systeem.

Als een gebruiker met beheerdersrechten is aangemeld, kan een aanvaller die misbruik weet te maken van dit beveiligingslek volledige controle krijgen over een systeem waarin dit probleem optreedt. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige gebruikersrechten maken. Gebruikers met accounts waarvoor minder gebruikersrechten op het systeem zijn ingesteld, lopen minder risico dan gebruikers die met beheerdersrechten werken.

**Waardoor wordt het beveiligingslek veroorzaakt?**  
Als een gebruiker een speciaal gemaakt Word-bestand met een verkeerd ingedeeld object opent, kan het systeemgeheugen dusdanig corrupt raken dat een aanvaller willekeurige code kan uitvoeren.

**Met welk doel kan een aanvaller het beveiligingslek misbruiken?**  
Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan volledige controle krijgen over het systeem waarin dit probleem optreedt.

**Hoe kan een aanvaller dit beveiligingslek misbruiken?**  
In het geval van een aanval vanaf het web moet de aanvaller beschikken over een website waarop zich een Office-bestand bevindt waarmee wordt geprobeerd dit beveiligingslek te misbruiken. Een aanvaller kan een gebruiker er niet toe dwingen om naar de schadelijke website te gaan. De aanvaller moet een gebruiker er dus van overtuigen een bezoek te brengen aan de website. In de meeste gevallen doet de aanvaller dat door de gebruiker ertoe te bewegen op een koppeling te klikken waarmee de gebruiker naar de website van de aanvaller gaat.
In een aanval via e-mail kan een aanvaller misbruik maken van het beveiligingslek door een speciaal gemaakt bestand als bijlage naar de gebruiker te versturen en de gebruiker ertoe zien te bewegen deze bijlage te openen.

**Voor welke systemen vormt dit beveiligingslek het grootste risico?**  
Voor werkstations en terminalservers is het risico het grootst. Servers lopen een groter risico als gebruikers die niet over voldoende beheerdersmachtigingen beschikken de mogelijkheid krijgen om zich bij servers aan te melden en programma's uit te voeren. Dit is echter volkomen in strijd met de aanbevolen richtlijnen voor gebruik.

**Zou ik automatisch kunnen worden misbruikt wanneer ik een website bezoek vanuit Word 2000**?
