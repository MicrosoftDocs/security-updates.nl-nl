Security Bulletin

Microsoft-beveiligingsbulletin MS07-006 - Belangrijk
====================================================

Beveiligingslekken in Windows kunnen leiden tot misbruik van bevoegdheden (928255)
----------------------------------------------------------------------------------

Gepubliceerd: dinsdag 13 februari 2007

#### Samenvatting

**Voor wie is dit document bedoeld:** Klanten die Microsoft Windows gebruiken

**Gevolgen van het beveiligingslek:** Misbruik van bevoegdheden

**Maximaal prioriteitsniveau:** Belangrijk

**Aanbeveling:** Klanten wordt aangeraden deze update zo spoedig mogelijk te installeren.

**Vervanging van beveiligingsupdate:** Dit bulletin vervangt een eerdere beveiligingsupdate. Zie de sectie Veelgestelde vragen van dit bulletin voor de volledige lijst.

**Voorbehoud:** Geen

Informatie over de beveiligingslekken
-------------------------------------

<span></span>
#### Beveiligingslek met betrekking tot de Shell Hardware Detection-service in Windows - CVE-2007-0211:

Er is in de Windows-shell een beveiligingslek met betrekking tot misbruik van bevoegdheden aangetroffen in de manier waarop het besturingssysteem detectie en registratie van nieuwe hardware uitvoert. Als gevolg van dit beveiligingslek kan een geverifieerde gebruiker volledige controle over het systeem krijgen.

#### Beperkende factoren voor het beveiligingslek met betrekking tot de Shell Hardware Detection-service in Windows - CVE-2007-0211:

-   Een aanvaller moet geldige aanmeldingsreferenties hebben om dit beveiligingslek te kunnen misbruiken. Het beveiligingslek kan niet worden misbruikt door anonieme gebruikers.
-   In Windows XP Service Pack 2 en Windows Server 2003 Service Pack 1 moet een aanvaller beheerdersrechten hebben om vanaf een externe locatie misbruik van dit beveiligingslek te kunnen maken.

#### Tijdelijke oplossingen voor het beveiligingslek met betrekking tot de Shell Hardware Detection-service in Windows - CVE-2007-0211:

Microsoft heeft een aantal methoden getest om het probleem te omzeilen. Het probleem wordt er niet door verholpen, maar bekende aanvalsvectoren kunnen ermee worden geblokkeerd. Wanneer de functionaliteit wordt beperkt door de oplossing, wordt dat aangegeven in het volgende gedeelte.

-   D**e Shell Hardware Detection-service uitschakelen**

    Door de Shell Hardware Detection-service uit te schakelen zorgt u ervoor dat er via dit beveiligingslek geen aanvallen op uw systeem kunnen worden uitgevoerd. Volg deze stappen om de Shell Hardware Detection-service uit te schakelen:

    1.  Klik op **Start** en klik vervolgens op **Configuratiescherm** .(Of klik op Start, wijs **Instellingen** aan en klik op **Configuratiescherm**.)
    2.  Dubbelklik op **Systeembeheer**.
    3.  Dubbelklik op **Services**.
    4.  Dubbelklik op **Shell Hardware Detection**-service.
    5.  Klik in de lijst **Opstarttype** op **Uitgeschakeld**.
    6.  Klik op **Stoppen** en op **OK**.

    U kunt de Shell Hardware Detection-service ook uitschakelen door de volgende opdracht achter de opdrachtprompt in te voeren:

    `sc stop ShellHWDetection & sc config ShellHWDetection start= disabled`

    **Gevolgen van de tijdelijke oplossing:** Als u de **Shell Hardware Detection**-service uitschakelt, kunt u mogelijk geen gebruikmaken van de mogelijkheden voor snelle gebruikerswisseling. Daarom raden we deze tijdelijke oplossing alleen aan bij systemen waarop geen communicatie met digitale imaging-apparaten is vereist.

#### Veelgestelde vragen over het beveiligingslek met betrekking tot de Shell Hardware Detection-service in Windows - CVE-2007-0211:

**Wat is de omvang van het probleem?**
Dit beveiligingslek heeft betrekking op [misbruik van bevoegdheden](http://go.microsoft.com/fwlink/?linkid=21142). Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan volledige controle krijgen over een systeem waarin dit probleem optreedt. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige gebruikersrechten maken. Als een aanvaller misbruik wil maken van het beveiligingslek, moet deze zich bij het systeem kunnen aanmelden en een programma kunnen uitvoeren.

**Wat is de Shell Hardware Detection-service?**
De Shell Hardware Detection-service geeft meldingen voor AutoPlay-hardwaregebeurtenissen.

**Waardoor wordt het beveiligingslek veroorzaakt?**
Een functieparameter in de functies voor hardware-initialisatie en -detectie van Microsoft Windows is niet gevalideerd.

**Met welk doel kan een aanvaller het beveiligingslek misbruiken?**
Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan volledige controle krijgen over het systeem waarin dit probleem optreedt.

**Wie kan dit beveiligingslek misbruiken?**
Als een aanvaller misbruik wil maken van het beveiligingslek, moet deze zich bij het systeem kunnen aanmelden en een programma kunnen uitvoeren.

**Hoe kan een aanvaller dit beveiligingslek misbruiken?**
Om misbruik van dit beveiligingslek te maken zou een aanvaller zich eerst bij het systeem moeten aanmelden en een speciaal vervaardigde toepassing moeten uitvoeren. Een aanvaller zou vervolgens het systeem waarin dit probleem optreedt, kunnen dwingen om een hardwaredetectie- en initialiseringsgebeurtenis uit te voeren. De aanvraag kan lokaal of via een Terminal Services-sessie worden verzonden door een speciaal vervaardigde toepassing, hardwareapparaat of opslagvolume aan te roepen en misbruik te maken van het beveiligingslek, en op die manier volledige controle krijgen over het systeem waarin dit probleem optreedt.

**Voor welke systemen vormt dit beveiligingslek het grootste risico?**
Voor werkstations en terminalservers is het risico het grootst. Servers zouden eventueel meer gevaar kunnen lopen als beheerders toestaan dat gebruikers zich op servers kunnen aanmelden en programma's kunnen uitvoeren. Dit is echter volkomen in strijd met de aanbevolen richtlijnen voor gebruik.

**Kan het beveiligingslek via internet worden misbruikt?**
Nee. Een aanvaller moet zich kunnen aanmelden bij het systeem waarop de aanval is gericht. Een niet-geverifieerde aanvaller kan als gevolg van dit beveiligingslek niet een programma vanaf een externe locatie laden en uitvoeren.

**Wat doet de update?**
De update verwijdert het beveiligingslek door de parameter van de functie waarin het probleem optreedt te valideren.

**Was dit beveiligingslek openbaar gemaakt op het moment dat dit beveiligingsbulletin werd uitgegeven?**
Nee. Microsoft heeft uit discrete bronnen informatie over dit beveiligingslek ontvangen. Microsoft heeft geen informatie ontvangen die erop wees dat dit beveiligingslek openbaar was gemaakt op het moment dat dit beveiligingsbulletin oorspronkelijk werd uitgegeven.

**Was aan Microsoft gerapporteerd dat er misbruik was gemaakt van dit beveiligingslek op het moment dat dit beveiligingsbulletin werd uitgegeven?**
Nee. Microsoft beschikte niet over informatie dat dit beveiligingslek was gebruikt voor aanvallen op klanten en had geen voorbeelden van gepubliceerde conceptcode gezien als bewijs toen dat dit beveiligingsbulletin oorspronkelijk werd uitgegeven.

**Welk verband bestaat er tussen dit beveiligingslek en het beveiligingslek dat wordt verholpen door MS06-045?**
Ondanks dat beide beveiligingslekken voorkwamen in de Windows-shell, wordt in deze update een nieuw beveiligingslek beschreven dat niet is opgenomen in [MS06-045](http://technet.microsoft.com/security/bulletin/ms06-045). MS06-045 biedt beveiliging tegen het lek dat in dat bulletin wordt behandeld, maar dicht dit nieuwe beveiligingslek niet. Deze update vervangt MS06-045. U moet deze update installeren om uw systeem tegen beide beveiligingslekken te beveiligen.
