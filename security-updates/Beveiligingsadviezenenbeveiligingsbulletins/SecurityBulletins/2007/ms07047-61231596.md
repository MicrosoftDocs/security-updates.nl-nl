Security Bulletin

Microsoft-beveiligingsbulletin MS07-047 - Belangrijk
====================================================

Door een beveiligingslek in Windows Media Player kan externe code worden uitgevoerd (936782)
--------------------------------------------------------------------------------------------

Gepubliceerd: dinsdag 14 augustus 2007 | Bijgewerkt: woensdag 10 september 2008

**Versie:** 2.1

### Algemene informatie

#### Samenvatting

Met deze belangrijke beveiligingsupdate worden twee privé gemelde beveiligingslekken opgelost. Door deze beveiligingslekken kan code worden uitgevoerd als een gebruiker in Windows Media Player een speciaal vervaardigd bestand heeft weergegeven. Gebruikers met accounts waarvoor minder gebruikersrechten op het systeem zijn ingesteld, lopen minder risico dan gebruikers die met beheerdersrechten werken.

Dit is een belangrijke beveiligingsupdate voor ondersteunde versies van Windows Media Player 7.1, 9, 10 en 11. Zie voor meer informatie de subsectie **Software waarin dit probleem optreedt en software waarin dit probleem niet optreedt** in deze sectie.

Zie de subsectie Veelgestelde vragen over het specifieke beveiligingslek in de volgende sectie, **Informatie over het beveiligingslek**, voor meer informatie over de beveiligingslekken.

**Aanbeveling.** Microsoft adviseert klanten de update toe te passen.

**Bekende problemen.** In [Microsoft Knowledge Base-artikel 936782](http://support.microsoft.com/kb/936782) worden de bekende problemen beschreven die klanten mogelijk ondervinden bij de installatie van deze beveiligingsupdate. In het artikel worden ook de aanbevolen oplossingen voor deze problemen toegelicht.

#### Software waarin dit probleem optreedt en software waarin dit probleem niet optreedt

De hier vermelde software is getest om te bepalen in welke versies of edities dit probleem optreedt. Andere versies of edities hebben het einde van hun ondersteuningscyclus bereikt of hebben geen last van dit beveiligingslek. Ga naar [Microsoft Support Lifecycle](http://go.microsoft.com/fwlink/?linkid=21742) om de ondersteuningscyclus voor uw product en versie te bepalen.

Beveiligingslek in Windows Media Player met betrekking tot het parseren van weergavebestanden kan leiden tot het uitvoeren van code – CVE-2007-3037  
---------------------------------------------------------------------------------------------------------------------------------------------------
  
<span></span>
In Windows Media Player komt een beveiligingslek voor dat mogelijk de uitvoering van code toestaat bij het parseren van weergavebestanden. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan volledige controle krijgen over een systeem waarin dit probleem optreedt.
  
Zie [CVE-2007-3037](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-3037) als u dit beveiligingslek wilt weergeven als standaardvermelding in de lijst met veelvoorkomende beveiligingslekken en blootstellingen.
  
#### Beperkende factoren voor het beveiligingslek in Windows Media Player met betrekking tot het parseren van weergavebestanden als gevolg waarvan code kan worden uitgevoerd – CVE-2007-3037
  
Met beperkende factoren wordt verwezen naar een instelling, de algehele configuratie of algemene tips voor een standaardstatus waarmee de ernst van misbruik via een beveiligingslek kan worden verminderd. De volgende beperkende factoren kunnen in uw situatie van pas komen:
  
-   Wanneer een gebruiker probeert een Windows Media Player-weergavebestand te installeren, wordt er een downloadvenster weergegeven voordat de weergave wordt toegepast.  
-   Bij pogingen om via WMZ- en WMD-bestanden misbruik te maken van het beveiligingslek dreigt er pas gevaar wanneer de gebruiker het weergavebestand na het downloaden bekijkt of toepast.
  
#### Tijdelijke oplossingen voor het beveiligingslek in Windows Media Player met betrekking tot het parseren van weergavebestanden als gevolg waarvan code kan worden uitgevoerd – CVE-2007-3037
  
**De koppelingen van de bestandsextensies WMZ en WMD verwijderen**
  
Door de koppelingen van WMZ- en WMD-bestanden te verwijderen, voorkomt u dat deze bestanden in Windows Media Player worden geopend of in de voorbeeldweergave worden weergegeven.
  
1.  Klik op **Start,** vervolgens op **Uitvoeren**, typ **regedit** en klik op OK.  
2.  Vouw **HKEY\_CLASSES\_ROOT** uit, klik op .**WMZ** en klik met de rechtermuisknop op **Exporteren.** Opmerking:&lt;/B2&gt; Hierdoor wordt er standaard een back-up van deze registersleutel in de map 'Mijn documenten' gemaakt.  
3.  Typ in het venster **Registerbestand exporteren** 'Registerback-up van WMZ-bestandskoppeling.reg' en klik op **Opslaan.**  
    Hierdoor wordt er standaard een back-up van deze registersleutel in de map 'Mijn documenten' gemaakt.  
4.  Druk op **Verwijderen** om de registersleutel te verwijderen. Selecteer **Ja** om de verwijdering te bevestigen.  
5.  Herhaal stap 2-5 voor de **WMD**-sleutel.
  
**Gevolgen van de tijdelijke oplossing**: Met deze tijdelijke oplossing wordt voorkomen dat gebruikers weergavebestanden op Windows Media Player kunnen toepassen door op deze bestanden te dubbelklikken. Gebruikers kunnen wel de weergavebestanden uit de directory met standaard weergavebestanden blijven gebruiken.
  
**WMP.dll uit het register verwijderen**
  
1.  Als u de registersleutel Wmp.dll uit het register verwijdert, kan er op het systeem geen misbruik meer van het beveiligingslek worden gemaakt. Ga als volgt te werk als u de Wmp.dll-registersleutel wilt wijzigen.  
2.  Klik op **Start** en op **Uitvoeren**, typ "**regsvr32 -u %windir%\\system32\\wmp.dll**" (zonder de aanhalingstekens) en klik vervolgens op **OK**.  
3.  Klik op **OK** in het dialoogvenster waarin wordt gemeld dat het proces is voltooid.  
4.  Klik op het tabblad **Bestandstypen**.
  
**Gevolgen van de tijdelijke oplossing**: Door deze tijdelijke oplossing worden Windows Media Player en toepassingen uitgeschakeld die het ingesloten ActiveX-besturingselement van Windows Media gebruiken.
  
#### Veelgestelde vragen over het beveiligingslek in Windows Media Player met betrekking tot het parseren van weergavebestanden als gevolg waarvan code kan worden uitgevoerd – CVE-2007-3037
  
**Wat is de omvang van het beveiligingslek? **  
In Windows Media Player komt een beveiligingslek voor dat mogelijk de uitvoering van code toestaat bij het parseren van weergavebestanden. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan volledige controle krijgen over een systeem waarin dit probleem optreedt. Een aanvaller zou dan programma's kunnen installeren en gegevens kunnen weergeven, wijzigen of verwijderen.
  
**Waardoor wordt het beveiligingslek veroorzaakt? **  
Headerinformatie in weergavebestanden wordt door Windows Media Player niet correct verwerkt.
  
**Wat is een weergavebestand? **  
Weergavebestanden bestaan uit script-, afbeeldings-, media- en tekstbestanden die kunnen worden gecombineerd om zo een nieuwe vormgeving voor Windows Media Player te maken. Met weergavebestanden kunt u niet alleen het uiterlijk van Windows Media Player aanpassen, maar ook wijzigingen aanbrengen in de manier waarop Media Player functioneert. Weergavebestanden voor Windows Media Player hebben de bestandsindeling .WMZ en .WMD.
  
**Met welk doel kan een aanvaller het beveiligingslek misbruiken? **  
Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan volledige controle krijgen over een systeem waarin dit probleem optreedt. Gebruikers met een account die over minder gebruikersrechten beschikt voor het gastbesturingssysteem, lopen minder risico dan gebruikers met beheerdersrechten voor het gastbesturingssysteem.
  
**Hoe kan een aanvaller dit beveiligingslek misbruiken? **  
Bij een aanval via internet kan een kwaadwillende gebruiker bijvoorbeeld een speciaal gemaakt weergavebestand inzetten om misbruik te maken van dit beveiligingslek in Windows Media Player en een gebruiker ertoe verleiden het weergavebestand te bekijken. Een aanvaller kan echter een gebruiker er niet toe dwingen om naar deze websites te gaan. De aanvaller moet een gebruiker er dus van overtuigen een bezoek te brengen aan de website. In de meeste gevallen doet de aanvaller dat door de gebruiker ertoe te bewegen op een koppeling in een e-mailbericht of een verzoek in een expresbericht te klikken waarmee de gebruiker naar de website van de aanvaller gaat.
  
Bij een aanval per e-mail kan een kwaadwillende gebruiker misbruik maken van het beveiligingslek door een speciaal gemaakt weergavebestand naar de gebruiker te verzenden en deze ertoe verleiden het bestand te openen.
  
**Voor welke systemen vormt dit beveiligingslek het grootste risico? **  
Bij dit beveiligingslek kan pas schade worden aangericht wanneer een gebruiker is aangemeld en deze het schadelijke weergavebestand in Windows Media Player opent. Daarom is het risico het grootst voor computers waarop Windows Media Player regelmatig wordt gebruikt, zoals werkstations en terminalservers.
  
**Wat doet de update? **  
Met de update wordt het beveiligingslek verwijderd doordat Windows Media Player zodanig wordt aangepast dat headerinformatie in weergavebestanden voortaan correct wordt verwerkt.
  
**Was dit beveiligingslek openbaar gemaakt op het moment dat dit beveiligingsbulletin werd uitgegeven? **  
Nee. Microsoft heeft uit discrete bronnen informatie over dit beveiligingslek ontvangen. Microsoft heeft geen informatie ontvangen die erop wees dat dit beveiligingslek openbaar was gemaakt op het moment dat dit beveiligingsbulletin oorspronkelijk werd uitgegeven.

#### Veelgestelde vragen over het beveiligingslek in Windows Media Player met betrekking tot het uitpakken van weergavebestanden als gevolg waarvan code kan worden uitgevoerd – CVE-2007-3035
  
**Wat is de omvang van het beveiligingslek? **  
In Windows Media Player komt een beveiligingslek voor dat mogelijk de uitvoering van code toestaat bij het parseren van weergavebestanden. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan volledige controle krijgen over een systeem waarin dit probleem optreedt. Een aanvaller zou dan programma's kunnen installeren en gegevens kunnen weergeven, wijzigen of verwijderen.
  
**Waardoor wordt het beveiligingslek veroorzaakt? **  
Headerinformatie in weergavebestanden wordt door Windows Media Player niet correct verwerkt.
  
**Wat is een weergavebestand? **  
Weergavebestanden bestaan uit script-, afbeeldings-, media- en tekstbestanden die kunnen worden gecombineerd om zo een nieuwe vormgeving voor Windows Media Player te maken. Met weergavebestanden kunt u niet alleen het uiterlijk van Windows Media Player aanpassen, maar ook wijzigingen aanbrengen in de manier waarop Media Player functioneert. Weergavebestanden voor Windows Media Player hebben de bestandsindeling .WMZ en .WMD.
  
**Met welk doel kan een aanvaller het beveiligingslek misbruiken? **  
Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan volledige controle krijgen over een systeem waarin dit probleem optreedt. Gebruikers met een account die over minder gebruikersrechten beschikt voor het gastbesturingssysteem, lopen minder risico dan gebruikers met beheerdersrechten voor het gastbesturingssysteem.
  
**Hoe kan een aanvaller dit beveiligingslek misbruiken? **  
Bij een aanval via internet kan een kwaadwillende gebruiker bijvoorbeeld een speciaal gemaakt weergavebestand inzetten om misbruik te maken van dit beveiligingslek in Windows Media Player en een gebruiker ertoe verleiden het weergavebestand te bekijken. Een aanvaller kan echter een gebruiker er niet toe dwingen om naar deze websites te gaan. De aanvaller moet een gebruiker er dus van overtuigen een bezoek te brengen aan de website. In de meeste gevallen doet de aanvaller dat door de gebruiker ertoe te bewegen op een koppeling in een e-mailbericht of een verzoek in een expresbericht te klikken waarmee de gebruiker naar de website van de aanvaller gaat.
  
Bij een aanval per e-mail kan een kwaadwillende gebruiker misbruik maken van het beveiligingslek door een speciaal gemaakt weergavebestand naar de gebruiker te verzenden en deze ertoe verleiden het bestand te openen.
  
**Voor welke systemen vormt dit beveiligingslek het grootste risico? **  
Bij dit beveiligingslek kan pas schade worden aangericht wanneer een gebruiker is aangemeld en deze het schadelijke weergavebestand in Windows Media Player opent. Daarom is het risico het grootst voor computers waarop Windows Media Player regelmatig wordt gebruikt, zoals werkstations en terminalservers.
  
**Wat doet de update? **  
Met de update wordt het beveiligingslek verwijderd doordat Windows Media Player zodanig wordt aangepast dat headerinformatie in weergavebestanden voortaan correct wordt verwerkt.
  
**Was dit beveiligingslek openbaar gemaakt op het moment dat dit beveiligingsbulletin werd uitgegeven? **  
Nee. Microsoft heeft uit discrete bronnen informatie over dit beveiligingslek ontvangen. Microsoft heeft geen informatie ontvangen die erop wees dat dit beveiligingslek openbaar was gemaakt op het moment dat dit beveiligingsbulletin oorspronkelijk werd uitgegeven.
