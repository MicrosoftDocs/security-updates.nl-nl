Security Bulletin

Microsoft-beveiligingsbulletin MS08-062 - Belangrijk
====================================================

Beveiligingslek in Windows Internet Printing Service kan leiden tot uitvoering van externe code (953155)
--------------------------------------------------------------------------------------------------------

Gepubliceerd: dinsdag 14 oktober 2008 | Bijgewerkt: woensdag 29 oktober 2008

**Versie:** 2.2

### Algemene informatie

#### Samenvatting

Deze update lost een privé gemeld beveiligingslek in de Windows-service Internet-afdrukken op waardoor externe code kan worden uitgevoerd. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan volledige controle krijgen over een systeem waarin dit probleem optreedt. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts maken.

Het prioriteitsniveau van deze update is Belangrijk voor alle ondersteunde edities van Microsoft Windows 2000, Windows XP, Windows Server 2003 en Windows Server 2008. Zie de subsectie **Software waarin dit probleem optreedt en software waarin dit probleem niet optreedt** in deze sectie voor meer informatie.

De beveiligingsupdate lost dit beveiligingslek op door de wijze aan te passen waarop binnen de IPP-service (Internet Printing Protocol) geheugen wordt toegewezen. Zie de subsectie Veelgestelde vragen over het specifieke beveiligingslek in de volgende sectie, **Informatie over het beveiligingslek**, voor meer informatie over het beveiligingslek.

**Aanbeveling.** Microsoft raadt klanten aan de update zo spoedig mogelijk te installeren.

**Bekende problemen.** Geen

Veelgestelde vragen met betrekking tot deze beveiligingsupdate
--------------------------------------------------------------

<span></span>
**Waarom zijn er op 29 oktober 2008 wijzigingen in dit bulletin aangebracht?**  
Microsoft reviseerde dit bulletin om klanten te informeren dat de updates voor Windows Server 2008 voor Itanium-systemen en alle ondersteunde versies van Windows Vista nu beschikbaar zijn op Windows Update, Microsoft Update, Windows Software Update Services (WSUS), Systems Management Server en System Center Configuration Manager. Toen dit bulletin voor het eerst werd vrijgegeven, waren de updates voor deze platforms uitsluitend verkrijgbaar bij het Microsoft Downloadcentrum.

**Waarom zijn er op 16 oktober 2008 wijzigingen in dit bulletin aangebracht?**
Microsoft legt in dit bulletin uit dat de Windows-service Internet-afdrukken wordt uitgevoerd in de context van de spoolerservice die wordt uitgevoerd onder systeemrechten. Een aanvaller die het beveiligingslek dat in deze beveiligingsupdate wordt beschreven, weet te misbruiken, kan willekeurige code onder de systeemrechten uitvoeren en volledige controle over het getroffen systeem krijgen.

**Waarom zijn er op 15 oktober 2008 wijzigingen in dit bulletin aangebracht?**
Microsoft reviseerde dit bulletin om het prioriteitsniveau voor Windows Server 2008 voor Itanium-systemen te verwijderen. Ook werkte Microsoft dit bulletin bij om duidelijk te maken dat de update voor Windows Server 2008 voor Itanium-systemen beschikbaar is via het Microsoft Downloadcentrum en dat deze update zo snel mogelijk via andere distributiekanalen zal worden vrijgegeven.

**Waarom is Windows Server 2008 voor Itanium-systemen en Windows** **Vista** **opgenomen in de tabel met software waarin dit probleem optreedt terwijl het geen prioriteitsniveau heeft?**
Windows Server 2008 voor Itanium-systemen en alle ondersteunde versies van Windows Vista bevatten het onderdeel gedeelde afdruk-spooler dat door deze beveiligingsupdate wordt bijgewerkt. Maar omdat Internet-afdrukken niet wordt ondersteund door Windows Server 2008 voor Itanium-systemen, kan de in dit beveiligingsbulletin behandelde gevoelige vector niet worden misbruikt. De beveiligingsupdate is een grondige beschermingsmaatregel tegen mogelijk nieuwe vectoren die in de toekomst bekend kunnen worden en daarom wordt gebruikers van Windows Server 2008 voor Itanium-systemen en van Windows Vista aangeraden de beveiligingsupdate toe te passen.

Beveiligingslek met betrekking tot integeroverloop in IPP-service - CVE-2008-1446  
---------------------------------------------------------------------------------
  
<span></span>
Er komt een beveiligingslek voor op Windows-systemen waarop Microsoft Internet Information Services (IIS) wordt uitgevoerd en waarop de Windows-service Internet-afdrukken is ingeschakeld, waardoor externe code kan worden uitgevoerd. Hierdoor kan een externe geverifieerde aanvaller willekeurige code uitvoeren op het systeem waarop dit probleem optreedt.
  
Zie [CVE-2008-1446](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1446) als u dit beveiligingslek wilt weergeven als standaardvermelding in de lijst met veelvoorkomende beveiligingslekken en blootstellingen.
  
#### Beperkende factoren met betrekking tot integeroverloop in IPP-service - CVE-2008-1446
  
Met beperkende factoren wordt verwezen naar een instelling, de algehele configuratie of algemene tips voor een standaardstatus waarmee de ernst van misbruik via een beveiligingslek kan worden verminderd. De volgende beperkende factoren kunnen in uw situatie van pas komen:
  
-   Internet Printing Protocol (IPP) is in IIS voor Microsoft Windows 2000 en Windows XP standaard ingeschakeld. De standaardbestandmachtigingen voor alle platforms worden echter zo ingesteld dat alleen gebruikers in de "Geverifieerde" groep toegang hebben tot IPP. Wegens deze beperking hebben niet-geverifieerde gebruikers standaard niet de noodzakelijke mapmachtigingen om dit beveiligingslek extern te misbruiken.  
-   Installaties van IIS in Windows Server 2003 zijn niet kwetsbaar in hun standaardconfiguratie. Alleen Windows Server 2003-installaties waarin het ISAPI-filter voor Internet Printing Protocol is ingeschakeld in IIS, zijn kwetsbaar. Zowel de component Internet-afdrukken als Active Server Pages is noodzakelijk om printerbeheer en afdrukken via internet in te schakelen. Indien de component Active Server Pages op "Toegestaan" wordt ingesteld en de component Internet-afdrukken op "Verboden" wordt ingesteld, wordt alleen printerbeheer via internet ingeschakeld. Gebruikers kunnen via Internet-afdrukken geen verbinding met gedeelde printers maken.  
-   Installaties van IIS in Windows Server 2008 zijn standaard niet kwetsbaar. Een computer met Windows Server 2008 kan fungeren als clientcomputer die gebruikmaakt van afdrukken via internet. Gebruikers van de computer die afdrukaanvragen maken, moeten echter worden geverifieerd door de afdrukserver voordat zij de printers kunnen gebruiken die op die server zijn aangesloten. Om een computer met Windows Server 2008 als server met ondersteuning voor internetafdrukken te laten fungeren, moet u de rol Webserver (IIS), de rol Afdrukservices en de rolservice Internet-afdrukken in de rol Afdrukservices installeren.  
-   Voor misbruik van dit beveiligingslek moet de IIS-server via poort 445 naar buiten communiceren met een schadelijke computer. Beheerders die uitgaande SMB-verbindingen vanaf de IIS-server naar internet blokkeren, kunnen misbruik van dit beveiligingslek vanuit niet-vertrouwde systemen voorkomen.
  
#### Tijdelijke oplossingen voor het beveiligingslek met betrekking tot integeroverloop in IPP-service - CVE-2008-1446
  
Tijdelijke oplossingen zijn wijzigingen in een instelling of configuratie waardoor het beveiligingslek zelf niet wordt gedicht, maar waarmee aanvalsvectoren wel worden geblokkeerd voordat u de update uitvoert. Microsoft heeft de volgende methoden en statussen om het probleem te omzeilen getest om uit te zoeken of een tijdelijke oplossing functionaliteit vermindert:
  
-   **De IPP-service uitschakelen**
  
    IPP-service uitschakelen in IIS 5.0 en IIS 5.1
  
    1.  Open **Internetservicebeheer**.  
    2.  Klik met de rechtermuisknop op de **standaardwebsite** en klik op **Eigenschappen**.  
    3.  Ga naar het tabblad **Basismap** en klik op **Configuratie**.  
    4.  Blader naar beneden naar de extensie **.printer** en het overeenkomende pad naar **msw3prt.dll**.  
    5.  Klik op **Verwijderen**.  
    6.  Klik op **OK**.  
    7.  Start de IIS-service opniew op.
  
    **Opmerking** Internet-afdrukken kan worden geconfigureerd via groepsbeleid en via Internetservicebeheer. Indien er een conflict is tussen de groepsbeleidinstellingen en de instellingen in Internetservicebeheer, krijgen de groepsbeleidinstellingen voorrang. Indien u Internet-afdrukken verwijdert via Internetservicebeheer, controleer dan of Internet-afdrukken niet opnieuw wordt ingeschakeld door lokaal groepsbeleid of groepsbeleid op het domein. (Het standaardgroepsbeleid schakelt Internet-afdrukken niet in of uit). Controleer Computerconfiguratie | Beheersjablonen | Afdrukken | Afdrukken via het web in de module MMC-groepsbeleid.
  
    IPP-service uitschakelen in IIS 6.0
  
    1.  Start **IIS Manager** of de **IIS-module**.  
    2.  Vouw **\* server\_naam** uit, waarbij server\_naam de naam van de server is.  
    3.  Klik op **Webserver-extensies**.  
    4.  Klik in het rechterdeelvenster op **Internet-afdrukken** en klik vervolgens op **Niet toegestaan**.  
    5.  Sluit IIS Manager.
  
    IPP-service uitschakelen in IIS 7.0 op Windows Server 2008
  
    1.  Als **Serverbeheer** niet wordt uitgevoerd, klikt u achtereenvolgens op **Start**, **Systeembeheer** en **Serverbeheer**. (Als het dialoogvenster **Gebruikersaccountbeheer** verschijnt, controleert u of de gewenste actie wordt weergegeven en klikt u vervolgens op **Doorgaan**)  
    2.  Vouw het rollenknooppunt uit, klik met de rechtermuisknop op **Afdrukservices** en selecteer **Functieservices verwijderen**.  
    3.  De wizard Functieservices verwijderen wordt weergegeven. Indien de optie **Internet-afdrukken** is ingeschakeld, schakelt u dat vakje uit. Klik op **Volgende** en vervolgens op **Verwijderen** om de wizard af te ronden.
  
    **Gevolgen van de tijdelijke oplossing:** Gebruikers kunnen via Internet-afdrukken geen verbinding maken met gedeelde printers.
  
    **Zo kunt u tijdelijke oplossingen ongedaan maken.**
  
    IPP-service inschakelen in IIS 5.0 en IIS 5.1
  
    1.  Open **Internetservicebeheer**.  
    2.  Klik met de rechtermuisknop op de **standaardwebsite** en klik op **Eigenschappen**.  
    3.  Ga naar het tabblad **Basismap** en klik op **Configuratie**.  
    4.  Blader naar beneden naar de extensie **.printer** en het overeenkomende pad naar **msw3prt.dll**.  
    5.  Klik op **Toevoegen**.  
    6.  Typ het volgende in het veld **Uitvoerbaar bestand**:  
        %systemroot%\\system32\\msw3prt.dll  
    7.  Typ het volgende in het veld **Extensie**:  
        .printer  
    8.  Typ het volgende in het veld **Beperken tot**:  
        GET,POST  
    9.  Start de IIS-service opnieuw zodra deze stap voltooid is.
  
    IPP-service inschakelen in IIS 6.0
  
    1.  Start **IIS Manager** of de **IIS-module**.  
    2.  Vouw **\* server\_naam** uit, waarbij server\_naam de naam van de server is.  
    3.  Klik op **Webserver-extensies**.  
    4.  Klik in het rechterdeelvenster op **Internet-afdrukken** en klik vervolgens op **Toestaan**.  
    5.  Sluit IIS Manager.
  
    IPP-service inschakelen in IIS 7.0
  
    1.  Als **Serverbeheer** niet wordt uitgevoerd, klikt u achtereenvolgens op **Start**, **Systeembeheer** en **Serverbeheer**. (Als het dialoogvenster **Gebruikersaccountbeheer** verschijnt, controleert u of de gewenste actie wordt weergegeven en klikt u vervolgens op **Doorgaan**)  
    2.  Vouw het rollenknooppunt uit, klik met de rechtermuisknop op **Print Services** en selecteer **Functieservices toevoegen**.  
    3.  De wizard Functieservices toevoegen wordt weergegeven. Schakel de optie **Internet-afdrukken** in, klik op **Volgende** en vervolgens op **Installeren** om de wizard af te ronden.
  
-   **IIS Lockdown Tool 2.1 uitvoeren**
  
    In Microsoft Windows 2000 met IIS 5.0 of Windows XP met IIS 5.1 kunt u IIS Lockdown Wizard 2.1 gebruiken om onnodige functies, waaronder Internet-afdrukken, uit te schakelen. Om deze wizard uit te voeren, moet u de volgende acties voltooien:
  
    1.  U kunt IIS Lockdown Tool 2.1 [hier](http://www.microsoft.com/downloads/details.aspx?familyid=dde9efc0-bb30-47eb-9a61-fd755d23cdec&displaylang=en) downloaden.  
    2.  Ga op een van de volgende manieren verder:  
        Klik als u onmiddellijk met de installatie wilt beginnen, op **Openen** of **Programma uitvoeren vanaf de huidige locatie**.  
        Als u de installatie later wilt uitvoeren en de download naar uw computer wilt kopiëren, klik dan op **Opslaan** of **Dit programma opslaan op schijf**.
  
    **Gevolgen van de tijdelijke oplossing.**
  
    Gebruikers kunnen via Internet-afdrukken geen verbinding maken met gedeelde printers. Gebruikers van IIS Lockdown Tool kunnen via Internet-afdrukken ook geen verbinding maken met gedeelde printers. Zie het volgende [MSDN-artikel](http://msdn.microsoft.com/en-us/library/aa302365.aspx) voor een volledige lijst met wijzigingen die door IIS Lockdown Tool zijn aangebracht.
  
    **Zo kunt u tijdelijke oplossingen ongedaan maken.**
  
    Om de wijzigingen ongedaan te maken die door IIS Lockdown Tool zijn aangebracht, voert u **IISlockd.exe** nogmaals uit en geeft u aan dat u de wijzigingen ongedaan wilt maken. Tijdens het ongedaan maken worden de systeeminstellingen hersteld die van kracht waren voordat u IISLockdown in eerste instantie uitvoerde. Deze details worden in het logboekbestand \\WINNT\\System32\\inetsrv\\0blt-log opgenomen. Daarom is het belangrijk dat u het systeem onmiddellijk test nadat u IISLockdown hebt uitgevoerd. Als wijzigingen ongedaan moeten worden gemaakt, doet u dit onmiddellijk.
  
    **Opmerking** Het ISAPI-filter URLScan, dat bij IIS Lockdown Tool wordt geïnstalleerd, wordt niet verwijderd wanneer u de wijzigingen ongedaan maakt. U kunt URLScan handmatig verwijderen via het tabblad ISAPI-filters op serverniveau in Internetservicebeheer.
  
#### Veelgestelde vragen voor het beveiligingslek met betrekking tot integeroverloop in IPP-service - CVE-2008-1446
  
**Wat is de omvang van het beveiligingslek?**    
Dit beveiligingslek heeft betrekking op de uitvoering van externe code. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan volledige controle krijgen over een systeem waarin dit probleem optreedt. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts maken.
  
**Waardoor wordt het beveiligingslek veroorzaakt?**    
De IPP-functie (Internet Printing Protocol) wordt geïmplementeerd als ISAPI-extensie voor IIS. Er komt in de implementatie van Microsoft IPP op Windows-servers waarop IIS wordt uitgevoerd, een extern, geverifieerd beveiligingslek voor. Er komt een integeroverloop in de IPP-implementatie voor, waardoor een geverifieerde aanvaller externe code kan uitvoeren op IIS-servers waarop dit probleem optreedt.
  
**Wat is ISAPI?**  
ISAPI (Internet Services Application Programming Interface) is een technologie waarmee webontwikkelaars de functionaliteit van hun webservers kunnen uitbreiden door aangepaste code te schrijven die nieuwe services voor een webserver biedt. De aangepaste code kan worden geïmplementeerd in een ISAPI-filter (als de nieuwe functionaliteit een service op laag niveau omvat) of als ISAPI-extensie (als de nieuwe functionaliteit een service of hoog niveau omvat). In dit geval is de getroffen code een ISAPI-extensie.
  
**In welke ISAPI-extensie komt het probleem voor?**  
In de getroffen ISAPI-extensie wordt Internet Printing Protocol (IPP) geïmplementeerd, een industriestandaard in RFC 2910 en 2911. IPP biedt een manier om afdrukservices aan te vragen en de status van afdruktaken op internet op te vragen via HTTP. Bijvoorbeeld: een reizende gebruiker kan via internet een afdruktaak verzenden die op een printer in het bedrijfsnetwerk moet worden uitgevoerd. De gebruiker kan ook zien of de afdrukaanvraag foutloos is voltooid.
  
In Microsoft Windows 2000 is ingebouwde ondersteuning voor Internet-afdrukken geïntroduceerd. Met behulp van de Microsoft Windows 2000-implementatie kunnen gebruikers rechtstreeks naar een URL afdrukken en informatie over afdruktaken via hun browser bekijken. De ondersteuning voor Internet-afdrukken is in Microsoft Windows 2000 Server-versies standaard ingeschakeld. Het kan ook als optioneel onderdeel voor Microsoft Windows 2000 Professional, Windows XP, Windows Server 2003 en Windows Server 2008 worden geïnstalleerd.
  
**Met welk doel kan een aanvaller het beveiligingslek misbruiken?**    
Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan met systeemrechten willekeurige code op een gebruikerssysteem uitvoeren. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan volledige controle krijgen over een systeem waarin dit probleem optreedt. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts maken.
  
**Hoe kan een aanvaller dit beveiligingslek misbruiken?**    
Een aanvaller kan een speciaal ontworpen HTTP POST-aanvraag naar een kwetsbare webserver verzenden, waardoor de server verbinding maakt met een computer die door de aanvaller wordt bediend en die via IPP als printer fungeert. De aanvaller kan vervolgens met speciaal ontworpen antwoorden externe code op de doelserver uitvoeren.
  
**Voor welke systemen vormt dit beveiligingslek het grootste risico?**    
Voor werkstations en servers waarop IIS en de IPP-service (Internet Printing Protocol) worden uitgevoerd, vormt dit beveiligingslek het grootste risico.
  
**Wat doet de update?**    
Deze beveiligingsupdate lost dit beveiligingslek op door de gegevens die van externe computers worden ontvangen, via de IPP-service (Internet Printing Protocol) te valideren.
  
**Was dit beveiligingslek openbaar gemaakt op het moment dat dit beveiligingsbulletin werd uitgegeven?**    
Nee. Microsoft heeft uit discrete bronnen informatie over dit beveiligingslek ontvangen. Microsoft heeft geen informatie ontvangen die erop wees dat dit beveiligingslek openbaar was gemaakt op het moment dat dit beveiligingsbulletin oorspronkelijk werd uitgegeven.
  
**Was aan Microsoft gerapporteerd dat er misbruik was gemaakt van dit beveiligingslek op het moment dat dit beveiligingsbulletin werd uitgegeven?**    
Ja, Microsoft is op de hoogte van gerichte aanvallen waarbij wordt geprobeerd het beveiligingslek te misbruiken. Microsoft had geen voorbeelden van gepubliceerde conceptcode gezien als bewijs toen dit beveiligingsbulletin oorspronkelijk werd uitgegeven.
