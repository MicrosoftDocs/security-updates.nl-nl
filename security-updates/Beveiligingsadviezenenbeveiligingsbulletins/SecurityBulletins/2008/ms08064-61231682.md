Security Bulletin

Microsoft-beveiligingsbulletin MS08-064 - Belangrijk
====================================================

Beveiligingslek door manipulatie van de virtuele adresbeschrijving kan leiden tot misbruik van bevoegdheden (956841)
--------------------------------------------------------------------------------------------------------------------

Gepubliceerd: dinsdag 14 oktober 2008 | Bijgewerkt: woensdag 15 oktober 2008

**Versie:** 1.1

### Algemene informatie

#### Samenvatting

Deze beveiligingsupdate lost een privé gemeld beveiligingslek in Virtual Address Descriptor op. Door het beveiligingslek kan misbruik van bevoegdheden worden gemaakt als een gebruiker een speciaal vervaardigde toepassing uitvoert. Een geverifieerde aanvaller die gebruik weet te maken van dit beveiligingslek, kan op het getroffen systeem misbruik van bevoegdheden maken. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige beheerdersrechten maken.

Dit is een kritieke beveiligingsupdate voor alle ondersteunde versies van Windows XP, Windows Server 2003, Windows Vista en Windows Server 2008. Zie de subsectie **Software waarin dit probleem optreedt en software waarin dit probleem niet optreedt** in deze sectie voor meer informatie.

De beveiligingsupdate dicht het beveiligingslek door de manier te wijzigen waarop variabelen van de geheugentoewijzing door de virtuele adresbeschrijving worden verwerkt. Zie de subsectie Veelgestelde vragen over het specifieke beveiligingslek in de volgende sectie, **Informatie over het beveiligingslek**, voor meer informatie over het beveiligingslek.

**Aanbeveling.** Microsoft raadt klanten aan de update zo spoedig mogelijk te installeren.

**Bekende problemen.** Geen

Beveiligingslek dat kan leiden tot misbruik van bevoegdheden in de virtuele adresbeschrijving - CVE-2008-4036  
-------------------------------------------------------------------------------------------------------------
  
<span></span>
Er is sprake van een beveiligingslek met betrekking tot misbruik van bevoegdheden door de manier waarop door geheugenbeheer de geheugentoewijzing en de virtuele adresbeschrijvingen (VAD's) worden verwerkt. Door het beveiligingslek kunnen bevoegdheden worden misbruikt als een geverifieerde aanvaller een speciaal vervaardigd programma op een getroffen systeem uitvoert. Een aanvaller die gebruik weet te maken van dit beveiligingslek, kan op het getroffen systeem misbruik van bevoegdheden maken. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige beheerdersrechten maken.
  
Zie [CVE-2008-4036](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-4036) als u dit beveiligingslek wilt weergeven als standaardvermelding in de lijst met veelvoorkomende beveiligingslekken en blootstellingen.
  
#### Beperkende factoren voor het beveiligingslek dat kan leiden tot misbruik van bevoegdheden in de virtuele adresbeschrijving - CVE-2008-4036
  
Met beperkende factoren wordt verwezen naar een instelling, de algehele configuratie of algemene tips voor een standaardstatus waarmee de ernst van misbruik via een beveiligingslek kan worden verminderd. De volgende beperkende factoren kunnen in uw situatie van pas komen:
  
-   Een aanvaller moet zijn geverifieerd om misbruik van dit beveiligingslek te kunnen maken.
  
#### Tijdelijke oplossingen voor het beveiligingslek dat kan leiden tot misbruik van bevoegdheden in de virtuele adresbeschrijving - CVE-2008-4036
  
Microsoft heeft nog geen oplossingen voor dit beveiligingslek.
  
#### Veelgestelde vragen over het beveiligingslek dat kan leiden tot misbruik van bevoegdheden in de virtuele adresbeschrijving - CVE-2008-4036
  
**Wat is de omvang van het beveiligingslek?**    
Dit beveiligingslek heeft betrekking op misbruik van bevoegdheden. Een aanvaller die gebruik weet te maken van dit beveiligingslek, kan op het getroffen systeem misbruik van bevoegdheden maken. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige beheerdersrechten maken.
  
**Waardoor wordt het beveiligingslek veroorzaakt?**    
Door geheugenbeheer wordt een geheel-getaloverloop niet goed verwerkt wanneer een variabele wordt verlaagd, waardoor er een fout bij het toewijzen van geheugen optreedt.
  
**Wat is virtuele adresbeschrijving?**    
Virtuele adresbeschrijving (VAD) is een vorm van virtueel geheugen voor toepassingen waardoor elke toepassing zijn eigen privéadresruimte heeft. Zie de MSDN-artikelen [Virtual-memory Manager in Windows NT (Virtueel-geheugenbeheer in Windows NT)](http://msdn.microsoft.com/en-us/library/ms810616.aspx) en [Managing Virtual memory in Win32 (Virtueel geheugen beheren in Win32)](http://msdn.microsoft.com/en-us/library/ms810627.aspx) voor meer informatie over VAD
  
**Met welk doel kan een aanvaller het beveiligingslek misbruiken?**    
Een aanvaller die gebruik weet te maken van dit beveiligingslek, kan op het getroffen systeem misbruik van bevoegdheden maken. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige beheerdersrechten maken.
  
**Hoe kan een aanvaller dit beveiligingslek misbruiken?**    
Een geverifieerde aanvaller kan een speciaal vervaardigd programma uitvoeren waardoor er een geheel-getaloverloop kan ontstaan en het geheugen dermate beschadigd kan raken dat er externe code kan worden uitgevoerd.
  
**Voor welke systemen vormt dit beveiligingslek het grootste risico?**    
Windows XP, Windows Server 2003, Windows Vista en Windows Server 2008 zijn kwetsbaar voor dit beveiligingslek.
  
**Wat doet de update?**    
Het beveiligingslek is veroorzaakt door een geheel-getaloverloop. De update verwijdert het beveiligingslek door extra validatie rond VAD-parameters toe te voegen.
  
**Was dit beveiligingslek openbaar gemaakt op het moment dat dit beveiligingsbulletin werd uitgegeven?**    
Nee. Microsoft heeft uit discrete bronnen informatie over dit beveiligingslek ontvangen. Microsoft heeft geen informatie ontvangen die erop wees dat dit beveiligingslek openbaar was gemaakt op het moment dat dit beveiligingsbulletin oorspronkelijk werd uitgegeven.
  
**Was aan Microsoft gerapporteerd dat er misbruik was gemaakt van dit beveiligingslek op het moment dat dit beveiligingsbulletin werd uitgegeven?**    
Nee. Microsoft beschikte niet over informatie dat dit beveiligingslek was gebruikt voor aanvallen op klanten en had geen voorbeelden van gepubliceerde conceptcode gezien als bewijs toen dat dit beveiligingsbulletin oorspronkelijk werd uitgegeven.
