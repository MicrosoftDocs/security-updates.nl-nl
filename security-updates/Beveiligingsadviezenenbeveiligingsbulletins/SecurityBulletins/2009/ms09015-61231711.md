Security Bulletin

Microsoft-beveiligingsbulletin MS09-015 - Matig
===============================================

Dreiging van beveiligingslek in SearchPath kan leiden tot misbruik van bevoegdheden (959426)
--------------------------------------------------------------------------------------------

Gepubliceerd: dinsdag 14 april 2009 | Bijgewerkt: woensdag 15 april 2009

**Versie:** 1.1

### Algemene informatie

#### Samenvatting

Met deze beveiligingsupdate wordt een openbaar gemaakt beveiligingslek opgelost in de Windows SearchPath-functie waardoor misbruik van bevoegdheden kan optreden als een gebruiker een speciaal vervaardigd bestand naar een specifieke locatie downloadt en vervolgens een toepassing opent waardoor het bestand onder bepaalde omstandigheden kan worden geladen.

Dit is een gemiddelde beveiligingsupdate voro alle ondersteunde edities van Windows XP, Windows Server 2003, Windows Vista en Windows Server 2008 geschat. Op Microsoft Windows 2000 wordt deze update uitsluitend als ingrijpende wijziging beschouwd. Zie voor meer informatie het onderdeel **Veelgestelde vragen met betrekking tot deze Beveiligingsupdate**. Zie de subsectie **Software waarin dit probleem optreedt en software waarin dit probleem niet optreedt** in deze sectie voor meer informatie.

De beveiligingsupdate lost het beveiligingslek op door de manier te wijzigen waarop Windows bestanden van het bureaublad laadt. Zie de subsectie Veelgestelde vragen over het specifieke beveiligingslek in de volgende sectie, **Informatie over het beveiligingslek**, voor meer informatie over het beveiligingslek.

Deze beveiligingsupdate verwijdert ook het beveiligingslek dat voor het eerst werd beschreven in [Microsoft-beveiligingsadvies 953818](http://technet.microsoft.com/security/advisory/953818).

**Aanbeveling.** De meeste klanten hebben automatische updates ingeschakeld en hoeven geen actie te ondernemen omdat deze beveiligingsupdate automatisch wordt gedownload en geïnstalleerd. Klanten die automatische updates niet hebben ingeschakeld, moeten controleren op updates en deze update handmatig installeren. Zie voor informatie over specifieke configuratieopties in automatische updates [Microsoft Knowledge Base-artikel 294781](http://support.microsoft.com/kb/294871).

Als het gaat om beheerders en bedrijfinstallaties of eindgebruikers die deze beveiligingsupdate handmatig willen installeren, adviseert Microsoft klanten te overwegen de update toe te passen met behulp van de software voor updatebeheer of door op updates te controleren met de service [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Veelgestelde vragen met betrekking tot deze beveiligingsupdate
--------------------------------------------------------------

<span></span>
**Waar is de informatie over de bestandsgegevens?**  
De informatie over de bestandsgegevens vindt u in [Microsoft Knowledge Base-artikel 959426](http://support.microsoft.com/kb/959426).

**Welk verband bestaat er tussen deze beveiligingsupdate en MS09-014?**  
De dreiging van het beveiligingslek met betrekking tot misbruik van bevoegdheden, [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540), die wordt beschreven in dit beveiligingsbulletin, is ook van toepassing op Internet Explorer. Microsoft raadt aan dat gebruikers beveiligingsupdate [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659), Cumulatieve beveiligingsupdate voor Internet Explorer (963027), installeren.

**Bevat deze update andere functionaliteitswijzigingen met betrekking tot beveiliging?**  
Ja. Naast de wijzigingen die in de sectie **Informatie over het beveiligingslek** van dit bulletin worden beschreven, bevat deze update een ingrijpende wijziging met een nieuwe API, genaamd SetSearchPathMode.

**Wat staat de ingrijpende wijziging SetSearchPathMode toe?**  
Door deze nieuwe API is een per-procesmodus mogelijk bij het gebruik van de SearchPath-functie bij het zoeken naar bestanden. Hierdoor kunnen toepassingen de huidige map doorzoeken nadat de toepassings- en systeemlocaties zijn doorzocht.

**Als ik alleen deze beveiligingsupdate toepas, ben ik dan volledig beschermd tegen de dreiging die wordt beschreven in CVE-2008-2540?**  
Nee, u moet ook afzonderlijk de ingrijpende veiligheidsmaatregel van de cumulatieve beveiligingsupdate voor Internet Explorer, [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659), toepassen. U dient echter te controleren of toepassing van deze ingrijpende veiligheidsmaatregel tegen de dreiging geschikt is voor uw omgeving.

Om de ingrijpende veiligheidsmaatregel voor Internet Explorer toe te passen, dient u de SearchPath-functiebesturingssleutel in te schakelen die wordt beschreven in de sectie Veelgestelde vragen over deze beveiligingsupdate, in [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659). Zie het beveiligingslek met nummer [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540) voor meer informatie over de dreiging.

**Waarom staat Microsoft Windows 2000 zonder gevolgen van het beveiligingslek en zonder prioriteitsniveau vermeld in de tabel met software waarin dit probleem optreedt?**
Microsoft Windows 2000 wordt vermeld zonder gevolgen van het beveiligingslek en zonder prioriteitsniveau omdat deze update voor Microsoft Windows 2000 een andere oplossing heeft dan voor andere ondersteunde Windows-besturingssystemen. In plaats daarvan heeft deze update slechts een ingrijpende wijziging die specifiek is voor Microsoft Windows 2000.

**Waarom is de oplossing voor de dreiging in Microsoft Windows 2000 anders dan de oplossing voor andere platformen? **
De architectuur voor het implementeren van de SetSearchPathMode-functie bestaat niet in Microsoft Windows 2000, waardoor het beveiligingslek niet kan worden opgelost in Microsoft Windows 2000. Om het probleem op te lossen moet een significant deel van het Microsoft Windows 2000-besturingssysteem een nieuwe architectuur krijgen, niet alleen het onderdeel waarin dit probleem optreedt. Het eindresultaat van een dergelijke inspanning zou dusdanig incompatibel zijn met Microsoft Windows 2000 dat niet kan worden gegarandeerd dat toepassingen die zijn ontwikkeld voor Microsoft Windows 2000 nog kunnen worden uitgevoerd op het systeem met de nieuwe architectuur. Microsoft is op dit moment echter niet op de hoogte van geldige aanvalsvectoren die zich richten op dit beveiligingslek in Microsoft Windows 2000.

Door deze architecturele wijzigingen moet u voor Microsoft Windows 2000 de ingrijpende wijziging specifiek voor Microsoft Windows 2000 toepassen door de registersleutel HKLM\\System\\CCS\\control\\Session Manager\\SafeDLLSearchMode in te schakelen. Zie [Microsoft Knowledge Base-artikel 959426](http://support.microsoft.com/kb/959426) voor instructies. Zie het MSDN-artikel [Dynamic-Link Library Search Order](http://msdn.microsoft.com/en-us/library/ms682586.aspx) voor meer informatie.

**Wat is de SearchPath-functie?**  
De SearchPath-functie wordt gebruikt om te zoeken naar een specifiek bestand in een opgegeven pad. Zie het MSDN-artikel [SearchPath Function](http://msdn.microsoft.com/en-us/library/aa365527.aspx) voor meer informatie.

**Wat is de LoadLibrary-functie?**  
De LoadLibrary-functie laadt de opgegeven module in de adresruimte van het aanroepproces. De opgegeven module zorgt dat andere modules worden geladen. Zie het MSDN-artikel [LoadLibrary Function](http://msdn.microsoft.com/en-us/library/ms684175(vs.85).aspx) voor meer informatie.

Dreiging van beveiligingslek met betrekking tot misbruik van bevoegdheden - CVE-2008-2540  
-----------------------------------------------------------------------------------------
  
<span></span>
Er bestaat een dreiging van een beveiligingslek met betrekking tot misbruik van bevoegdheden door de manier waarop de SearchPath-functie in Windows bestanden op het systeem zoekt en opent. Een aanvaller kan misbruik maken van dit beveiligingslek door een gebruiker ervan te overtuigen een speciaal vervaardigd bestand te downloaden naar een specifieke locatie en vervolgens een toepassing te openen waardoor het bestand onder bepaalde omstandigheden kan worden geladen.
  
Zie [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540) als u dit beveiligingslek wilt weergeven als standaardvermelding in de lijst met veelvoorkomende beveiligingslekken en blootstellingen.
  
#### Beperkende factoren voor de dreiging van het beveiligingslek met betrekking tot misbruik van bevoegdheden - CVE-2008-2540:
  
Microsoft heeft nog geen beperkende factoren voor dit beveiligingslek.
  
#### Tijdelijke oplossingen voor de dreiging van het beveiligingslek met betrekking tot misbruik van bevoegdheden - CVE-2008-2540:
  
Microsoft heeft nog geen oplossingen voor dit beveiligingslek.
  
#### Veelgestelde vragen over de dreiging van het beveiligingslek met betrekking tot misbruik van bevoegdheden - CVE-2008-2540:
  
**Wat is de omvang van het beveiligingslek?**    
Dit is een dreiging van een beveiligingslek met betrekking tot misbruik van bevoegdheden. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan op afstand volledige controle krijgen over een systeem waarin dit probleem optreedt. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige gebruikersrechten maken.
  
**Waardoor wordt het beveiligingslek veroorzaakt?**    
Internet Explorer zou een speciaal vervaardigd bestand vanaf het bureaublad kunnen openen door de manier waarop de toepassingsbibliotheken zich in het Windows-systeem bevinden. Door een dreiging waarbij bestanden ongevraagd kunnen worden gedownload naar het systeem van een gebruiker kan Internet Explorer onbedoeld het gedownloade bestand vanaf het bureaublad laden in plaats van het bestand in het Windows-systeem.
  
**Met welk doel kan een aanvaller het beveiligingslek misbruiken?**    
Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan willekeurige code uitvoeren. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige gebruikersrechten maken.
  
**Hoe kan een aanvaller dit beveiligingslek misbruiken?**    
Een aanvaller kan een speciaal vervaardigd bestand maken en vervolgens een gebruiker ervan overtuigen het bestand te downloaden naar het bureaublad. De gebruiker moet vervolgens een toepassing starten waarmee dit speciaal vervaardigde bestand wordt geopend. Een aanvaller kan gebruikers echter niet dwingen dergelijke bestanden te downloaden en op het bureaublad te plaatsen.
  
**Voor welke systemen vormt dit beveiligingslek het grootste risico?**    
Alle systemen waarop gebruikers bestanden downloaden en toepassingen starten, zoals werkstations en terminalservers, lopen risico op dit beveiligingslek.
  
**Wat doet de update?**    
De update wijzigt de zoekvolgorde voor toepassingsbibliotheken, zodat de systeemmap wordt doorzocht voordat de huidige werkmappen worden doorzocht. De update biedt ook een ingrijpende wijziging, een nieuwe API genaamd SetSearchPathMode, om geavanceerde gebruikers te helpen de gewenste zoekvolgorde voor de SearchPath-functie in te stellen. Zie ook de sectie **Veelgestelde vragen met betrekking tot deze beveiligingsupdate** voor meer informatie over deze ingrijpende wijziging.
  
**Waarom is aan dit beveiligingslek en aan het beveiligingslek door bedreiging met betrekking tot het uitvoeren van externe code hetzelfde CVE-nummer (CVE-2008-2540) toegewezen in MS09-014, Cumulatieve beveiligingsupdate voor Internet Explorer (963027)?**  
[CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540) treft Internet Explorer 7 voor Windows XP Service Pack 2 en Windows XP Service Pack 3 en Internet Explorer 7 voor Windows XP Professional x64 Edition en Windows XP Professional x64 Edition Service Pack 2 enerzijds, en de SearchPath-functie van Windows anderzijds. Microsoft raadt aan dat gebruikers beveiligingsupdate [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659), Cumulatieve beveiligingsupdate voor Internet Explorer (963027), installeren. De ingrijpende wijziging voor dit beveiligingslek in deze update, de API SetSearchPathMode, komt ook overeen met de oplossing in [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659).
