Security Bulletin

Microsoft-beveiligingsbulletin MS09-063 - Kritiek
=================================================

Door een beveiligingslek met betrekking tot de API voor WSD (Web Services on Devices - webservices op apparaten) kan externe code worden uitgevoerd (973565)
------------------------------------------------------------------------------------------------------------------------------------------------------------

Gepubliceerd: dinsdag 10 november 2009

**Versie:** 1.0

### Algemene informatie

#### Samenvatting

Met deze beveiligingsupdate wordt een privé gemeld beveiligingslek in de API voor WSD (WSDAPI) op het Windows-besturingssysteem opgelost. Door het beveiligingslek kan externe code worden uitgevoerd als een getroffen Windows-systeem een speciaal vervaardigd pakket ontvangt. Alleen aanvallers op het lokale subnet kunnen misbruik maken van dit beveiligingslek.

Het prioriteitsniveau van deze beveiligingsupdate is Kritiek voor ondersteunde versies van Windows Vista en Windows Server 2008. Zie de subsectie **Software waarin dit probleem optreedt en software waarin dit probleem niet optreedt** in deze sectie voor meer informatie.

De beveiligingsupdate lost het beveiligingslek op door de verwerking van kopteksten in WSD-berichten te verbeteren. Zie de subsectie Veelgestelde vragen over het specifieke beveiligingslek in de volgende sectie, **Informatie over het beveiligingslek**, voor meer informatie over het beveiligingslek.

**Aanbeveling.** De meeste klanten hebben automatische updates ingeschakeld en hoeven geen actie te ondernemen omdat deze beveiligingsupdate automatisch wordt gedownload en geïnstalleerd. Klanten die automatische updates niet hebben ingeschakeld, moeten controleren op updates en deze update handmatig installeren. Zie [Microsoft Knowledge Base-artikel 294871](http://support.microsoft.com/kb/294871) voor informatie over specifieke configuratieopties voor het uitvoeren van automatische updates.

Als het gaat om beheerders en bedrijfinstallaties of eindgebruikers die deze beveiligingsupdate handmatig willen installeren, adviseert Microsoft klanten de update onmiddellijk toe te passen met behulp van de software voor updatebeheer of door op updates te controleren met de service [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Zie ook de sectie **Hulpmiddelen en richtlijnen voor detecteren en implementeren**, verderop in dit bulletin.

**Wat is Microsoft Web Services on Devices (WSD)?**  
Met Web Services on Devices kan een Windows-client via een netwerk externe apparaten detecteren en toegang krijgen tot deze apparaten, zoals PDA's en randapparatuur waaronder printers en camera's, evenals consumentenelektronica en de bijbehorende services. De service ondersteunt detectie, beschrijving en besturing van de apparaten. Ontwikkelaars kunnen WSDAPI-clientproxy's en overeenkomstige stubs maken voor apparaathosts.

De API voor WSD (WSDAPI) gebruikt het apparaatprofiel voor webservices (DPWS) voor Windows Vista en Windows Server 2008. Het DPWS beperkt de specificaties van webservices zodat Windows-clients apparaten eenvoudig kunnen detecteren. Zodra een apparaat is gedetecteerd, kan een client een beschrijving ophalen van de services van dat apparaat en gebruikmaken van die services.

Beveiligingslek met betrekking tot beschadigd geheugen in de API voor webservices voor apparaten - CVE-2009-2512  
----------------------------------------------------------------------------------------------------------------
  
<span></span>
Er bestaat een beveiligingslek met betrekking tot het uitvoeren van externe code in de API voor WSD (WSDAPI) op Windows-systemen. Het beveiligingslek ontstaat doordat de service een WSDAPI-bericht met een speciaal vervaardigde koptekst niet juist verwerkt. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan volledige controle krijgen over een systeem waarin dit probleem optreedt.
  
Zie [CVE 2009-2512](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2512) als u dit beveiligingslek wilt weergeven als standaardvermelding in de lijst met veelvoorkomende beveiligingslekken en blootstellingen.
  
#### Beperkende factoren voor het beveiligingslek met betrekking tot beschadigd geheugen in de API voor webservices voor apparaten - CVE-2009-2512
  
Met beperkende factoren wordt verwezen naar een instelling, de algehele configuratie of algemene tips voor een standaardstatus waarmee de ernst van misbruik via een beveiligingslek kan worden verminderd. De volgende beperkende factoren kunnen in uw situatie van pas komen:
  
-   Met aanbevolen procedures voor firewalls en standaardfirewallconfiguraties kunt u netwerken beveiligen tegen aanvallen die van buiten het bedrijfsnetwerk komen. Het wordt aanbevolen op de systemen die met internet zijn verbonden zo min mogelijk poorten bloot te stellen aan deze aanvallen.  
-   Alleen aanvallers op het lokale subnet kunnen zonder interactie misbruik maken van dit beveiligingslek. Zelfs op privénetwerken wordt de kwetsbare service alleen blootgesteld aan inkomende verbindingen van het lokale subnet in Windows Firewall.
  
#### Tijdelijke oplossingen voor het beveiligingslek met betrekking tot beschadigd geheugen in de API voor webservices voor apparaten - CVE-2009-2512
  
Tijdelijke oplossingen zijn wijzigingen in een instelling of configuratie waardoor het beveiligingslek zelf niet wordt gedicht, maar waarmee aanvalsvectoren wel worden geblokkeerd voordat u de update uitvoert. Microsoft heeft de volgende methoden en statussen om het probleem te omzeilen getest om uit te zoeken of een tijdelijke oplossing functionaliteit vermindert:
  
-   **Inkomende TCP-poorten 5357 en 5358 en uitgaande UDP-poort 3702 blokkeren**
  
    Configureer Windows Firewall zodanig dat inkomende TCP-poorten 5357 en 5358 en uitgaande UDP-poort 3702 worden geblokkeerd.
  
    Door de inkomende poorten te blokkeren, voorkomt de gebruiker dat schadelijke externe hosts speciaal vervaardigde kopteksten naar het getroffen onderdeel kunnen verzenden. De getroffen computer kan ook een verbinding tot stand brengen met externe, mogelijk schadelijke hosts wanneer de gebruiker naar apparaten op het subnet bladert. Door de uitgaande poort te blokkeren, voorkomt de gebruiker dat schadelijke hosts speciaal vervaardigde reacties in dit scenario kunnen verzenden.
  
    Er dient te worden opgemerkt dat een gebruiker nog steeds handmatig de URL van een schadelijk apparaat kan invoeren en een verbinding tot stand kan brengen waardoor het schadelijke apparaat een aanval kan uitvoeren op het systeem van de gebruiker. Voor deze aanval is een aanzienlijke mate van social engineering nodig.
  
    **Gevolgen van de tijdelijke oplossing:** De apparaatdetectie werkt niet zolang deze tijdelijke oplossing is ingeschakeld.
  
#### Veelgestelde vragen over het beveiligingslek met betrekking tot beschadigd geheugen in de API voor webservices voor apparaten - CVE-2009-2512
  
**Wat is de omvang van het beveiligingslek?**    
Dit beveiligingslek heeft betrekking op de uitvoering van externe code. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan op afstand volledige controle krijgen over een systeem waarin dit probleem optreedt. Een aanvaller die misbruik weet te maken van dit lek, kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige gebruikersrechten maken.
  
**Waardoor wordt het beveiligingslek veroorzaakt?**    
Dit beveiligingslek wordt veroorzaakt doordat de API voor webservices voor apparaten, op zowel clients als servers, specifieke kopteksten van een ontvangen WSD-bericht niet juist valideert.
  
**Wat is Microsoft Web Services on Devices (WSD)?**    
Met Web Services on Devices kan een Windows-client via een netwerk externe apparaten detecteren en toegang krijgen tot deze apparaten, zoals PDA's en randapparatuur waaronder printers en camera's, evenals consumentenelektronica en de bijbehorende services. De service ondersteunt detectie, beschrijving en besturing van de apparaten. Ontwikkelaars kunnen WSDAPI-clientproxy's en overeenkomstige stubs maken voor apparaathosts.
  
De API voor WSD (WSDAPI) gebruikt het apparaatprofiel voor webservices (DPWS) voor Windows Vista en Windows Server 2008. Het DPWS beperkt de specificaties van webservices zodat Windows-clients apparaten eenvoudig kunnen detecteren. Zodra een apparaat is gedetecteerd, kan een client een beschrijving ophalen van de services van dat apparaat en gebruikmaken van die services.
  
**Zijn webservices voor apparaten standaard ingeschakeld?**    
Webservices voor apparaten is een API (Application Programming Interface) die door verschillende systeemonderdelen kan worden gebruikt. Op alle getroffen platforms is de API standaard beschikbaar. Er kan echter alleen extern en zonder gebruikersinteractie misbruik worden gemaakt van het beveiligingslek wanneer een service of toepassing de API gebruikt en TCP-poort 5357 of 5358 blootstelt of externe apparaten probeert te detecteren. Dit is waarschijnlijk het geval in algemene scenario's waarbij ondersteunde apparaten zoals printers en videoapparatuur worden aangesloten op het netwerk.
  
**Met welk doel kan een aanvaller het beveiligingslek misbruiken?**    
Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan volledige controle krijgen over het Windows-systeem waarin dit probleem optreedt. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige gebruikersrechten maken.
  
**Hoe kan een aanvaller dit beveiligingslek misbruiken?**    
Een aanvaller kan proberen misbruik te maken van het beveiligingslek door een speciaal vervaardigd bericht naar WSD TCP-poorten 5357 of 5358 te verzenden op een getroffen systeem. Ook kan een aanvaller een speciaal vervaardigde reactie sturen op een WSD-bericht waarbij wordt gezocht naar apparaten, wanneer deze zoekopdracht wordt gestart door de Windows-client. Toepassingen die gebruikmaken van de WSDAPI, kunnen andere poorten dan de standaard TCP-poorten 5357 en 5358 gebruiken.
  
**Voor welke systemen vormt dit beveiligingslek het grootste risico?**    
Alle systemen die gebruikmaken van WSDAPI voor communicatie met DPWS-apparaten of -services op het netwerk, worden getroffen. Werkstations lopen met name risico wanneer Windows Firewall een andere instelling dan 'Openbaar' heeft.
  
**Wat doet de update?**    
De update lost dit beveiligingslek door kopteksten in WSD-berichten juist te verwerken.
  
**Was dit beveiligingslek openbaar gemaakt op het moment dat dit beveiligingsbulletin werd uitgegeven?**    
Nee. Microsoft heeft uit discrete bronnen informatie over dit beveiligingslek ontvangen.
  
**Was aan Microsoft gerapporteerd dat er misbruik was gemaakt van dit beveiligingslek op het moment dat dit beveiligingsbulletin werd uitgegeven?**    
Nee. Microsoft beschikte niet over informatie dat dit beveiligingslek was gebruikt voor aanvallen op klanten en had geen voorbeelden van gepubliceerde conceptcode gezien als bewijs toen dat dit beveiligingsbulletin oorspronkelijk werd uitgegeven.

De Application Compatibility Toolkit (ACT) bevat de noodzakelijke hulpprogramma's en documentatie om problemen met de compatibiliteit van toepassingen te evalueren en te verminderen voordat Microsoft Windows Vista, een Windows-update, een Microsoft-beveiligingsupdate of een nieuwe versie van Windows Internet Explorer op uw systeem wordt geïnstalleerd.
