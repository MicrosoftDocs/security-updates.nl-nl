Security Bulletin

Microsoft-beveiligingsbulletin MS10-009 - Kritiek
=================================================

Door beveiligingslekken in Windows TCP/IP kan externe code worden uitgevoerd (974145)
-------------------------------------------------------------------------------------

Gepubliceerd: dinsdag 9 februari 2010 | Bijgewerkt: woensdag 10 februari 2010

**Versie:** 1.1

### Algemene informatie

#### Samenvatting

Met deze beveiligingsupdate worden vier privé gemelde beveiligingslekken in Microsoft Windows opgelost. Als gevolg van de ernstigste beveiligingslekken kan externe code worden uitgevoerd als speciaal vervaardigde pakketten worden verzonden naar een computer waarop IPv6 is ingeschakeld. Een aanvaller kan proberen het beveiligingslek te misbruiken door speciaal vervaardigde ICMPv6-pakketten te maken en deze te verzenden naar een systeem waarop IPv6 is ingeschakeld. Dit beveiligingslek kan alleen worden misbruikt als de aanvaller on-link is.

Dit is een belangrijke beveiligingsupdate voor Windows Vista en Windows Server 2008. Zie de subsectie **Software waarin dit probleem optreedt en software waarin dit probleem niet optreedt** in deze sectie voor meer informatie.

De beveiligingsupdate lost de beveiligingslekken op doordat de manier wordt gewijzigd waarop Windows TCP/IP grenscontroles en andere pakketverwerkingsprocessen uitvoert. Zie de subsectie Veelgestelde vragen over het specifieke beveiligingslek in de volgende sectie, **Informatie over het beveiligingslek**, voor meer informatie over de beveiligingslekken.

**Aanbeveling.** De meeste klanten hebben automatische updates ingeschakeld en hoeven geen actie te ondernemen omdat deze beveiligingsupdate automatisch wordt gedownload en geïnstalleerd. Klanten die automatische updates niet hebben ingeschakeld, moeten controleren op updates en deze update handmatig installeren. Zie [Microsoft Knowledge Base-artikel 294871](http://support.microsoft.com/kb/294871) voor informatie over specifieke configuratieopties voor het uitvoeren van automatische updates.

Als het gaat om beheerders en bedrijfinstallaties of eindgebruikers die deze beveiligingsupdate handmatig willen installeren, adviseert Microsoft klanten de update onmiddellijk toe te passen met behulp van de software voor updatebeheer of door op updates te controleren met de service [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Zie ook de sectie **Hulpmiddelen en richtlijnen voor detecteren en implementeren**, verderop in dit bulletin.

Beveiligingslek met betrekking tot ICMPv6-router-advertisements - CVE-2010-0239  
-------------------------------------------------------------------------------
  
<span></span>
Er bestaat een beveiligingslek met betrekking tot het uitvoeren van externe code in de Windows TCP/IP-stack vanwege onvoldoende grenscontrole bij het verwerken van speciaal vervaardigde pakketten met ICMPv6-router-advertisements. Een anonieme aanvaller kan misbruik maken van het beveiligingslek door speciaal vervaardigde pakketten met ICMPv6-router-advertisements te verzenden naar een computer waarop IPv6 is ingeschakeld. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan volledige controle krijgen over een systeem waarin dit probleem optreedt. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige gebruikersrechten maken.
  
Zie [CVE 2010-0239](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0239) als u dit beveiligingslek wilt weergeven als standaardvermelding in de lijst met veelvoorkomende beveiligingslekken en blootstellingen.
  
#### Beperkende factoren voor het beveiligingslek met betrekking tot ICMPv6-router-advertisements - CVE-2010-0239
  
Microsoft heeft nog geen beperkende factoren voor dit beveiligingslek.
  
#### Tijdelijke oplossingen voor het beveiligingslek met betrekking tot ICMPv6-router-advertisements - CVE-2010-0239
  
Tijdelijke oplossingen zijn wijzigingen in een instelling of configuratie waardoor het beveiligingslek zelf niet wordt gedicht, maar waarmee aanvalsvectoren wel worden geblokkeerd voordat u de update uitvoert. Microsoft heeft de volgende methoden en statussen om het probleem te omzeilen getest om uit te zoeken of een tijdelijke oplossing functionaliteit vermindert:
  
-   **De firewallregel voor inkomend verkeer "Core Networking – Router Advertisement (ICMPv6-In)" uitschakelen**
  
    Voer de volgende opdracht uit vanaf een opdrachtprompt met verhoogde bevoegdheden om deze firewallregel uit te schakelen:
  
    `netsh advfirewall firewall set rule name="Core Networking – Router Advertisement (ICMPv6-In)" dir=in new enable=No`
  
    **Gevolgen van de tijdelijke oplossing:** ICMPv6-router-advertisements worden geblokkeerd. Dit kan een ongunstig effect hebben op de IPv6-functionaliteit in veelvoorkomende scenario's, waarbij het Router Discovery Protocol wordt gebruikt voor hostconfiguratie.
  
    **De tijdelijke oplossing ongedaan maken.**
  
    Voer de volgende opdracht uit vanaf een opdrachtprompt met verhoogde bevoegdheden om deze firewallregel weer in te schakelen:
  
    `netsh advfirewall firewall set rule name="Core Networking – Router Advertisement (ICMPv6-In)" dir=in new enable=Yes`
  
#### Veelgestelde vragen over het beveiligingslek met betrekking tot ICMPv6-router-advertisements - CVE-2010-0239
  
**Wat is de omvang van het beveiligingslek?**    
Dit beveiligingslek heeft betrekking op de uitvoering van externe code. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan willekeurige code uitvoeren door op kwetsbare systemen misbruik van bevoegdheden te maken. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige gebruikersrechten maken.
  
**Waardoor wordt het beveiligingslek veroorzaakt?**    
Het beveiligingslek wordt veroorzaakt doordat de Windows TCP/IP-stack onvoldoende grenscontroles uitvoert op speciaal vervaardigde pakketten met ICMPv6-router-advertisements.
  
**Wat is TCP/IP?**    
TCP/IP is een reeks netwerkprotocollen die veel op internet wordt gebruikt. TCP/IP biedt verbindingen over computernetwerken met verschillende hardware-architecturen en verschillende besturingssystemen. TCP/IP bevat standaarden voor de communicatie tussen computers en afspraken voor het verbinden van netwerken en het routeren van verkeer. Zie het TechNet-artikel [Overview of networking and TCP/IP](http://technet.microsoft.com/en-us/library/cc739443(ws.10).aspx) voor meer informatie over TCP/IP.
  
**Wat is IPv6?**    
Internet Protocol versie 6 (IPv6), een nieuwe suite van standaardprotocollen voor de netwerklaag van internet, is ingebouwd in Microsoft Windows XP en latere versies. IPv6 lost veel problemen met de huidige versie van IP op (bekend als IPv4) zoals adreslediging, beveiliging, automatische configuratie en uitbreidbaarheid. Voor meer informatie over IPv6 verwijzen wij u naar TechNet-website met [veelgestelde vragen over IPv6](http://technet.microsoft.com/en-us/network/bb530961.aspx).
  
**Wat zijn router-advertisements?**    
Met router-advertisements kunnen routers hosts instructies geven voor het uitvoeren van automatische adresconfiguratie. Routers kunnen bijvoorbeeld aangeven of hosts DHCPv6 en/of autonome (staatloze) adresconfiguratie moeten gebruiken. Routers kondigen hun aanwezigheid aan met diverse koppeling- en internetparameters, periodiek of in reactie op een routeraanvraag. Router-advertisements bevatten voorvoegsels die worden gebruikt om te bepalen of een ander adres dezelfde koppeling (koppelingsbepaling) en/of adresconfiguratie, een voorgestelde waarde voor de hoplimiet enz. deelt. Raadpleeg [RFC 4861](http://tools.ietf.org/html/rfc4861) voor meer informatie over router-advertisements en neighbor-detectie in IPv6.
  
**Met welk doel kan een aanvaller het beveiligingslek misbruiken?**    
Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan willekeurige code uitvoeren. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige gebruikersrechten maken.
  
**Hoe kan een aanvaller dit beveiligingslek misbruiken?**    
Een aanvaller kan proberen het beveiligingslek te misbruiken door speciaal vervaardigde ICMPv6-pakketten te maken en deze te verzenden naar een systeem waarop IPv6 is ingeschakeld. Dit beveiligingslek kan alleen worden misbruikt als de aanvaller on-link is. Tunneling-protocollen, zoals ISATAP, kunnen ervoor zorgen dat een aanvaller het schadelijke pakket kan leveren aan de doelcomputer, ook al maakt de aanvaller geen gebruik van dezelfde fysieke koppeling.
  
**Wanneer is een aanvaller met het systeem verbonden?**    
In verband met dit beveiligingslek is een aanvaller met het systeem verbonden als de aanvaller zich op dezelfde fysieke of virtuele koppeling bevindt en een geldig neighbor-detectiebericht naar de doelhost kan verzenden. Een voorbeeld van een virtuele koppeling is een koppeling via een ISATAP-tunnel. Raadpleeg [RFC 4861](http://tools.ietf.org/html/rfc4861) voor meer informatie over koppelingsadressen en neighbor-detectie in IPv6.
  
**Voor welke systemen vormt dit beveiligingslek het grootste risico?**    
Dit beveiligingslek vormt een risico voor alle besturingssystemen waarop de IPv6-functies zijn ingeschakeld. Standaard zijn alle ondersteunde functies voor IPv6 ingeschakeld in Windows Vista en Windows Server 2008.
  
**Wat doet de update?**    
De update lost dit beveiligingslek op doordat de manier wordt aangepast waarop de Windows TCP/IP-stack grenscontroles uitvoert op speciaal vervaardigde pakketten met ICMPv6-router-advertisements.
  
**Was dit beveiligingslek openbaar gemaakt op het moment dat dit beveiligingsbulletin werd uitgegeven?**    
Nee. Microsoft heeft uit discrete bronnen informatie over dit beveiligingslek ontvangen.
  
**Was aan Microsoft gerapporteerd dat er misbruik was gemaakt van dit beveiligingslek op het moment dat dit beveiligingsbulletin werd uitgegeven?**    
Nee. Microsoft heeft geen informatie ontvangen die erop wees dat dit beveiligingslek was gebruikt om aanvallen uit te voeren op klanten op het moment dat dit beveiligingsbulletin oorspronkelijk werd uitgegeven.
  
Beveiligingslek met betrekking tot MDL-fragmentatie van headers – CVE-2010-0240  
-------------------------------------------------------------------------------
  
<span></span>
Er bestaat een beveiligingslek met betrekking tot het uitvoeren van externe code in de Windows TCP/IP-stack als gevolg van de manier waarop de TCP/IP-stack speciaal vervaardigde ESP's (Encapsulating Security Payloads) over UDP-datagramfragmenten verwerkt wanneer er een aangepast netwerkstuurprogramma wordt uitgevoerd. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan volledige controle krijgen over een systeem waarin dit probleem optreedt. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige gebruikersrechten maken.
  
Zie [CVE 2010-0240](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0240) als u dit beveiligingslek wilt weergeven als standaardvermelding in de lijst met veelvoorkomende beveiligingslekken en blootstellingen.
  
#### Beperkende factoren voor het beveiligingslek met betrekking tot MDL-fragmentatie van headers - CVE-2010-0240
  
Met beperkende factoren wordt verwezen naar een instelling, de algehele configuratie of algemene tips voor een standaardstatus waarmee de ernst van misbruik via een beveiligingslek kan worden verminderd. De volgende beperkende factoren kunnen in uw situatie van pas komen:
  
-   Dit beveiligingslek treft alleen Windows-systemen met een aangepast netwerkstuurprogramma dat de UDP-header verdeelt in meerdere MDL's. Microsoft is niet op de hoogte van stuurprogramma's waarin deze handeling wordt uitgevoerd.
  
#### Tijdelijke oplossingen voor het beveiligingslek met betrekking tot MDL-fragmentatie van headers - CVE-2010-0240
  
Tijdelijke oplossingen zijn wijzigingen in een instelling of configuratie waardoor het beveiligingslek zelf niet wordt gedicht, maar waarmee aanvalsvectoren wel worden geblokkeerd voordat u de update uitvoert. Microsoft heeft de volgende methoden en statussen om het probleem te omzeilen getest om uit te zoeken of een tijdelijke oplossing functionaliteit vermindert:
  
-   **Geavanceerde TCP/IP-filtering inschakelen op systemen die deze functie ondersteunen**
  
    Om uw computer te beschermen tegen netwerkaanvallen waarin dit beveiligingslek wordt misbruikt, kunt u het beste geavanceerde TCP/IP-filtering inschakelen op systemen die deze functie ondersteunen. U kunt de functie voor geavanceerde TCP/IP-filtering inschakelen om al het ongewenste binnenkomende verkeer te blokkeren. Zie artikel [309798 in de Microsoft Knowledge Base](http://support.microsoft.com/kb/309798) voor meer informatie over het configureren van TCP/IP-filtering.
  
#### Veelgestelde vragen over het beveiligingslek met betrekking tot MDL-fragmentatie van headers - CVE-2010-0240
  
**Wat is de omvang van het beveiligingslek?**    
Dit beveiligingslek heeft betrekking op de uitvoering van externe code. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan willekeurige code uitvoeren door op kwetsbare systemen misbruik van bevoegdheden te maken. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige gebruikersrechten maken.
  
**Waardoor wordt het beveiligingslek veroorzaakt?**    
Het beveiligingslek wordt veroorzaakt doordat de Windows TCP/IP-stack speciaal vervaardigde IP-datagramfragmenten niet juist verwerkt wanneer een aangepast netwerkstuurprogramma wordt uitgevoerd.
  
**Wat is TCP/IP?**    
TCP/IP is een reeks netwerkprotocollen die veel op internet wordt gebruikt. TCP/IP biedt verbindingen over computernetwerken met verschillende hardware-architecturen en verschillende besturingssystemen. TCP/IP bevat standaarden voor de communicatie tussen computers en afspraken voor het verbinden van netwerken en het routeren van verkeer. Zie het TechNet-artikel [Overview of networking and TCP/IP](http://technet.microsoft.com/en-us/library/cc739443(ws.10).aspx) voor meer informatie over TCP/IP.
  
**Wat is een Encapsulating Security Payload (ESP)?**    
Encapsulating Security Payload (ESP) zorgt voor vertrouwelijkheid (naast verificatie, integriteit en bescherming tegen herhaling) voor de IP-nettolading. Bij ESP in de transportmodus wordt niet het volledige pakket ondertekend. Alleen de IP-nettolading (niet de IP-header) wordt beschermd. ESP kan afzonderlijk of in combinatie met Authentication Header (AH) worden gebruikt.
  
**Wat is een Memory Descriptor List (MDL)?**    
Een Memory Descriptor List (MDL) is een systeemgedefinieerde structuur die een buffer beschrijft aan de hand van een reeks fysieke adressen. Een stuurprogramma dat direct I/O uitvoert, ontvangt een pointer naar een MDL van de I/O-manager en leest en schrijft gegevens via de MDL. Bepaalde stuurprogramma's gebruiken ook MDL's wanneer zij direct I/O uitvoeren om te voldoen aan de I/O-controle van een apparaat. Raadpleeg het Windows Hardware Developer Central-artikel [What is really in that MDL?](http://www.microsoft.com/whdc/driver/tips/mdl.mspx) voor meer informatie over MDL's.
  
**Moet het UDP-verkeer voor deze aanval worden gefragmenteerd?**    
Nee. Voor deze aanval is geen gefragmenteerd netwerkverkeer nodig. In plaats daarvan moet het netwerkstuurprogramma op het doelssysteem de UDP-header in meerdere MDL's verdelen voor de fragmentatie.
  
**Met welk doel kan een aanvaller het beveiligingslek misbruiken?**    
Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan willekeurige code uitvoeren. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige gebruikersrechten maken.
  
**Hoe kan een aanvaller dit beveiligingslek misbruiken?**    
Een aanvaller kan proberen misbruik te maken van het beveiligingslek door speciale IP-datagramfragmenten te maken en de pakketten te verzenden naar een systeem waarop een aangepast netwerkstuurprogramma is geïnstalleerd.
  
**Voor welke systemen vormt dit beveiligingslek het grootste risico?**    
Dit beveiligingslek vormt een risico voor alle getroffen besturingssystemen waarop een aangepast netwerkstuurprogramma is geïnstalleerd dat de UDP-header verdeelt in meerdere MDL's. Er zijn geen Windows-netwerkstuurprogramma's die deze handeling kunnen uitvoeren. Microsoft is niet op de hoogte van netwerkstuurprogramma's die deze handeling wel kunnen uitvoeren.
  
**Wat doet de update?**

U kunt het testen en valideren van Windows-updates ten opzichte van geïnstalleerde toepassingen stroomlijnen met de [Update Compatibility Evaluator](http://technet2.microsoft.com/windowsvista/en/library/4279e239-37a4-44aa-aec5-4e70fe39f9de1033.mspx?mfr=true)-componenten die onderdeel zijn van [Application Compatibility Toolkit](http://www.microsoft.com/downloads/details.aspx?familyid=24da89e9-b581-47b0-b45e-492dd6da2971&displaylang=en).
  
De Application Compatibility Toolkit (ACT) bevat de noodzakelijke hulpprogramma's en documentatie om problemen met de compatibiliteit van toepassingen te evalueren en te verminderen voordat Microsoft Windows Vista, een Windows-update, een Microsoft-beveiligingsupdate of een nieuwe versie van Windows Internet Explorer op uw systeem wordt geïnstalleerd.
