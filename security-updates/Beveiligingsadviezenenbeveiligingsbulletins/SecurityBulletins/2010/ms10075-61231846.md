Security Bulletin

Microsoft-beveiligingsbulletin MS10-075 - Kritiek
=================================================

Door een beveiligingslek in de Media Player Network Sharing-service kan externe code worden uitgevoerd (2281679)
----------------------------------------------------------------------------------------------------------------

Gepubliceerd: dinsdag 12 oktober 2010

**Versie:** 1.0

### Algemene informatie

#### Samenvatting

Met deze beveiligingsupdate wordt een privé gemeld beveiligingslek in de Microsoft Windows Media Player Network Sharing-service opgelost. Door het beveiligingslek kan externe code worden uitgevoerd als een aanvaller een speciaal vervaardigd RTSP-pakket naar een getroffen systeem verzendt. Internettoegang tot media op computers thuis is echter standaard uitgeschakeld. In deze standaardconfiguratie kan het beveiligingslek alleen worden misbruikt door een aanvaller in hetzelfde subnet.

Het prioriteitsniveau van dit beveiligingsbulletin is Kritiek voor alle ondersteunde edities van Windows 7 en Belangrijk voor alle ondersteunde edities van Windows Vista. Zie de subsectie **Software waarin dit probleem optreedt en software waarin dit probleem niet optreedt** in deze sectie voor meer informatie.

De beveiligingsupdate lost het beveiligingslek op door de manier te verbeteren waarop de Windows Media Player Network Sharing-service speciaal vervaardigde RTSP-pakketten afhandelt. Zie de subsectie Veelgestelde vragen over het specifieke beveiligingslek in de volgende sectie, **Informatie over het beveiligingslek**, voor meer informatie over het beveiligingslek.

**Aanbeveling.** De meeste klanten hebben automatische updates ingeschakeld en hoeven geen actie te ondernemen omdat deze beveiligingsupdate automatisch wordt gedownload en geïnstalleerd. Klanten die automatische updates niet hebben ingeschakeld, moeten controleren op updates en deze update handmatig installeren. Zie [Microsoft Knowledge Base-artikel 294871](http://support.microsoft.com/kb/294871) voor informatie over specifieke configuratieopties voor het uitvoeren van automatische updates.

Als het gaat om beheerders en bedrijfinstallaties of eindgebruikers die deze beveiligingsupdate handmatig willen installeren, adviseert Microsoft klanten de update onmiddellijk toe te passen met behulp van de software voor updatebeheer of door op updates te controleren met de service [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Zie ook de sectie **Hulpmiddelen en richtlijnen voor detecteren en implementeren**, verderop in dit bulletin.

Beveiligingslek met betrekking tot RTSP Use After Free - CVE-2010-3225  
----------------------------------------------------------------------
  
<span></span>
Er bestaat een beveiligingslek in Microsoft Windows Media Player Network Sharing-service waardoor een externe gebruiker een speciaal vervaardigd netwerkpakket kan verzenden naar een exemplaar van de netwerkstreamingservice van de toepassing en externe code kan worden uitgevoerd in de context van de huidige toepassing.
  
Zie [CVE-2010-3225](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3225) als u dit beveiligingslek wilt weergeven als standaardvermelding in de lijst met veelvoorkomende beveiligingslekken en blootstellingen.
  
#### Beperkende factoren voor het beveiligingslek met betrekking tot RTSP Use After Free - CVE-2010-3225
  
Met beperkende factoren wordt verwezen naar een instelling, de algehele configuratie of algemene tips voor een standaardstatus waarmee de ernst van misbruik via een beveiligingslek kan worden verminderd. De volgende beperkende factoren kunnen in uw situatie van pas komen:
  
-   Internettoegang tot media op computers thuis is echter standaard uitgeschakeld. In deze standaardconfiguratie kan het beveiligingslek alleen worden misbruikt door computers in hetzelfde subnet.  
-   In ondersteunde edities van Windows Vista en op Windows 7 Professional-, Windows 7 Enterprise- en Windows 7 Ultimate-edities, is de Windows Media Player Network Sharing-service niet standaard ingeschakeld.
  
#### Tijdelijke oplossingen voor het beveiligingslek met betrekking tot RTSP Use After Free - CVE-2010-3225
  
Tijdelijke oplossingen zijn wijzigingen in een instelling of configuratie waardoor het beveiligingslek zelf niet wordt gedicht, maar waarmee aanvalsvectoren wel worden geblokkeerd voordat u de update uitvoert. Microsoft heeft de volgende methoden en statussen om het probleem te omzeilen getest om uit te zoeken of een tijdelijke oplossing functionaliteit vermindert:
  
-   **Schakel de Windows Media Player Network Sharing-service uit op getroffen systemen**
  
    **Interactieve methode**
  
    **Opmerking** Onjuist gebruik van de Register-editor kan ernstige problemen veroorzaken die ertoe kunnen leiden dat u het besturingssysteem opnieuw moet installeren. Microsoft kan niet garanderen dat problemen die voortkomen uit een onjuist gebruik van de Register-editor, kunnen worden opgelost. Het gebruik van de Register-editor is voor eigen risico. Zie voor meer informatie over het bewerken van het register het Help-onderwerp 'Sleutels en waarden wijzigen' in de Register-editor (regedit.exe) of de Help-onderwerpen 'Toevoegen of verwijderen van registerinformatie' en 'Registergegevens bewerken' in regedt32.exe.
  
    1.  Klik achtereenvolgens op **Start** en **Uitvoeren**, typ **Regedit** in het vak **Openen** en klik op **OK**.  
    2.  Zoek de volgende registersleutel op en klik erop:  
        HKEY\_LOCAL\_MACHINE\\SYSTEM\\CurrentControlSet\\services\\WMPNetworkSvc  
    3.  Klik op het menu **Bestand** en klik dan op **Exporteren**.  
    4.  Typ in het dialoogvenster **Registerbestand exporteren** de tekst **WMPNetworkSvc\_Backup.reg** en klik op **Opslaan**.  
        **Opmerking** Hierdoor wordt er standaard een back-up van deze registersleutel in de map Mijn documenten gemaakt.  
    5.  Selecteer de waarde **Start** in het rechtervenster van de Register-editor. Druk op **Enter** om de waarde van de sleutel te bewerken. Verander het gegevensveld **Waarde** in **4** en druk **Enter**.  
    6.  . Stop de service vanaf een opdrachtprompt met verhoogde bevoegdheden door de volgende opdracht in te voeren:  
        `sc stop WMPNetworkSvc`
  
    **Script voor een beheerde implementatie**
  
    1.  Maak een back-up van de registersleutel met de volgende opdracht:  
        `regedit /e WMPNetworkSvc_Backup.reg HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\services\WMPNetworkSvc`  
    2.  Sla de volgende gegevens op in een bestand met de extensie .REG, bijvoorbeeld Disable\_WMPNetworkSvc.reg:  
        Windows Registry Editor Version 5.00  
        HKEY\_LOCAL\_MACHINE\\System\\CurrentControlSet\\Services\\service\_name\]  
        Start"=dword:00000004  
    3.  Voer het registerscript dat u in stap 2 hebt gemaakt uit op het doelsysteem met de volgende opdracht:  
        `Regedit /s Disable_WMPNetworkSvc.reg`  
    4.  Stop de service vanaf een opdrachtprompt met verhoogde bevoegdheden door de volgende opdracht in te voeren:  
        `sc stop WMPNetworkSvc`
  
    **Gevolgen van de tijdelijke oplossing.** Windows Media Sharing wordt uitgeschakeld.
  
    **De tijdelijke oplossing ongedaan maken.**
  
    De interactieve methode gebruiken:
  
    1.  Klik achtereenvolgens op **Start** en **Uitvoeren**, typ **Regedit** in het vak **Openen** en klik op **OK**.  
    2.  Klik op het menu **Bestand** en klik dan op **Importeren**.  
    3.  Selecteer in het dialoogvenster **Registerbestand importeren** het bestand **WMPNetworkSvc\_Backup.reg** dat u eerder hebt gemaakt in 'Interactieve methode' en klik op **Openen**.  
    4.  Sluit Register-editor af en start de computer opnieuw op.
  
    Met het script voor een beheerde implementatie:
  
    1.  Voer het registerscript dat u in stap 2 hebt gemaakt uit op het doelsysteem met de volgende opdracht:  
        `Regedit /s WMPNetworkSvc_Backup.reg`  
    2.  Start de computer opnieuw op.
  
#### Veelgestelde vragen over het beveiligingslek met betrekking tot RTSP Use After Free - CVE-2010-3225
  
**Wat is de omvang van het probleem?**    
Dit beveiligingslek heeft betrekking op de uitvoering van externe code. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan externe code uitvoeren in de beveiligingscontext van de NetworkService-account.
  
**Waardoor wordt het beveiligingslek veroorzaakt?**    
Het beveiligingslek wordt veroorzaakt door de manier waarop de Microsoft Windows Media Player Network Sharing-service RTSP-pakketten (Real Time Streaming Protocol) verwerkt.
  
**Hoe kan een aanvaller dit beveiligingslek misbruiken?**    
Een aanvaller kan misbruik maken van het beveiligingslek door een speciaal vervaardigd RTSP-netwerkpakket te maken en naar een getroffen systeem te sturen.
  
**Wat is RTSP?**    
RTSP (Real Time Streaming Protocol) is een protocol op toepassingsniveau waarmee de bezorging van gegevens met realtime eigenschappen wordt bestuurd. RTSP biedt een flexibel raamwerk voor gecontroleerde bezorging op aanvraag van realtime gegevens, zoals audio en video. Gegevensbronnen kunnen zowel live gegevensfeeds als opgeslagen clips zijn. Dit protocol is bedoeld voor het beheer van meerdere gegevensbezorgingssessies, als een manier om leveringskanalen te kiezen, zoals UDP, multicast-UDP en TCP en als een manier om leveringssystemen te kiezen op basis van RTP (RFC 1889). RTSP is gedefinieerd in [RFC 2326](http://www.ietf.org/rfc/rfc2326.txt).
  
RTSP ondersteunt acties van spelerbesturingen, zoals het stoppen, onderbreken, terugspoelen en snel vooruit spoelen van geïndexeerde Windows Media-bestanden. Met RTSP kunt u inhoud streamen naar computers waarop Windows Media Player 9 Series of later of Windows Media Services 9 Series of later wordt uitgevoerd.
  
**Wat is de NetworkService-account?**    
De NetworkService-account is een ingebouwde account die meer toegang heeft tot bronnen en objecten dan leden van de groep Gebruikers. Services die als de NetworkService-account worden uitgevoerd, hebben toegang tot netwerkbronnen met behulp van de referenties van de computeraccount. De eigenlijke naam van de account is NT AUTHORITY\\NETWORK SERVICE. Zie het MSDN-artikel NetworkService Account voor meer informatie.
  
**Wat is de Windows Media Player Network Sharing-service?**    
Met Windows Media Player kunt u bestanden in uw bibliotheek delen met andere gebruikers in uw thuisnetwerk en bladeren naar bestanden in bibliotheken die andere gebruikers in uw netwerk delen. Windows Media Sharing maakt gebruikt van de Windows Media Player Network Sharing-service.
  
**Hoe kan ik bepalen of de Windows Media Player Network Sharing-service op mijn systeem wordt uitgevoerd?**    
Klik op **Start** en **Uitvoeren**, typ **Services.msc** in en klik vervolgens op **OK**. Controleer in het onderdeel Services van het Configuratiescherm of de Windows Media Player Network Sharing-service de status 'Gestart' heeft.
  
**Met welk doel kan een aanvaller het beveiligingslek misbruiken?**    
Een aanvaller die misbruik heeft gemaakt van dit beveiligingslek, kan de controle van het getroffen systeem overnemen in de beveiligingscontext van de NetworkService-account.
  
**Voor welke systemen vormt dit beveiligingslek het grootste risico?**    
Systemen waarop Windows Media Player Network Sharing-service (wmpnetwk.exe) wordt uitgevoerd om media te streamen lopen het meeste risico.
  
**Wat doet de update?**    
De update lost het beveiligingslek op door de manier te verbeteren waarop de Windows Media Player Network Sharing-service speciaal vervaardigde RTSP-pakketten afhandelt.
  
**Was dit beveiligingslek openbaar gemaakt op het moment dat dit beveiligingsbulletin werd uitgegeven?**    
Nee. Microsoft heeft uit betrouwbare bronnen informatie over dit beveiligingslek ontvangen.
  
**Was aan Microsoft gerapporteerd dat er misbruik was gemaakt van dit beveiligingslek op het moment dat dit beveiligingsbulletin werd uitgegeven?**    
Nee. Microsoft heeft geen informatie ontvangen die erop wees dat dit beveiligingslek was gebruikt om aanvallen uit te voeren op klanten op het moment dat dit beveiligingsbulletin oorspronkelijk werd uitgegeven.
