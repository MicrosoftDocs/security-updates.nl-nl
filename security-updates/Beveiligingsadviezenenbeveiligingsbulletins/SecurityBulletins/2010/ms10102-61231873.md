Security Bulletin

Microsoft-beveiligingsbulletin MS10-102 - Belangrijk
====================================================

Beveiligingslek in Hyper-V kan leiden tot denial of service (2345316)
---------------------------------------------------------------------

Gepubliceerd: dinsdag 14 december 2010 | Bijgewerkt: woensdag 12 januari 2011

**Versie:** 1.1

### Algemene informatie

#### Samenvatting

Deze beveiligingsupdate lost een privé gemeld beveiligingslek in Windows Server 2008 Hyper-V en Windows Server 2008 R2 Hyper-V op. Het beveiligingslek kan leiden tot een denial of service als een geverifieerde gebruiker in een van de virtuele gastmachines die door de Hyper-V-server worden gehost, een speciaal vervaardigd pakket naar de VMBus stuurt. Een aanvaller moet geldige aanmeldingsreferenties hebben en moet speciaal vervaardigde inhoud kunnen verzenden vanaf een virtuele gastmachine om misbruik te kunnen maken van dit beveiligingslek. Het beveiligingslek kan niet vanaf een externe locatie of door anonieme gebruikers worden misbruikt.

Het prioriteitsniveau van dit bulletin is Belangrijk voor alle ondersteunde x64-versies van Windows Server 2008 en Windows Server 2008 R2. Zie de subsectie **Software waarin dit probleem optreedt en software waarin dit probleem niet optreedt** in deze sectie voor meer informatie.

De beveiligingsupdate lost het beveiligingslek op doordat de manier wordt verbeterd waarop de Hyper-V-server verkeerd ingedeelde pakketten valideert die naar de VMBus in de virtuele gastmachines worden verzonden. Zie de subsectie Veelgestelde vragen over het specifieke beveiligingslek in de volgende sectie, **Informatie over het beveiligingslek**, voor meer informatie over het beveiligingslek.

**Aanbeveling.** De meeste klanten hebben automatische updates ingeschakeld en hoeven geen actie te ondernemen omdat deze beveiligingsupdate automatisch wordt gedownload en geïnstalleerd. Klanten die automatische updates niet hebben ingeschakeld, moeten controleren op updates en deze update handmatig installeren. Zie [Microsoft Knowledge Base-artikel 294871](http://support.microsoft.com/kb/294871) voor informatie over specifieke configuratieopties voor het uitvoeren van automatische updates.

Als het gaat om beheerders en bedrijfinstallaties of eindgebruikers die deze beveiligingsupdate handmatig willen installeren, adviseert Microsoft klanten de update zo snel mogelijk toe te passen met behulp van de software voor updatebeheer of door op updates te controleren met de service [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Zie ook de sectie **Hulpmiddelen en richtlijnen voor detecteren en implementeren**, verderop in dit bulletin.

**Bekende problemen.**  In [Microsoft Knowledge Base-artikel 2345316](http://support.microsoft.com/kb/2345316) worden bekende problemen beschreven die klanten mogelijk ondervinden bij de installatie van deze beveiligingsupdate. In het artikel worden ook de aanbevolen oplossingen voor deze problemen toegelicht.

Veelgestelde vragen met betrekking tot deze beveiligingsupdate
--------------------------------------------------------------

<span></span>
**Ik heb Hyper-V niet ingeschakeld. Kan ik deze update installeren?**
Nee. De update kan alleen worden geïnstalleerd op getroffen besturingssystemen waarop de Hyper-V-functie is ingeschakeld. Systemen waarop de Hyper-V-functie niet is ingeschakeld, worden niet door het beveiligingslek getroffen.

**Krijg ik deze update ook aangeboden als ik Hyper-V niet heb ingeschakeld?**  
Deze update wordt uitsluitend aangeboden via de functie Automatische updates als de Hyper-V-functie is ingeschakeld. Dit is van toepassing ongeacht of het getroffen besturingssysteem is geïnstalleerd met de ServerCore-installatieoptie.

**Moet deze update worden toegepast op virtuele Hyper-V-gastmachines?**  
Deze update hoeft niet te worden toegepast op virtuele Hyper-V-gastmachines, alleen op servers waarop de Hyper-V-functie is ingeschakeld. Via de functie Automatische updates wordt de update uitsluitend aangeboden aan het Hyper-V-serversysteem, niet aan de virtuele gastmachines.

**Wat is Hyper-V Server en wordt dit ook door dit beveiligingslek getroffen?**  
Microsoft Hyper-V Server is een zelfstandig product dat een betrouwbare en geoptimaliseerde virtualisatieoplossing biedt waarmee bedrijven hun servergebruik kunnen verbeteren en kosten kunnen verlagen. Hyper-V Server is een speciaal zelfstandig product dat alleen de Windows Hypervisor, het Windows Server-stuurprogrammamodel en virtualisatieonderdelen bevat. Daardoor is het product klein en zijn de overheadkosten laag.

Microsoft Hyper-V Server 2008, Microsoft Hyper-V Server 2008 Service Pack 2 en Microsoft Hyper-V Server 2008 R2 worden door dit beveiligingslek getroffen. De updates voor Windows Server 2008 en Windows Server 2008 R2 moeten worden toegepast voor bescherming tegen dit beveiligingslek. Klanten die de functie Automatische updates hebben ingeschakeld, krijgen deze updates automatisch ter installatie aangeboden.

**Wat zijn de bekende problemen die klanten mogelijk ondervinden wanneer zij deze beveiligingsupdate verwijderen?**  
Bij het verwijderen van deze update moet het systeem twee keer opnieuw worden opgestart vanwege bepaalde binaire bestanden in de update. Bij systemen die niet twee keer opnieuw zijn opgestart nadat de update is verwijderd, kan verlies van netwerkconnectiviteit optreden. Zie [Microsoft Knowledge Base-artikel 2345316](http://support.microsoft.com/kb/2345316) voor meer informatie over dit vastgestelde probleem.

Beveiligingslek in Hyper-V met betrekking tot VMBus - CVE-2010-3960  
-------------------------------------------------------------------
  
<span></span>
Er bestaat een beveiligingslek in Windows Server 2008 Hyper-V en Windows Server 2008 R2 Hyper-V dat kan leiden tot een denial of service als een speciaal vervaardigd pakket naar de VMBus wordt verzonden door een geverifieerde gebruiker in een van de virtuele gastmachines van de Hyper-V-server. Een aanvaller moet geldige aanmeldingsreferenties hebben en zich lokaal kunnen aanmelden bij een virtuele gastmachine om dit beveiligingslek te kunnen misbruiken. Het beveiligingslek kan niet vanaf een externe locatie of door anonieme gebruikers worden misbruikt.
  
Zie [CVE-2010-3960](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3960) als u dit beveiligingslek wilt weergeven als standaardvermelding in de lijst met veelvoorkomende beveiligingslekken en blootstellingen.
  
#### Beperkende factoren voor het beveiligingslek in Hyper-V met betrekking tot VMBus - CVE-2010-3960
  
Met beperkende factoren wordt verwezen naar een instelling, de algehele configuratie of algemene tips voor een standaardstatus waarmee de ernst van misbruik via een beveiligingslek kan worden verminderd. De volgende beperkende factoren kunnen in uw situatie van pas komen:
  
-   Een aanvaller moet geldige aanmeldingsreferenties hebben en zich lokaal bij een virtuele Hyper-V-machine kunnen aanmelden om dit beveiligingslek te kunnen misbruiken. Het beveiligingslek kan niet vanaf een externe locatie of door anonieme gebruikers worden misbruikt.
  
#### Tijdelijke oplossingen voor het beveiligingslek in Hyper-V met betrekking tot VMBus - CVE-2010-3960
  
Microsoft heeft nog geen oplossingen voor dit beveiligingslek.
  
#### Veelgestelde vragen over het beveiligingslek in Hyper-V met betrekking tot VMBus - CVE-2010-3960
  
**Wat is de omvang van het probleem?**    
Dit beveiligingslek heeft betrekking op een denial of service. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan ervoor zorgen dat de Hyper-V-server waarin dit probleem optreedt, niet meer reageert en opnieuw moet worden opgestart. Het beveiligingslek betreft een 'denial-of-service'-probleem. Een aanvaller kan geen code uitvoeren of zichzelf gebruikersrechten toekennen, maar kan er wel voor zorgen dat het systeem waarin dit probleem optreedt geen aanvragen meer accepteert.
  
**Waardoor wordt het beveiligingslek veroorzaakt?**    
Dit beveiligingslek wordt veroorzaakt wanneer de Hyper-V-server speciaal ingekapselde pakketten die worden verzonden naar de Hyper-V VMBus niet goed valideert. Vanwege dit gebrek aan validatie kan de Hyper-V-servertoepassing niet meer reageren als deze pakketten worden verwerkt.
  
**Wat is Hyper-V?**    
Hyper-V is een technologie op basis van hypervisor en vormt een hoofdfunctie van Windows Server 2008 en Windows Server 2008 R2. Het biedt een virtualisatieplatform met een hoge mate van schaalbaarheid, betrouwbaarheid en beschikbaarheid. Een virtualisatieomgeving is nuttig omdat gebruikers virtuele machines kunnen maken en beheren, waardoor ze meerdere besturingssystemen kunnen uitvoeren op één fysieke computer en de besturingssystemen van elkaar kunnen isoleren.
  
**Wat is VMBus?**    
De VMBus is een logisch inter-partitiecommunicatiekanaal. Zie het MSDN-artikel [Hyper-V Architecture](http://msdn.microsoft.com/en-us/library/cc768520(bts.10).aspx) voor meer informatie over VMBus en Hyper-V-architectuur.
  
**Met welk doel kan een aanvaller het beveiligingslek misbruiken?**    
Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan ervoor zorgen dat het systeem van een gebruiker niet meer reageert tot het opnieuw wordt opgestart. Door misbruik van dit beveiligingslek kan de Hyper-V-server niet meer reageren en daarmee ook alle virtuele gastmachines die worden gehost door die server.
  
**Hoe kan een aanvaller dit beveiligingslek misbruiken?**    
Een aanvaller moet een geverifieerde gebruiker zijn in een van de virtuele gastmachines die worden gehost door de Hyper-V-server en moet willekeurige code kunnen uitvoeren op het systeem. Een aanvaller moet vervolgens een aangepast stuurprogramma maken dat speciale pakketten maakt die worden verzonden naar de VMBus en daardoor zorgen dat de Hyper-V-server niet meer reageert.
  
**Voor welke systemen vormt dit beveiligingslek het grootste risico?**    
Dit beveiligingslek vormt het grootste risico voor Hyper-V-servers die virtuele gastmachines hosten die toegang bieden aan niet-vertrouwde gebruikers.
  
**Wat doet de update?**    
De update lost het beveiligingslek op doordat de manier wordt verbeterd waarop de Hyper-V-server speciaal vervaardigde pakketten die worden verzonden van de virtuele gastmachines naar de Hyper-V VMBus valideert en verwerkt.
  
**Was dit beveiligingslek openbaar gemaakt op het moment dat dit beveiligingsbulletin werd uitgegeven?**    
Nee. Microsoft heeft uit betrouwbare bronnen informatie over dit beveiligingslek ontvangen.
  
**Was aan Microsoft gerapporteerd dat er misbruik was gemaakt van dit beveiligingslek op het moment dat dit beveiligingsbulletin werd uitgegeven?**    
Nee. Microsoft heeft geen informatie ontvangen die erop wees dat dit beveiligingslek was gebruikt om aanvallen uit te voeren op klanten op het moment dat dit beveiligingsbulletin oorspronkelijk werd uitgegeven.
