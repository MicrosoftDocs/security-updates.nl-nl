Security Bulletin

Microsoft-beveiligingsbulletin MS05-043 - Kritiek
=================================================

Door een beveiligingslek in de Print Spooler-service kan leiden tot uitvoering van externe code (896423)
--------------------------------------------------------------------------------------------------------

Gepubliceerd: dinsdag 9 augustus 2005 | Bijgewerkt: woensdag 18 april 2007

**Versie:** 1.2

#### Samenvatting

**Voor wie is dit document bedoeld:** Klanten die Microsoft Windows gebruiken

**Gevolgen van het beveiligingslek:** Uitvoering van externe code mogelijk

**Maximaal prioriteitsniveau:** Kritiek

**Aanbeveling:** klanten moeten de update onmiddellijk toepassen.

**Vervanging van beveiligingsupdate:** Geen

**Voorbehoud:** In [Microsoft Knowledge Base-artikel 908506](http://support.microsoft.com/kb/908506) worden de bekende problemen beschreven die klanten mogelijk ondervinden bij de installatie van deze beveiligingsupdate. In het artikel worden ook de aanbevolen oplossingen voor deze problemen toegelicht. Zie [Microsoft Knowledge Base-artikel 908506](http://support.microsoft.com/kb/908506) voor meer informatie.

Informatie over de beveiligingslekken
-------------------------------------

<span></span>
#### Beveiligingslek in Print Spooler - CAN-2005-1984:

Er bestaat een beveiligingslek in de Print Spooler-service dat kan leiden tot het uitvoeren van externe code, waardoor een aanvaller die misbruik maakt van dit beveiligingslek volledige controle kan krijgen over het systeem waarin dit probleem optreedt.

#### Beperkende factoren voor beveiligingslek in Print Spooler - CAN-2005-1984:

-   In Windows XP Service Pack 2 en Windows Server 2003 is dit beveiligingslek beperkt tot geverifieerde gebruikers. Voordat dit probleem een externe aanvalsvector kan vormen, moet een gebruiker die de juiste rechten heeft bovendien eerst een printer delen of proberen verbinding te maken met een gedeelde printer. Als geen enkele gebruiker met de juiste rechten een printer deelt of probeert verbinding te maken met een gedeelde printer, kan een aanvaller dit beveiligingslek alleen misbruiken indien deze beschikt over geldige aanmeldingsreferenties en zich lokaal kan aanmelden.
-   In Windows XP Service Pack 2 en Windows Server 2003 kan dit probleem leiden tot een denial of service. In Windows XP Service Pack 2 and Windows Server 2003 kan dit probleem niet worden misbruikt om externe code uit te voeren of bevoegdheden te misbruiken.
    In andere besturingssysteemversies zouden aanvallen waarin wordt geprobeerd dit beveiligingslek te misbruiken kunnen leiden tot een denial of service. Het kan echter ook zijn dat in dit geval ook uitvoering van externe code mogelijk wordt.
-   Met aanbevolen procedures voor firewalls en standaardfirewallconfiguraties kunt u netwerken beveiligen tegen aanvallen die van buiten het bedrijfsnetwerk komen. Het wordt aanbevolen op de systemen die met internet zijn verbonden zo min mogelijk poorten bloot te stellen aan deze aanvallen.

#### Tijdelijke oplossingen voor het beveiligingslek in Print Spooler - CAN-2005-1984:

Microsoft heeft een aantal methoden getest om het probleem te omzeilen. Het probleem wordt er niet door verholpen, maar bekende aanvalsvectoren kunnen ermee worden geblokkeerd. Wanneer de functionaliteit wordt beperkt door een oplossing, wordt dat aangegeven in het volgende gedeelte.

-   **Schakel de Print Spooler-service uit**

    Door de Print Spooler-service uit te schakelen zorgt u ervoor dat er via dit beveiligingslek geen aanvallen op uw systeem kunnen worden uitgevoerd. Als u de Print Spooler-service wilt uitschakelen, gaat u als volgt te werk:

    1.  Klik op **Start** en klik vervolgens op **Configuratiescherm** .(Of klik op Start, wijs **Instellingen** aan en klik op **Configuratiescherm**.)
    2.  Dubbelklik op **Systeembeheer**.
    3.  Dubbelklik op **Services**.
    4.  Dubbelklik op **Print Spooler**.
    5.  Klik in de lijst **Opstarttype** op **Uitgeschakeld**.
    6.  Klik op **Stoppen** en op **OK**.

    U kunt de Print Spooler-service ook uitschakelen door de volgende opdracht achter de opdrachtprompt in te voeren:

    `sc stop Spooler & sc config Spooler start= disabled`

    **Gevolgen van de oplossing:** Als u de Print Spooler-service uitschakelt, kunt u niet lokaal of extern afdrukken. Wij raden u daarom aan deze tijdelijke oplossing alleen te gebruiken op systemen waarmee u niet hoeft af te drukken.

-   **In Windows 2000 Server Service Pack 4: verwijder de Print Spooler-service uit de registersleutel in NullSessionPipes:**
    Windows 2000 Server Service Pack 4 en Windows XP Service Pack 1 staan anonieme verbindingen met de getroffen service toe. Om te voorkomen dat dit beveiligingslek wordt misbruikt door anonieme aanvallers, dient u de Print Spooler-service te verwijderen uit de subsleutel in NullSessionPipes. Deze tijdelijke oplossing voorkomt niet dat geverifieerde gebruikers een aanval kunnen plegen. Gebruik deze tijdelijke oplossing alleen als u de Printer Spooler-service niet kunt uitschakelen.

    **Opmerking** Onjuist gebruik van de Register-editor kan ernstige problemen veroorzaken die ertoe kunnen leiden dat u het besturingssysteem opnieuw moet installeren. Microsoft kan niet garanderen dat problemen die voortkomen uit een onjuist gebruik van de Register-editor, kunnen worden opgelost. Het gebruik van de Register-editor is voor eigen risico. Zie voor meer informatie over het wijzigen van het register het Help-onderwerp 'Sleutels en waarden wijzigen' in de Register-editor (regedit.exe) of de Help-onderwerpen 'Toevoegen of verwijderen van registerinformatie' en 'Registergegevens bewerken' in regedt32.exe.

    **Opmerking** Het is aan te raden een back-up te maken van het register voordat u het bewerkt.

    1.  Klik op **Start** en op **Uitvoeren**, typ '**regedt32**' (zonder de aanhalingstekens) en klik vervolgens op **OK**.
    2.  Zoek in de Register-editor de volgende registersleutel:

        **HKEY\_LOCAL\_MACHINE\\SYSTEM\\CurrentControlSet\\Services\\LanmanServer\\Parameters\\NullSessionPipes**

    3.  Bewerk de registersleutel en verwijder de waarde **SPOOLSS**.
    4.  Start het systeem waarin dit probleem optreedt opnieuw op als u dit hebt gedaan.

    **Gevolgen van de oplossing:** Anonieme verbindingen met de Print Spooler-service zijn niet toegestaan. Dit is de standaardwerking van latere besturingssysteemversies.

#### Veelgestelde vragen over het beveiligingslek in Print Spooler - CAN-2005-1984:

**Wat is de omvang van het probleem?**

Dit beveiligingslek heeft betrekking op de uitvoering van externe code. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan op afstand volledige controle krijgen over een kwetsbaar systeem. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige gebruikersrechten maken. Misbruik van dit beveiligingslek zou kunnen resulteren in een denial of service.

**Waardoor wordt het beveiligingslek veroorzaakt?**
Een ongecontroleerde buffer in de Print Spooler-service.

**Wat is de Print Spooler-service?**
De Print Spooler-service, Spoolsv.exe, is een uitvoerbaar bestand dat wordt geïnstalleerd als een service. De spooler wordt geladen bij het opstarten van het systeem en blijft actief zolang het besturingssysteem actief is. De Print Spooler-service beheert het afdrukproces, met inbegrip van handelingen zoals het verkrijgen van de locatie van het juiste printerstuurprogramma, het laden van dat stuurprogramma, het in de wachtrij plaatsen van grote aanvragen in een afdruktaak, en het plannen van afdruktaken. Zodra de handelingen voor een bepaalde afdruktaak zijn voltooid, verzendt de Print Spooler-service de afdruktaak naar de afdrukrouter. Bezoek de volgende [website](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/techref/d58ce7b9-49cf-4f5e-95e9-1ade005c13e0.mspx) voor meer informatie over de Print Spooler-service.

**Met welk doel kan een aanvaller het beveiligingslek misbruiken?**
Een aanvaller die misbruik weet te maken van dit beveiligingslek waardoor uitvoering van externe code mogelijk wordt, kan volledige controle krijgen over het systeem waarin dit probleem optreedt. In Windows XP Service Pack 2 en Windows Server 2003 kan dit probleem leiden tot een denial of service. In andere besturingssysteemversies zouden pogingen om dit beveiligingslek te misbruiken kunnen leiden tot een denial of service. Het kan echter ook zijn dat in dit geval ook uitvoering van externe code mogelijk wordt.

**Wie kan dit beveiligingslek misbruiken?**
In Windows 2000 en Windows XP Service Pack 1 kan elke anonieme gebruiker die een speciaal samengesteld bericht op het systeem waarvoor dit probleem geldt kan afleveren, dit beveiligingslek misbruiken. In Windows XP Service Pack 2 en Windows Server 2003 is dit beveiligingslek beperkt tot geverifieerde gebruikers. In alle besturingssystemen waarin dit probleem optreedt kan een geverifieerde aanvaller zich ook lokaal op een systeem aanmelden en dit beveiligingslek misbruiken.

**Hoe kan een aanvaller dit beveiligingslek misbruiken?**
Een aanvaller kan het beveiligingslek op afstand misbruiken door een speciaal ontworpen bericht te maken en te verzenden naar een betrokken systeem. Door dit bericht zou het getroffen systeem vervolgens code kunnen uitvoeren in versies en configuraties van besturingssystemen die kwetsbaar zijn voor externe aanvalsvectoren. Standaard zijn Windows 2000 en Windows XP Service Pack 1 kwetsbaar voor aanvallen op afstand. Een externe aanvalsvector kan alleen worden gemaakt in Windows XP SP2 of in Windows Server 2003 als een gebruiker die de juiste machtiging heeft een printer deelt of probeert verbinding te maken met een gedeelde printer.
Om dit beveiligingslek lokaal te misbruiken in een willekeurige besturingssysteemversie moet een aanvaller zich eerst aanmelden bij het systeem. Een aanvaller zou dan een speciaal ontworpen programma kunnen uitvoeren om misbruik te maken van het beveiligingslek.

**Voor welke systemen vormt dit beveiligingslek het grootste risico?**
Dit beveiligingslek vormt het grootste risico voor Windows 2000 en Windows XP Service Pack 1. Systemen met Windows XP Service Pack 2 en Windows Server 2003 lopen minder risico, omdat in deze besturingssysteemversies aanvullende beperkende factoren van toepassing zijn. Systemen die als afdrukserver zijn geconfigureerd zijn echter extra kwetsbaar door dit beveiligingslek.

**Kan het beveiligingslek via internet worden misbruikt?**
Ja. Een aanvaller kan proberen via internet misbruik te maken van het beveiligingslek. Met aanbevolen procedures voor firewalls en standaardfirewallconfiguraties kunt u een betere beveiliging realiseren tegen aanvallen die vanaf internet komen. Microsoft biedt informatie over de beveiliging van uw pc. Eindgebruikers kunnen naar de volgende website gaan: [Beveilig uw pc](http://go.microsoft.com/fwlink/?linkid=21169). IT-professionals kunnen naar de volgende website gaan: [Security Guidance Center](http://go.microsoft.com/fwlink/?linkid=21171).

**Wat doet de update?**
Met de update verwijdert u het beveiligingslek door aanpassing van de manier waarop de Print Spooler-service de lengte van een bericht valideert voordat dit wordt doorgegeven aan de toegewezen buffer.

**Was dit beveiligingslek openbaar gemaakt op het moment dat dit beveiligingsbulletin werd uitgegeven?**
Nee. Microsoft heeft uit discrete bronnen informatie over dit beveiligingslek ontvangen. Microsoft heeft geen informatie ontvangen die erop wees dat dit beveiligingslek openbaar was gemaakt op het moment dat dit beveiligingsbulletin oorspronkelijk werd uitgegeven.

**Was aan Microsoft gerapporteerd dat er misbruik was gemaakt van dit beveiligingslek op het moment dat dit beveiligingsbulletin werd uitgegeven?**
Nee. Microsoft beschikte niet over informatie dat dit beveiligingslek was gebruikt voor aanvallen op klanten en had geen voorbeelden van gepubliceerde conceptcode gezien als bewijs toen dat dit beveiligingsbulletin oorspronkelijk werd uitgegeven.

Microsoft SMS (Systems Management Server) is een uitstekend configureerbare bedrijfsoplossing voor het beheer van updates. Met SMS kunnen beheerders bepalen of beveiligingsupdates nodig zijn voor Windows-systemen, en deze updates in de gehele organisatie gecontroleerd implementeren met minimaal ongemak voor de eindgebruikers. Ga naar de website [SMS 2003 Security Patch Management](http://go.microsoft.com/fwlink/?linkid=22939) voor meer informatie over hoe beheerders met SMS 2003 beveiligingsupdates kunnen implementeren. SMS 2.0-gebruikers kunnen ook het [Software Updates Service Feature Pack](http://go.microsoft.com/fwlink/?linkid=33340) gebruiken voor de implementatie van beveiligingsupdates. Ga naar de [SMS-website](http://go.microsoft.com/fwlink/?linkid=21158) voor informatie over SMS.

**Opmerking** SMS maakt gebruik van de Microsoft Baseline Security Analyzer, de Microsoft Office Detection Tool en de Enterprise Update Scan Tool om brede ondersteuning te kunnen bieden voor het zoeken en uitvoeren van beveiligingsupdates. Bepaalde software-updates worden mogelijk niet door deze hulpprogramma's gedetecteerd. Beheerders kunnen in deze gevallen de inventarisatiefuncties van SMS gebruiken om de updates op bepaalde systemen uit te voeren. Ga naar de volgende [website](http://go.microsoft.com/fwlink/?linkid=33341) voor meer informatie over deze procedure. Voor bepaalde beveiligingsupdates zijn beheerdersrechten vereist na het opnieuw opstarten van het systeem. Beheerders kunnen voor het installeren van deze updates gebruik maken van de Elevated Rights Deployment Tool (die deel uitmaakt van het [SMS 2003 Administration Feature Pack](http://go.microsoft.com/fwlink/?linkid=33387) en het [SMS 2.0 Administration Feature Pack](http://go.microsoft.com/fwlink/?linkid=21161)).
