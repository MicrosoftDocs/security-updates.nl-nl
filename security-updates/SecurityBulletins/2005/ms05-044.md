Security Bulletin

Microsoft-beveiligingsbulletin MS05-044 - Matig
===============================================

Door een beveiligingslek in de Windows FTP-client kan een bestandsoverdrachtlocatie onrechtmatig worden gewijzigd (905495)
--------------------------------------------------------------------------------------------------------------------------

Gepubliceerd: dinsdag 11 oktober 2005 | Bijgewerkt: woensdag 26 oktober 2005

**Versie:** 1.0

**Gepubliceerd:** 11 oktober 2005
**Versie:** 1.0

**Gevolgen van het beveiligingslek:** Onrechtmatige wijzigingen.

**Maximaal prioriteitsniveau:** Matig.

**Aanbeveling:** Klanten wordt aangeraden de beveiligingsupdate toe te passen.

Informatie over de beveiligingslekken
-------------------------------------

<span></span>
#### Beveiligingslek in de FTP-client - CAN-2005-2126:

Er komt een beveiligingslek voor in de FTP-client van Windows. Door dit beveiligingslek kan een aanvaller de beoogde doellocatie van een bestandsoverdracht wijzigen, wanneer een klant er handmatig voor heeft gekozen een bestand via FTP over te brengen. Hierdoor kan de aanvaller het bestand naar elk bestandssysteem schrijven op een systeem waarin dit probleem optreedt.

#### Beperkende factoren voor het beveiligingslek in de FTP-client - CAN-2005-2126:

-   Als een aanvaller erin is geslaagd een gebruiker over te halen naar een FTP-server te gaan die als host fungeert voor bestanden met speciaal geformuleerde bestandsnamen, kan de aanvaller geen bestanden overbrengen. Alleen een gebruiker kan een bestand overbrengen naar het systeem waarin dit probleem optreedt.
-   Als de schadelijke FTP-overdracht een bestaand bestand probeert te overschrijven in het systeem waarin dit probleem optreedt, wordt er een waarschuwing weergegeven dat het bestand wordt overschreven. Zodra dit waarschuwingsvenster wordt weergegeven, wordt het bestand alleen opgeslagen als de gebruiker daarvoor kiest.
-   De optie Mapweergave voor FTP-sites inschakelen van Internet Explorer is standaard uitgeschakeld in alle versies van besturingssystemen waarin dit probleem optreedt. Een aanval kan alleen lukken als de gebruiker de optie Mapweergave voor FTP-sites inschakelen van Internet Explorer handmatig heeft ingeschakeld in het systeem waarin dit probleem optreedt.

#### Tijdelijke oplossingen voor het beveiligingslek in de FTP-client - CAN-2005-2126:

Microsoft heeft een aantal methoden getest om het probleem te omzeilen. Het probleem wordt er niet door verholpen, maar bekende aanvalsvectoren kunnen ermee worden geblokkeerd. Wanneer de functionaliteit wordt beperkt door de oplossing, wordt dat aangegeven in het volgende gedeelte.

-   **Download geen bestanden van niet-vertrouwde FTP-servers**

    U kunt het risico op aanvallen beperken door alleen bestanden te downloaden van betrouwbare FTP-servers.

#### Veelgestelde vragen over het beveiligingslek in de FTP-client - CAN-2005-2126:

**Wat is de omvang van het probleem?**
Dit is een beveiligingslek dat betrekking heeft op onrechtmatige wijzigingen. Door dit beveiligingslek kan een aanvaller de beoogde doellocatie van een bestandsoverdracht wijzigen, wanneer een klant er handmatig voor heeft gekozen een bestand via FTP over te brengen.

**Waardoor wordt het beveiligingslek veroorzaakt?**
De FTP-client van Windows verifieert de namen van bestanden die afkomstig zijn van FTP-servers, niet op de juiste wijze.

**Met welk doel kan een aanvaller het beveiligingslek misbruiken?**
Een aanvaller die misbruik maakt van het beveiligingslek, kan bestanden opslaan op bepaalde locaties van een systeem waarin dit probleem optreedt. Hierdoor worden andere aanvallen mogelijk. Een aanvaller kan een uitvoerbaar bestand bijvoorbeeld opslaan in de map Opstarten. Vervolgens wordt dit overgebrachte bestand gestart wanneer een gebruiker zich aanmeldt.

**Wie kan dit beveiligingslek misbruiken?**
Iedereen die een gebruiker kan overhalen bestanden te openen en over te brengen die afkomstig zijn van een FTP-server die als host fungeert voor bestanden met speciaal geformuleerde bestandsnamen.

**Hoe kan een aanvaller dit beveiligingslek misbruiken?**
Een aanvaller kan misbruik maken van het beveiligingslek door op een FTP-server als host te fungeren voor een bestand met een speciaal geformuleerde bestandsnaam. De bestandsnaam moet zodanig zijn opgebouwd, dat deze de bestandsnaamverificatie door de FTP-client omzeilt en aan een geldige locatie op de computer van de gebruiker wordt toegewezen. Vervolgens moet een aanvaller de gebruiker zien over te halen dit bestand te downloaden.

**Kan het beveiligingslek automatisch worden misbruikt als ik naar een FTP-server ga?**
Nee. Alleen een gebruiker kan het bestand via FTP overbrengen en opslaan.

**Voor welke systemen vormt dit beveiligingslek het grootste risico?**
Vanwege dit beveiligingslek is het noodzakelijk dat een gebruiker verbinding maakt met een FTP-server en bestanden overbrengt met de FTP-server. Daarom lopen systemen waarin vaak FTP-overdrachten worden uitgevoerd, zoals werkstations, het grootste risico. Systemen die normaal gesproken niet worden gebruikt om naar FTP-servers te gaan (zoals de meeste serversystemen) lopen minder risico.

**Wat doet de update?**
Met de update wordt het beveiligingslek verwijderd door de wijze aan te passen waarop de Windows FTP-client bestandsnamen valideert die afkomstig zijn van FTP-servers.

**Was dit beveiligingslek openbaar gemaakt op het moment dat dit beveiligingsbulletin werd uitgegeven?**
Ja. Dit beveiligingslek is openbaar gemaakt. Aan dit beveiligingslek is nummer CAN-2005-2126 toegewezen

**Was aan Microsoft gerapporteerd dat er misbruik was gemaakt van dit beveiligingslek op het moment dat dit beveiligingsbulletin werd uitgegeven?**
Nee. Microsoft heeft geen informatie gehad dat dit beveiligingslek is gebruikt voor aanvallen op klanten. Er waren echter wel al voorbeelden van gepubliceerde conceptcode gepubliceerd toen dit beveiligingsbulletin werd uitgegeven.

**Beschermt het toepassen van deze beveiligingsupdate klanten tegen de openbaar gemaakte code die is gericht op misbruik van dit beveiligingslek?**
Ja. Deze beveiligingsupdate dicht het beveiligingslek waarvoor voorbeelden van gepubliceerde conceptcode als bewijs bestaan.
