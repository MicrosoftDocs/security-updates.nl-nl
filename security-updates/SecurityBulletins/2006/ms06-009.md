Security Bulletin

Microsoft-beveiligingsbulletin MS06-009 - Kritiek
=================================================

Beveiligingslek in de Koreaanse IME kan leiden tot misbruik van bevoegdheden (901190)
-------------------------------------------------------------------------------------

Gepubliceerd: dinsdag 14 februari 2006

**Versie:** 1.0

#### Samenvatting

**Voor wie is dit document bedoeld:** Klanten die Microsoft Windows of Microsoft Office 2003 gebruiken

**Gevolgen van het beveiligingslek:** Misbruik van bevoegdheden

**Maximaal prioriteitsniveau:** Belangrijk

**Aanbeveling:** Klanten wordt aangeraden deze update zo spoedig mogelijk te installeren.

**Vervanging van beveiligingsupdate:** Deze beveiligingsupdate vervangt MS06-003 speciaal voor de Koreaanse versie van de Microsoft Office 2003 Multilingual User Interface Pack.

**Voorbehoud:** Geen

**Opmerking** Standaard hebben alleen de Koreaanse taalversies van Windows last van dit beveiligingslek. Klanten die met een Oost-Aziatische taalversie van Windows werken, hebben het onderdeel waarin dit probleem optreedt, op het systeem, maar hun systeem is alleen kwetsbaar als de IME voor de Koreaanse taal is ingeschakeld. Klanten die een andere taalversie van Windows gebruiken hoeven alleen actie ondernemen als ze de IME voor de Koreaanse taal hebben geïnstalleerd en ingeschakeld.

**Opmerking** Dit probleem geldt alleen voor de Koreaanse taalversies van de genoemde producten van Office 2003, met uitzondering van de taalprogramma's van Office 2003. Klanten die de taalprogramma's van Microsoft Office 2003 hebben geinstalleerd dienen deze beveiligingsupdate te installeren, ook al is het Koreaanse onderdeel van het taalprogramma niet geïnstalleerd.

-   Microsoft Windows 98, Microsoft Windows 98 Tweede Editie en Microsoft Windows Millennium Edition (ME)
-   Microsoft Office XP Service Pack 3
-   Microsoft Office 2000 Service Pack 3

De software in deze lijst is getest om te controleren of het probleem bij deze versies optreedt. Andere versies worden niet langer door de beveiligingsupdate ondersteund of hierin treedt het probleem niet op. Informatie over de beveiligingslekken
-------------------------------------

<span></span>
#### Beveiligingslek in Koreaanse IME - CVE-2006-0008:

Er bevindt zich een beveiligingslek met betrekking tot [misbruik van bevoegdheden](http://go.microsoft.com/fwlink/?linkid=21142) in de Koreaanse Input Method Editor (IME) van Windows en Office. Als gevolg van dit beveiligingslek kan een kwaadwillende gebruiker volledige controle krijgen over een systeem waarvoor dit probleem geldt. Om een aanval te laten slagen moet een aanvaller zich interactief kunnen aanmelden op het systeem waarvoor dit probleem geldt.

#### Beperkende factoren voor het beveiligingslek in de KoreaanseIME - CVE-2006-0008:

-   Om misbruik te maken van dit beveiligingslek moet een aanvaller toegang tot het systeem hebben om zich interactief aan te melden, lokaal of via een Remote Desktop Protocol (RDP)-sessie.
-   Met aanbevolen procedures voor firewalls en standaardfirewallconfiguraties kunt u netwerken beveiligen tegen aanvallen die van buiten het bedrijfsnetwerk komen. Het wordt aanbevolen op de systemen die met internet zijn verbonden zo min mogelijk poorten bloot te stellen aan deze aanvallen.
-   RDP is standaard op geen enkele besturingssysteemversie ingeschakeld. Op Windows XP en Windows Server 2003 kan de functie Hulp op afstand het RDP-protocol inschakelen. Op Windows XP Media Center Edition is het RDP-protocol ingeschakeld als er een Media Center Extender is geïnstalleerd. Ga naar de volgende [website](http://msdn.microsoft.com/library/default.asp?url=/library/en-us/mcxwp/html/windows_media_center_extender_and_windows_media_connect__gruu.asp) voor meer informatie over Media Center Extenders.

    Op Small Business Server 2000 en Windows Small Business Server 2003 is RDP standaard ingeschakeld. Op Windows Small Business Server 2003 en eerdere versies zijn de communicatiepoorten voor het RDP-protocol echter geblokkeerd voor internet. RDP is alleen op het lokale netwerk beschikbaar tenzij Terminal Services of Externe Internet-werkplek zijn ingeschakeld met behulp van de wizard E-mail en Internet-verbinding.

-   Als Extern bureaublad handmatig wordt ingeschakeld worden afhankelijk van de besturingssysteemversie de volgende wijzigingen van kracht in Windows Firewall:
    -   Op Windows XP Service Pack 2-systemen waarop Windows Firewall is ingeschakeld, wordt door het activeren van de functie Extern bureaublad automatisch de uitzondering hiervoor ingeschakeld in de firewall. Deze uitzondering is van toepassing op **alle computers, inclusief computers op internet**. Als u Extern bureaublad uitschakelt, wordt de firewalluitzondering automatisch uitgeschakeld.

        Op Windows XP Service Pack 1, Windows Server 2003 en Windows Server 2003 Service Pack 1 wordt door het activeren van de functie Extern bureaublad de firewalluitzondering niet automatisch ingeschakeld. Bij deze besturingssystemen wordt er een dialoogvenster weergegeven waarin u deze uitzondering handmatig moet inschakelen. Er staat een vermelding voor Extern bureaublad in de lijst met firewalluitzonderingen die een gebruiker handmatig dient in te schakelen. Het uitschakelen van de functie Extern bureaublad verandert er niets aan de uitzonderingsstatus in de firewall. Hoewel er op dit systeem niet langer via Extern bureaublad gebruik kan worden gemaakt van dit beveiligingslek, kan dit nog wel gebeuren via Hulp op afstand en Terminal Services, als deze functies beschikbaar zijn.

#### Tijdelijke oplossingen voor het beveiligingslek in de Koreaanse IME - CVE-2006-0008:

Microsoft heeft een aantal methoden getest om het probleem te omzeilen. Het probleem wordt er niet door verholpen, maar bekende externe aanvalsvectoren kunnen ermee worden geblokkeerd. Wanneer de functionaliteit wordt beperkt door de oplossing, wordt dat aangegeven in het volgende gedeelte.

-   **Schakel Terminal Services, Extern bureaublad, Hulp op afstand, en Externe Internet-werkplek (in Windows Small Business Server 2003) uit als u deze niet langer nodig hebt.**

    Als u deze services niet langer nodig hebt, is het een goed idee deze uit te schakelen. Hierdoor bent u beter beveiligd tegen beveiligingslekken.

    Ga naar de volgende [website](http://www.microsoft.com/resources/documentation/windows/xp/all/proddocs/en-us/rdesktop_disable.mspx) voor informatie over het handmatig uitschakelen van Extern bureaublad.

    Lees het volgende [Microsoft Knowledge Base-artikel](http://support.microsoft.com/kb/306300) voor informatie over het uitschakelen van Extern bureaublad met behulp van Groepsbeleid.

    Ga naar de volgende [website](http://www.microsoft.com/technet/prodtechnol/winxppro/maintain/intmgmt/20_xprem.mspx) voor informatie over Hulp op afstand en het handmatig en via Groepsbeleid uitschakelen van deze functie.

    Ga naar de volgende [website](http://www.microsoft.com/technet/security/secnews/articles/sec_sbs2003_network.mspx) voor informatie over de functies Terminal Services en Externe Internet-werkplek in Windows Small Business Server 2003.

-   **TCP-poort 3389 blokkeren op de bedrijfsfirewall.**

    Deze poort wordt gebruikt om een verbinding tot stand te brengen met het onderdeel waarin dit probleem optreedt. Door deze poort op de firewall te blokkeren, zorgt u ervoor dat op systemen achter deze firewall geen aanvallen kunnen worden uitgevoerd. Zo kunt u netwerken beveiligen tegen aanvallen die van buiten het bedrijfsnetwerk komen. Het blokkeren van de getroffen poorten op de firewall is de beste verdediging tegen internetaanvallen. Systemen kunnen echter nog steeds van binnen het bedrijf worden aangevallen. Op Windows XP en Windows Server 2003 kan Windows Firewall helpen bij de beveiliging van individuele computers. Windows Firewall staat standaard geen verbindingen toe via deze poort, behalve in Windows XP Service Pack 2 als de functie Extern bureaublad is ingeschakeld. Ga naar de volgende [website](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/serverhelp/8b5e3b52-b77b-4d98-a058-b8083f393583.mspx) voor informatie over hoe u de uitzondering voor de functie Extern bureaublad kunt uitschakelen in Windows Firewall. Als u de uitzondering voor de functie Extern bureaublad niet kunt uitschakelen in Windows Firewall, kunt u wellicht de gevolgen van het beveiligingslek beperken door de standaardwaarde in te stellen op de optie voor het lokale netwerk in plaats van op de optie voor **alle computers (inclusief computers op internet)**. Hierdoor verlaagt u de kans op aanvallen via internet.

    **Opmerking** In Windows Small Business Server 2003 zit een functie genaamd Externe Internet-werkplek. Deze functie gebruikt TCP-poort 4125 voor het verwerken van RDP-verbindingen. Als u deze functie gebruikt, dient u te controleren dat behalve poort 3389 ook deze poort is geblokkeerd voor internet.

    **Opmerking** U hebt wellicht andere poorten ingesteld voor de getroffen onderdelen. Als u dat hebt gedaan, moet u ook deze poorten blokkeren.

-   **Beveilig externe bureaubladverbindingen via IPsec-beleid.**

    Specifieke configuraties zijn afhankelijk van de individuele omgeving. Ga naar de volgende [website](http://www.microsoft.com/windowsserver2003/technologies/networking/ipsec/default.mspx) voor meer informatie over Internet Protocol Security (IPsec). Zie de Microsoft Knowledge Base-artikelen [313190](http://support.microsoft.com/kb/313190) en [813878](http://support.microsoft.com/kb/813878) voor uitgebreide informatie over IPSec en het toepassen van filters.

-   **Beveilig externe bureaubladverbindingen via een VPN-verbinding.**

    Specifieke configuraties zijn afhankelijk van de individuele omgeving. Ga naar de volgende [website](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/serverhelp/a08da8ea-a616-4422-bbd7-9cb8de066b29.mspx) voor meer informatie over virtuele particuliere netwerken.

#### Veelgestelde vragen over het beveiligingslek in de Koreaanse IME - CVE-2006-0008:

**Wat is de omvang van het probleem?**  
[Dit beveiligingslek heeft betrekking](http://go.microsoft.com/fwlink/?linkid=21142)op misbruik van bevoegdheden. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan volledige controle krijgen over een systeem waarin dit probleem optreedt. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige gebruikersrechten maken.

**Waardoor wordt het beveiligingslek veroorzaakt?**  
Onder bepaalde omstandigheden stelt de Input Method Editor (IME) voor de Koreaanse taal ongewenst functionaliteit bloot die wordt uitgevoerd in de context van LocalSystem.

**Wat is de Koreaanse IME?**  
Met IME's wordt een probleem opgelost dat te maken heeft met het invoeren van informatie in bepaalde talen via een toetsenbord. Talen als het Koreaans bestaan uit duizenden verschillende tekens, en die passen nu eenmaal niet allemaal op een toetsenbord. Met IME's kunnen de tekens worden opgebouwd aan de hand van een gewoon toetsenbord met 101 toetsen door de penseelstreken op te geven waaruit het teken is samengesteld.
Een IME bestaat uit een engine die toetsaanslagen omzet in fonetische en ideografische tekens en een woordenboek van veelgebruikte ideografische woorden. De toetsen die de gebruiker intoetst, worden door de IME herkend en omgezet in tekens.

**Welke IME's hebben last van dit beveiligingslek?**  
Dit probleem geldt alleen voor de IME voor de Koreaanse taal. De IME's voor alle andere talen herkennen wanneer ze actief zijn als deel van het aanmeldscherm, en stellen alleen de gewenste functies bloot.

**Waarom heeft Office 2003 ook last van het beveiligingslek in de Koreaanse IME?**  
Office 2003 kan met zijn meertalige capaciteiten zijn eigen Koreaanse IME installeren, die de versie van de Koreaanse IME uit het besturingssysteem overschrijft. Klanten die de Koreaanse IME van Office 2003 hebben geïnstalleerd kunnen het beste de Office-beveiligingsupdate uit dit beveiligingsbulletin zo spoedig mogelijk installeren.

**Met welk doel kan een aanvaller het beveiligingslek misbruiken?**  
Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan volledige controle krijgen over het systeem waarin dit probleem optreedt.

**Wie kan dit beveiligingslek misbruiken?**  
Een gebruiker die toegang heeft tot het aanmeldscherm van een systeem waarvoor dit probleem geldt, lokaal of op afstand.

**Hoe kan een aanvaller dit beveiligingslek misbruiken?**  
Een kwaadwillende gebruiker die lokaal of via Remote Desktop of Terminal Services toegang heeft tot het aanmeldscherm van een systeem waarvoor dit probleem geldt, kan deze functionaliteit gebruiken om code uit te voeren voor elke gewenste actie op het systeem.

**Voor welke systemen vormt dit beveiligingslek het grootste risico?**  
Voor werkstations en terminalservers waarop de IME voor de Koreaanse taal is geïnstalleerd, is het risico het grootst.

**Worden Windows 98, Windows 98 Tweede Editie of Windows Millennium Edition zwaar getroffen door dit beveiligingslek?**  
Nee. Windows 98, Windows 98 Tweede Editie en Windows Millennium Edition bevatten het betreffende onderdeel niet.

**Kan het beveiligingslek via internet worden misbruikt?**  
Ja. Een aanvaller kan proberen via internet misbruik te maken van het beveiligingslek. Met aanbevolen procedures voor firewalls en standaardfirewallconfiguraties kunt u een betere beveiliging realiseren tegen aanvallen die vanaf internet komen. Microsoft biedt informatie over de beveiliging van uw pc. Eindgebruikers kunnen naar de volgende website gaan: [Beveilig uw pc](http://go.microsoft.com/fwlink/?linkid=21169). IT-professionals kunnen naar de volgende website gaan: [TechNet Security Center](http://go.microsoft.com/fwlink/?linkid=21171).

**Wat doet de update?**  
De update verwijdert het beveiligingslek door de Koreaanse IME-functionaliteit die wordt uitgevoerd in de context van LocalSystem uit te schakelen.

**Was dit beveiligingslek openbaar gemaakt op het moment dat dit beveiligingsbulletin werd uitgegeven?**  
Nee. Microsoft heeft uit discrete bronnen informatie over dit beveiligingslek ontvangen. Microsoft heeft geen informatie ontvangen die erop wees dat dit beveiligingslek openbaar was gemaakt op het moment dat dit beveiligingsbulletin oorspronkelijk werd uitgegeven.

**Was aan Microsoft gerapporteerd dat er misbruik was gemaakt van dit beveiligingslek op het moment dat dit beveiligingsbulletin werd uitgegeven?**  
Nee. Microsoft heeft geen informatie gehad dat dit beveiligingslek is gebruikt voor aanvallen op klanten.

| **/q**       | Hiermee schakelt u de stille modus in, of onderdrukt u vragen aan de gebruiker wanneer bestanden worden uitgepakt.                      |

    *\[pad\\naam van EXE-bestand\]* /c /t:C:\\AdminUpdate
    **Opmerking**  Het MSP-bestand wordt niet uitgepakt als u op het EXE-bestand dubbelklikt. Als u dit doet, wordt de update geïnstalleerd op de lokale computer. Als u een update wilt uitvoeren voor een beheerdersimage moet u eerst het MSP-bestand uitpakken.

    msiexec /a *Admin Path\\MSI File* /p C:\\adminupdate\\*MSP File* SHORTFILENAMES=TRUE
