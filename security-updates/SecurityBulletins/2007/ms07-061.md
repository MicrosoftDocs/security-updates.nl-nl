Security Bulletin

Microsoft-beveiligingsbulletin MS07-061 - Kritiek
=================================================

Door een beveiligingslek in Windows met betrekking tot de afhandeling van URI's kan externe code worden uitgevoerd (943460)
---------------------------------------------------------------------------------------------------------------------------

Gepubliceerd: dinsdag 13 november 2007 | Bijgewerkt: woensdag 16 januari 2008

**Versie:** 1.2

### Algemene informatie

#### Samenvatting

Met deze update wordt een openbaar gemaakt beveiligingslek opgelost. Er bestaat een beveiligingslek met betrekking tot het uitvoeren van externe code dat wordt veroorzaakt door de manier waarop de Windows-shell speciaal vervaardigde URI's afhandelt die aan de shell worden doorgegeven. Wanneer de Windows-shell de URI's onvoldoende valideert, kan een aanvaller het beveiligingslek misbruiken en code uitvoeren. Microsoft heeft alleen ontdekt op welke manieren dit beveiligingslek kan worden misbruikt op systemen met Internet Explorer 7. Het beveiligingslek bevindt zich echter in een Windows-bestand, Shell32.dll, dat onderdeel uitmaakt van alle ondersteunde edities van Windows XP en Windows Server 2003.

Dit is een kritieke beveiligingsupdate voor alle ondersteunde edities van Windows XP en Windows Server 2003. Zie de subsectie **Software waarin dit probleem optreedt en software waarin dit probleem niet optreedt** in deze sectie voor meer informatie.

De beveiligingsupdate dicht het beveiligingslek door de wijze te veranderen waarop ongeldige URI's worden verwerkt door de Windows-shell. Zie de subsectie Veelgestelde vragen in de volgende sectie, **Informatie over het beveiligingslek**, voor meer informatie over het beveiligingslek.

Deze beveiligingsupdate biedt ook een oplossing voor het beveiligingslek dat voor het eerst is beschreven in [Microsoft-beveiligingsadvies 943521](http://technet.microsoft.com/security/advisory/943521). Zie de subsectie Veelgestelde vragen over het specifieke beveiligingslek in de volgende sectie voor meer informatie over het beveiligingslek.

**Aanbeveling.** Microsoft raadt klanten aan de update onmiddellijk toe te passen.

**Bekende problemen.** In [Microsoft Knowledge Base-artikel 9434601](http://support.microsoft.com/kb/943460) worden bekende problemen beschreven die klanten mogelijk ondervinden bij de installatie van deze beveiligingsupdate. In het artikel worden ook de aanbevolen oplossingen voor deze problemen toegelicht.

Beveiligingslek in Windows met betrekking tot de afhandeling van URI's - CVE-2007-3896
--------------------------------------------------------------------------------------

<span></span>
Er bestaat een beveiligingslek met betrekking tot het uitvoeren van externe code dat wordt veroorzaakt door de manier waarop de Windows-shell speciaal vervaardigde URI's afhandelt die aan de shell worden doorgegeven. Een aanvaller kan misbruik maken van dit beveiligingslek door een speciaal ontworpen URI in een toepassing of bijlage toe te voegen waardoor programmacode op afstand kan worden uitgevoerd.

Zie [CVE-2007-3896](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-3896) als u dit beveiligingslek wilt weergeven als standaardvermelding in de lijst met veelvoorkomende beveiligingslekken en blootstellingen.

#### Beperkende factoren voor het beveiligingslek in Windows met betrekking tot de afhandeling van URI's - CVE-2007-3896

Met beperkende factoren wordt verwezen naar een instelling, de algehele configuratie of algemene tips voor een standaardstatus waarmee de ernst van misbruik via een beveiligingslek kan worden verminderd. De volgende beperkende factoren kunnen in uw situatie van pas komen:

-   Windows 2000 Service Pack 4 heeft geen last van dit beveiligingslek
-   Windows Vista heeft geen last van dit beveiligingslek
-   Windows Vista x64 Edition heeft geen last van dit beveiligingslek
-   Microsoft heeft geen manier ontdekt waarop dit beveiligingslek kan worden misbruikt op enig Windows-besturingssysteem waarop Microsoft Internet Explorer versie 6 wordt uitgevoerd.
-   Een aanvaller die erin slaagt misbruik te maken van dit beveiligingslek, kan dezelfde rechten over het systeem krijgen als de lokale gebruiker. Gebruikers met accounts waarvoor minder gebruikersrechten op het systeem zijn ingesteld, lopen minder risico dan gebruikers die met beheerdersrechten werken.
-   Klanten die e-mailberichten lezen in de indeling Tekst zonder opmaak lopen minder risico bij een aanval van deze exploitatiecode via een e-mailbericht. Gebruikers lopen pas meer risico als gevolg van dit beveiligingslek wanneer zij op een koppeling klikken waarmee ze naar een schadelijke website gaan of een bijlage openen.

#### Tijdelijke oplossingen voor het beveiligingslek in Windows met betrekking tot de afhandeling van URI's - CVE-2007-3896

Microsoft heeft nog geen oplossingen voor dit beveiligingslek.

#### Veelgestelde vragen over het beveiligingslek in Windows met betrekking tot de afhandeling van URI's - CVE-2007-3896

**Wat is de omvang van het beveiligingslek?**  
Dit beveiligingslek heeft betrekking op de uitvoering van externe code. Een aanvaller die erin slaagt misbruik te maken van dit beveiligingslek kan dezelfde rechten op het systeem krijgen als de aangemelde gebruiker.

Als er een gebruiker met beheerdersrechten is aangemeld, kan een aanvaller die misbruik weet te maken van dit beveiligingslek, volledige controle krijgen over een systeem waarin dit probleem optreedt. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige gebruikersrechten maken. Gebruikers met accounts waarvoor minder gebruikersrechten op het systeem zijn ingesteld, lopen minder risico dan gebruikers die met beheerdersrechten werken.

**Waardoor wordt het beveiligingslek veroorzaakt?**  
De Windows-shell verwerkt ongeldige URI's onvoldoende.

**Wat is een URI?**
Een URI (Uniform Resource Identifier) is een tekenreeks die wordt gebruikt om iets te doen met internet- of netwerkbronnen of om die te identificeren. Een URL is een typisch voorbeeld van een URI die betrekking heeft op een bron, zoals een website. Zie [RFC 2396](http://www.ietf.org/rfc/rfc2396.txt) voor meer informatie over URI's.

**Met welk doel kan een aanvaller het beveiligingslek misbruiken?**  
Een aanvaller die een gebruiker weet te overtuigen om een bijlage in een e-mail te openen of om een koppeling naar de website van een aanvaller te volgen, kan zich voordoen als de aangemelde gebruiker en zo willekeurige programmacode uitvoeren. Gebruikers met accounts waarvoor minder gebruikersrechten op het systeem zijn ingesteld, lopen minder risico dan gebruikers die met beheerdersrechten werken.

**Hoe kan een aanvaller dit beveiligingslek misbruiken?**  
Een aanvaller zou een speciaal vervaardigde URI moeten maken en de URI als invoer naar een toepassing op een kwetsbaar systeem moeten doorgeven dat vervolgens toegang tot de bron zou moeten proberen te krijgen waarnaar de URI verwijst. Toepassingen die URI's accepteren van niet-betrouwbare bronnen, zoals bijlagen in e-mails, documenten of gegevens van het netwerk, en ervan uitgaan dat dit veilig is, zijn kwetsbaar voor dit beveiligingslek. Onder specifieke omstandigheden kan de afhandeling van speciaal vervaardigde URI's ertoe kunnen leiden dat er willekeurige code wordt uitgevoerd. Om het beveiligingslek te misbruiken, zou een aanvaller de gebruiker ertoe moeten bewegen de bijlage of toepassing te openen of de website te bezoeken. Een aanvaller kan over een website beschikken die speciaal is ontworpen om via Internet Explorer misbruik te maken van dit lek, en vervolgens de gebruiker ertoe overhalen deze website te bekijken. Dit beveiligingslek zou ook kunnen worden misbruikt door gemanipuleerde websites of websites die door gebruikers gehoste content of advertenties accepteren. Deze websites kunnen speciaal vervaardigde URI's bevatten, die gebruik van dit beveiligingslek kunnen maken. Speciaal vervaardigde URI's kunnen ook worden weergegeven via banneradvertenties of andere manieren waarop webinhoud bij de kwetsbare systemen wordt afgeleverd.

**Voor welke systemen vormt dit beveiligingslek het grootste risico?**  
Systemen waarop ondersteunde edities van Windows XP en Windows Server 2003 met Internet Explorer 7 worden uitgevoerd, zijn kwetsbaar.

**Wat doet de update? **
De update verwijdert het beveiligingslek doordat de manier wordt veranderd waarop de Windows-shell ongeldige URL's verwerkt.

**Ik heb Windows Internet Explorer niet geïnstalleerd. Waarom ontvang ik deze update? **
Het beveiligingslek bevindt zich echter in een Windows-bestand, Shell32.dll, dat onderdeel uitmaakt van alle ondersteunde edities van Windows XP en Windows Server 2003. Microsoft heeft geen manier ontdekt waarop dit beveiligingslek kan worden misbruikt op systemen met Internet Explorer 6. Dit is de browser die onderdeel uitmaakt van deze besturingssystemen. Deze beveiligingsupdate is als een ingrijpende beveiligingsmaatregel beschikbaar gesteld voor alle klanten met ondersteunde edities van Windows XP en Windows Server 2003, ongeacht welke versie van Internet Explorer is geïnstalleerd.

**Ik gebruik Windows** **Vista**. **, vormt dit beveiligingslek een risico voor mij? **
Nee. Windows Vista heeft geen last van dit beveiligingslek. Windows Internet Explorer 7 is onderdeel van Windows Vista, maar de Windows-shell in Windows Vista heeft geen last van dit beveiligingslek.

**Was dit beveiligingslek openbaar gemaakt op het moment dat dit beveiligingsbulletin werd uitgegeven? **
Ja. Dit beveiligingslek is openbaar gemaakt toen dit Beveiligingsbulletin oorspronkelijk is uitgebracht. Aan dit beveiligingslek is nummer [CVE-2007-3896](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-3896) toegewezen. Dit beveiligingslek is voor het eerst beschreven in [Microsoft-beveiligingsadvies 943521](http://technet.microsoft.com/security/advisory/943521).

**Was aan Microsoft gerapporteerd dat er misbruik was gemaakt van dit beveiligingslek op het moment dat dit beveiligingsbulletin werd uitgegeven? **
Ja. Op het moment dat het beveiligingsbulletin werd uitgegeven, was er aan Microsoft gerapporteerd dat er misbruik was gemaakt van dit beveiligingslek

**Helpt de toepassing van deze beveiligingsupdate klanten om zich te beschermen tegen de openbaar gemaakte code die gericht is op misbruik van dit beveiligingslek? **
Ja. De beveiligingsupdate dicht het beveiligingslek waar momenteel misbruik van wordt gemaakt. Aan het gedichte beveiligingslek is nummer [CVE-2007-3896](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-3896) toegewezen.

**Controleren of de update is toegepast**

-   **Microsoft Baseline Security Analyzer**

    Gebruik het hulpprogramma Microsoft Baseline Security Analyzer (MBSA) om te controleren of de beveiligingsupdate is toegepast op het systeem waarvoor het probleem geldt. Zie de sectie Hulpmiddelen en richtlijnen voor detecteren en implementeren eerder in dit bulletin voor meer informatie.

-   **Verificatie van bestandsversie**

    Omdat er verschillende versies zijn van Microsoft Windows, kunnen de volgende stappen voor uw computer enigszins afwijken. Als dit het geval is, raadpleeg dan de productdocumentatie om deze stappen uit te voeren.

    1.  Klik op **Start** en klik vervolgens op **Zoeken**.
    2.  Klik in het venster **Zoekresultaten** op **Alle bestanden en mappen** onder **Zoekassistent**.
    3.  Typ in het vak **De volledige of gedeeltelijke bestandsnaam** een bestandsnaam uit de betreffende bestandsinformatietabel en klik vervolgens op **Zoeken**.
    4.  Klik in de lijst met bestanden met de rechtermuisknop op een bestandsnaam uit de betreffende bestandsinformatietabel en klik vervolgens op **Eigenschappen**.
        **Opmerking** Afhankelijk van de versie van het geïnstalleerde besturingssysteem of de geïnstalleerde programma's worden sommige bestanden in de bestandsinformatietabel mogelijk niet geïnstalleerd.
    5.  Bepaal op het tabblad **Versie** de versie van het bestand dat op uw computer is geïnstalleerd door de versie te vergelijken met de versie in de betreffende bestandsinformatietabel.
        **Opmerking** Andere kenmerken dan de bestandsversie kunnen tijdens de installatie worden veranderd. Vergelijking van andere bestandskenmerken met de gegevens in de bestandsinformatietabel vormt geen ondersteunde methode om te controleren of de update is toegepast. Bovendien kunnen in bepaalde gevallen de namen van bestanden tijdens installatie worden gewijzigd. Als de bestands- of versiegegevens niet aanwezig zijn, gebruikt u een van de andere beschikbare methoden om te controleren of de update is geïnstalleerd.

-   **Verificatie van registersleutel**

    Aan de hand van de registersleutels in de **Referentietabel** in deze sectie kunt u ook controleren welke bestanden met deze beveiligingsupdate zijn geïnstalleerd.

    Deze registersleutels bevatten mogelijk geen volledige lijst met geïnstalleerde bestanden. Bovendien zijn deze registersleutels mogelijk niet correct als een beheerder of een OEM de beveiligingsupdate integreert in de bronbestanden van de Windows-installatie.

**Controleren of de update is toegepast**

-   **Microsoft Baseline Security Analyzer**

    Gebruik het hulpprogramma Microsoft Baseline Security Analyzer (MBSA) om te controleren of de beveiligingsupdate is toegepast op het systeem waarvoor het probleem geldt. Zie de sectie Hulpmiddelen en richtlijnen voor detecteren en implementeren eerder in dit bulletin voor meer informatie.

-   **Verificatie van bestandsversie**

    Omdat er verschillende versies zijn van Microsoft Windows, kunnen de volgende stappen voor uw computer enigszins afwijken. Als dit het geval is, raadpleeg dan de productdocumentatie om deze stappen uit te voeren.

    1.  Klik op **Start** en klik vervolgens op **Zoeken**.
    2.  Klik in het venster **Zoekresultaten** op **Alle bestanden en mappen** onder **Zoekassistent**.
    3.  Typ in het vak **De volledige of gedeeltelijke bestandsnaam** een bestandsnaam uit de betreffende bestandsinformatietabel en klik vervolgens op **Zoeken**.
    4.  Klik in de lijst met bestanden met de rechtermuisknop op een bestandsnaam uit de betreffende bestandsinformatietabel en klik vervolgens op **Eigenschappen**.
        **Opmerking** Afhankelijk van de versie van het geïnstalleerde besturingssysteem of de geïnstalleerde programma's worden sommige bestanden in de bestandsinformatietabel mogelijk niet geïnstalleerd.
    5.  Bepaal op het tabblad **Versie** de versie van het bestand dat op uw computer is geïnstalleerd door de versie te vergelijken met de versie in de betreffende bestandsinformatietabel.
        **Opmerking** Andere kenmerken dan de bestandsversie kunnen tijdens de installatie worden veranderd. Vergelijking van andere bestandskenmerken met de gegevens in de bestandsinformatietabel vormt geen ondersteunde methode om te controleren of de update is toegepast. Bovendien kunnen in bepaalde gevallen de namen van bestanden tijdens installatie worden gewijzigd. Als de bestands- of versiegegevens niet aanwezig zijn, gebruikt u een van de andere beschikbare methoden om te controleren of de update is geïnstalleerd.

-   **Verificatie van registersleutel**

    Aan de hand van de registersleutels in de **Referentietabel** in deze sectie kunt u ook controleren welke bestanden met deze beveiligingsupdate zijn geïnstalleerd.

    Deze registersleutels bevatten mogelijk geen volledige lijst met geïnstalleerde bestanden. Bovendien zijn deze registersleutels mogelijk niet correct als een beheerder of een OEM de beveiligingsupdate integreert in de bronbestanden van de Windows-installatie.
