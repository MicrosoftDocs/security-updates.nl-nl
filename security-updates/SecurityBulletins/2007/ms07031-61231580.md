Security Bulletin

Microsoft-beveiligingsbulletin MS07-031 - Kritiek
=================================================

Door een beveiligingslek in het Schannel-beveiligingspakket van Windows kan externe code worden uitgevoerd (935840)
-------------------------------------------------------------------------------------------------------------------

Gepubliceerd: dinsdag 12 juni 2007

**Versie:** 1.0

### Algemene informatie

#### Samenvatting

Deze kritieke beveiligingsupdate lost een privé gemeld beveiligingslek in het beveiligingspakket voor veilige kanalen (Schannel) in Windows op. Het Schannel-beveiligingspakket voert de standaardverificatieprotocollen Secure Sockets Layer (SSL) en Transport Layer Security (TLS) voor internet uit. Door dit beveiligingslek kan externe code worden uitgevoerd als een gebruiker een speciaal ontworpen webpagina heeft weergegeven in een webbrowser of een toepassing heeft gebruikt die gebruikmaakt van SSL/TLS. Pogingen om misbruik te maken van dit beveiligingslek resulteren echter waarschijnlijk in het afsluiten van de webbrowser of de toepassing. Het systeem zou geen verbinding kunnen maken met websites of bronnen die SSL of TLS gebruiken totdat het systeem opnieuw is gestart.

Deze beveiligingsupdate is kritiek voor ondersteunde edities van Windows XP, belangrijk voor edities van Windows 2003 en matig voor edities van Windows 2000. Zie de subsectie **Software waarin dit probleem optreedt en software waarin dit probleem niet optreedt** in deze sectie voor meer informatie.

Met deze beveiligingsupdate dicht u het beveiligingslek door de wijze aan te passen waarop de client de door de server verzonden uitwisselingsgegevens voor de serversleutel parseert. Zie de subsectie Veelgestelde vragen over het specifieke beveiligingslek in de volgende sectie, **Informatie over het beveiligingslek**, voor meer informatie over het beveiligingslek.

**Aanbeveling.** Microsoft raadt klanten aan de update onmiddellijk toe te passen.

**Bekende problemen.** Geen

**Vervanging van andere updates.** Deze beveiligingsupdate vervangt geen eerder uitgebrachte beveiligingsupdates.

Beveiligingslek in het Schannel-beveiligingspakket van Windows - CVE-2007-2218  
------------------------------------------------------------------------------
  
<span></span>
Als gevolg van de wijze waarop Windows Schannel op clientcomputers door de server verzonden digitale handtekeningen valideert, bestaat er een beveiligingslek waardoor externe code kan worden uitgevoerd. Een aanvaller kan over een website beschikken die speciaal is ontworpen om via Internet Explorer misbruik van dit beveiligingslek te maken, en vervolgens de gebruiker ertoe overhalen deze website te bezoeken. Een aanvaller kan echter een gebruiker er niet toe dwingen om naar deze websites te gaan. De aanvaller moet een gebruiker er dus van overtuigen een bezoek te brengen aan de website. In de meeste gevallen doet de aanvaller dat door de gebruiker ertoe te bewegen op een koppeling in een e-mailbericht of een verzoek in een expresbericht te klikken waarmee de gebruiker naar de website van de aanvaller gaat.
  
Zie [CVE-2007-2218](http://cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-2218) als u dit beveiligingslek wilt weergeven als standaardvermelding in de lijst met veelvoorkomende beveiligingslekken en blootstellingen.
  
#### Beperkende factoren voor het beveiligingslek in het Schannel-beveiligingspakket van Windows - CVE-2007-2218
  
Met beperkende factoren wordt verwezen naar een instelling, de algehele configuratie of algemene tips voor een standaardstatus waarmee de ernst van misbruik via een beveiligingslek kan worden verminderd. De volgende beperkende factoren kunnen in uw situatie van pas komen:
  
-   In het geval van een aanval vanaf het web kan de aanvaller beschikken over een website waarop zich een webpagina bevindt waarmee dit beveiligingslek wordt misbruikt. Daarnaast kunnen ook websites waarop gegevens of advertenties staan of gegevens of advertenties kunnen worden ingevoerd, schadelijke inhoud bevatten waarmee dit beveiligingslek kan worden misbruikt. Een aanvaller kan echter een gebruiker er niet toe dwingen om naar deze websites te gaan. De aanvaller moet een gebruiker er dus van overtuigen een bezoek te brengen aan de website. In de meeste gevallen doet de aanvaller dat door de gebruiker ertoe te bewegen op een koppeling in een e-mailbericht of een verzoek in een expresbericht te klikken waarmee de gebruiker naar de website van de aanvaller gaat.  
-   Pogingen om dit beveiligingslek te misbruiken zullen waarschijnlijk niet leiden tot externe uitvoering van code maar tot een denial of service, veroorzaakt door een onverwachte herstart van het desbetreffende systeem. De denial of service blijft in Windows 2000 beperkt tot bestaande en nieuwe SSL- en TLS-verbindingen. Als dat niet zo was, zou het systeem nog beschikbaar zijn. Als een aanval op Windows Server 2003 slaagt, wordt het systeem opnieuw opgestart.
  
#### Tijdelijke oplossingen voor het beveiligingslek in het Schannel-beveiligingspakket van Windows - CVE-2007-2218
  
Microsoft heeft nog geen oplossingen voor dit beveiligingslek.
  
#### Veelgestelde vragen over het beveiligingslek in het Schannel-beveiligingspakket van Windows - CVE-2007-2218
  
**Wat is de omvang van het beveiligingslek?**  
In Windows XP heeft dit beveiligingslek betrekking op de uitvoering van externe code. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan op afstand volledige controle krijgen over een kwetsbaar systeem. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige gebruikersrechten maken. Pogingen om dit beveiligingslek te misbruiken zullen waarschijnlijk eerder leiden tot een denial of service dan tot uitvoering van externe code.
  
In Windows 2000 kan dit beveiligingslek ervoor zorgen dat het systeem waarin dit probleem optreedt geen SSL- en TLS-verbindingen meer accepteert. Het systeem zou geen verbinding kunnen maken met websites of bronnen die SSL of TLS gebruiken totdat het systeem opnieuw is gestart.
  
In Windows 2003 kan dit beveiligingslek ertoe leiden dat het systeem waarvoor dit probleem geldt automatisch opnieuw wordt gestart.
  
Een aanvaller kan via dit beveiligingslek geen code uitvoeren of zichzelf rechtstreeks gebruikersrechten in Windows 2000 en Windows 2004 toekennen.
  
**Waardoor wordt het beveiligingslek veroorzaakt?**  
Schannel voert tijdens de SSL-handshake onvoldoende controles uit voor speciaal ontworpen, door de server verzonden digitale handtekeningen..
  
**Wat is Schannel?**  
Het beveiligingspakket Secure Channel (Schannel) is een Security Support Provider (SSP) waarmee de standaardverificatieprotocollen Secure Sockets Layer (SSL) en Transport Layer Security (TLS) worden uitgevoerd. Schannel maakt deel uit van het beveiligingspakket met daarin een verificatieservice voor veilige communicatie tussen clients en servers. Ga naar de volgende [MSDN-website](http://msdn2.microsoft.com/en-us/library/ms678421.aspx) voor meer informatie over Schannel.
  
**Met welk doel kan een aanvaller het beveiligingslek misbruiken?**  
Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan volledige controle krijgen over het systeem op edities van Windows XP waarop dit probleem voorkomt. Pogingen om misbruik te maken van dit beveiligingslek resulteren echter waarschijnlijk in het afsluiten van de webbrowser of de toepassing die gebruikmaakt van SSL/TLS. Het systeem zou geen verbinding kunnen maken met websites of bronnen die SSL of TLS gebruiken totdat het systeem opnieuw is gestart. In edities van Windows 2000 en Windows 2003 zal misbruik van dit beveiligingslek waarschijnlijk leiden tot een denial of service.
  
**Hoe kan een aanvaller dit beveiligingslek misbruiken?**  
Een aanvaller kan over een website beschikken die speciaal is ontworpen om via een webbrowser misbruik te maken van dit lek, en vervolgens de gebruiker ertoe overhalen deze website te bekijken. Het verwerken van deze speciaal ontworpen, door de server verzonden digitale handtekening zorgt ervoor dat toepassingen die deze API gebruiken, worden afgesloten. Bovendien wordt het SSL-protocol uitgeschakeld totdat het systeem opnieuw wordt opgestart. Een aanvaller kan echter een gebruiker er niet toe dwingen om naar deze websites te gaan. De aanvaller moet een gebruiker er dus van overtuigen een bezoek te brengen aan de website. In de meeste gevallen doet de aanvaller dat door de gebruiker ertoe te bewegen op een koppeling in een e-mailbericht of een verzoek in een expresbericht te klikken waarmee de gebruiker naar de website van de aanvaller gaat.
  
**Voor welke systemen vormt dit beveiligingslek het grootste risico?**  
Voor dit beveiligingslek is het noodzakelijk dat een gebruiker websites bezoekt en daarop is aangemeld voordat er een schadelijke actie kan plaatsvinden. Daarom lopen systemen waarin een webbrowser of toepassing regelmatig gebruikmaakt van SSL of TLS, zoals werkstations en terminalservers, het grootste risico.
  
**Wat doet de update?**  
Met de update dicht u het beveiligingslek door de wijze aan te passen waarop de client de door de server verzonden uitwisselingsgegevens voor de serversleutel parseert.
  
**Was dit beveiligingslek openbaar gemaakt op het moment dat dit beveiligingsbulletin werd uitgegeven?**  
Nee. Microsoft heeft uit discrete bronnen informatie over dit beveiligingslek ontvangen.
  
**Was aan Microsoft gerapporteerd dat er misbruik was gemaakt van dit beveiligingslek op het moment dat dit beveiligingsbulletin werd uitgegeven?**  
Nee. Microsoft beschikte niet over informatie dat dit beveiligingslek was gebruikt voor aanvallen op klanten en had geen voorbeelden van gepubliceerde conceptcode gezien als bewijs toen dat dit beveiligingsbulletin oorspronkelijk werd uitgegeven.

-   **Microsoft Baseline Security Analyzer**

    Gebruik het hulpprogramma Microsoft Baseline Security Analyzer (MBSA) om te controleren of de beveiligingsupdate is toegepast op het systeem waarvoor het probleem geldt. Zie de sectie Hulpmiddelen en richtlijnen voor detecteren en implementeren eerder in dit bulletin voor meer informatie.

-   **Verificatie van bestandsversie**

    Omdat er verschillende versies en edities zijn van Microsoft Windows, kunnen de volgende stappen voor uw systeem enigszins afwijken. Als dit het geval is, raadpleeg dan de productdocumentatie om deze stappen uit te voeren.

    1.  Klik op **Start** en klik vervolgens op **Zoeken**.
    2.  Klik in het venster **Zoekresultaten** op **Alle bestanden en mappen** onder **Zoekassistent**.
    3.  Typ in het vak **De volledige of gedeeltelijke bestandsnaam** een bestandsnaam uit de betreffende bestandsinformatietabel en klik vervolgens op **Zoeken**.
    4.  Klik in de lijst met bestanden met de rechtermuisknop op een bestandsnaam uit de betreffende bestandsinformatietabel en klik vervolgens op **Eigenschappen**.
        **Opmerking** Afhankelijk van de versie van het geïnstalleerde besturingssysteem of de geïnstalleerde programma's worden sommige bestanden in de bestandsinformatietabel mogelijk niet geïnstalleerd.
    5.  Bepaal op het tabblad **Versie** de versie van het bestand dat op uw systeem is geïnstalleerd door de versie te vergelijken met de versie in de betreffende bestandsinformatietabel.
        **Opmerking** Andere kenmerken dan de bestandsversie kunnen tijdens de installatie worden veranderd. Vergelijking van andere bestandskenmerken met de gegevens in de bestandsinformatietabel vormt geen ondersteunde methode om te controleren of de update is toegepast. Bovendien kunnen in bepaalde gevallen de namen van bestanden tijdens installatie worden gewijzigd. Als de bestands- of versiegegevens niet aanwezig zijn, gebruikt u een van de andere beschikbare methoden om te controleren of de update is geïnstalleerd.

    Aan de hand van de registersleutels in de **Referentietabel** in deze sectie kunt u ook controleren welke bestanden met deze beveiligingsupdate zijn geïnstalleerd.

    Deze registersleutels bevatten mogelijk geen volledige lijst met geïnstalleerde bestanden. Bovendien zijn deze registersleutels mogelijk niet correct als een beheerder of een OEM de beveiligingsupdate integreert in de bronbestanden van de Windows-installatie.

    Gebruik het hulpprogramma Microsoft Baseline Security Analyzer (MBSA) om te controleren of de beveiligingsupdate is toegepast op het systeem waarvoor het probleem geldt. Zie de sectie Hulpmiddelen en richtlijnen voor detecteren en implementeren eerder in dit bulletin voor meer informatie.

-   **Verificatie van bestandsversie**

    Omdat er verschillende versies en edities zijn van Microsoft Windows, kunnen de volgende stappen voor uw systeem enigszins afwijken. Als dit het geval is, raadpleeg dan de productdocumentatie om deze stappen uit te voeren.

    1.  Klik op **Start** en klik vervolgens op **Zoeken**.
    2.  Klik in het venster **Zoekresultaten** op **Alle bestanden en mappen** onder **Zoekassistent**.
    3.  Typ in het vak **De volledige of gedeeltelijke bestandsnaam** een bestandsnaam uit de betreffende bestandsinformatietabel en klik vervolgens op **Zoeken**.

        **Opmerking** Afhankelijk van de versie van het geïnstalleerde besturingssysteem of de geïnstalleerde programma's worden sommige bestanden in de bestandsinformatietabel mogelijk niet geïnstalleerd.
    5.  Bepaal op het tabblad **Versie** de versie van het bestand dat op uw systeem is geïnstalleerd door de versie te vergelijken met de versie in de betreffende bestandsinformatietabel.
        **Opmerking** Andere kenmerken dan de bestandsversie kunnen tijdens de installatie worden veranderd. Vergelijking van andere bestandskenmerken met de gegevens in de bestandsinformatietabel vormt geen ondersteunde methode om te controleren of de update is toegepast. Bovendien kunnen in bepaalde gevallen de namen van bestanden tijdens installatie worden gewijzigd. Als de bestands- of versiegegevens niet aanwezig zijn, gebruikt u een van de andere beschikbare methoden om te controleren of de update is geïnstalleerd.

**Controleren of de update is toegepast**

-   **Microsoft Baseline Security Analyzer**

    Gebruik het hulpprogramma Microsoft Baseline Security Analyzer (MBSA) om te controleren of de beveiligingsupdate is toegepast op het systeem waarvoor het probleem geldt. Zie de sectie Hulpmiddelen en richtlijnen voor detecteren en implementeren eerder in dit bulletin voor meer informatie.

-   **Verificatie van bestandsversie**

    Omdat er verschillende versies en edities zijn van Microsoft Windows, kunnen de volgende stappen voor uw systeem enigszins afwijken. Als dit het geval is, raadpleeg dan de productdocumentatie om deze stappen uit te voeren.

    1.  Klik op **Start** en klik vervolgens op **Zoeken**.
    2.  Klik in het venster **Zoekresultaten** op **Alle bestanden en mappen** onder **Zoekassistent**.
    3.  Typ in het vak **De volledige of gedeeltelijke bestandsnaam** een bestandsnaam uit de betreffende bestandsinformatietabel en klik vervolgens op **Zoeken**.
    4.  Klik in de lijst met bestanden met de rechtermuisknop op een bestandsnaam uit de betreffende bestandsinformatietabel en klik vervolgens op **Eigenschappen**.
        **Opmerking** Afhankelijk van de versie van het geïnstalleerde besturingssysteem of de geïnstalleerde programma's worden sommige bestanden in de bestandsinformatietabel mogelijk niet geïnstalleerd.
    5.  Bepaal op het tabblad **Versie** de versie van het bestand dat op uw systeem is geïnstalleerd door de versie te vergelijken met de versie in de betreffende bestandsinformatietabel.
        **Opmerking** Andere kenmerken dan de bestandsversie kunnen tijdens de installatie worden veranderd. Vergelijking van andere bestandskenmerken met de gegevens in de bestandsinformatietabel vormt geen ondersteunde methode om te controleren of de update is toegepast. Bovendien kunnen in bepaalde gevallen de namen van bestanden tijdens installatie worden gewijzigd. Als de bestands- of versiegegevens niet aanwezig zijn, gebruikt u een van de andere beschikbare methoden om te controleren of de update is geïnstalleerd.

-   **Verificatie van registersleutel**

    Aan de hand van de registersleutels in de **Referentietabel** in deze sectie kunt u ook controleren welke bestanden met deze beveiligingsupdate zijn geïnstalleerd.

    Deze registersleutels bevatten mogelijk geen volledige lijst met geïnstalleerde bestanden. Bovendien zijn deze registersleutels mogelijk niet correct als een beheerder of een OEM de beveiligingsupdate integreert in de bronbestanden van de Windows-installatie.
