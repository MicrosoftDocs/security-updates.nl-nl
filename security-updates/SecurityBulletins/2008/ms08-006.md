Security Bulletin

Microsoft-beveiligingsbulletin MS08-006 - Belangrijk
====================================================

Door een beveiligingslek in Microsoft Internet Information Services kan externe code worden uitgevoerd (942830)
---------------------------------------------------------------------------------------------------------------

Gepubliceerd: dinsdag 12 februari 2008 | Bijgewerkt: woensdag 20 februari 2008

**Versie:** 1.1

### Algemene informatie

#### Samenvatting

Deze belangrijke update lost een privé gemeld beveiligingslek in Internet Information Services (IIS) op. Er is een beveiligingslek met betrekking tot het uitvoeren van externe code aanwezig in de manier waarop de Internet Information Service invoer verwerkt op ASP-webpagina's. Een aanvaller die dit beveiligingslek weet te misbruiken, zou vervolgens handelingen op de IIS-server kunnen verrichten met dezelfde rechten als de werkprocesidentiteit (WPI). De WPI wordt standaard met netwerkservice-accountbevoegdheden geconfigureerd. IIS-servers met ASP-pagina's waarvan de toepassingsgroepen zijn geconfigureerd met een WPI die gebruikmaakt van een account met beheerdersbevoegdheden lopen mogelijk een groter gevaar dan IIS-servers waarvan de toepassingsgroep is geconfigureerd met de standaard WPI-instellingen.

De beveiligingsupdate heeft het prioriteitsniveau Belangrijk voor Microsoft Internet Information Services op alle ondersteunde edities van Windows XP en Windows Server 2003. Zie voor meer informatie de subsectie **Software waarin dit probleem optreedt en software waarin dit probleem niet optreedt**, verderop in deze sectie.

Zie de subsectie Veelgestelde vragen in de volgende sectie, **Informatie over het beveiligingslek**, voor meer informatie over het beveiligingslek.

**Aanbeveling.** Microsoft raadt klanten aan de update zo spoedig mogelijk te installeren.

**Bekende problemen.** Geen

Beveiligingslek met betrekking tot ASP - CVE-2008-0075  
------------------------------------------------------
  
<span></span>
Er is een beveiligingslek met betrekking tot het uitvoeren van externe code aanwezig in de manier waarop de Internet Information Service invoer verwerkt op ASP-webpagina's. Een aanvaller zou het beveiligingslek kunnen misbruiken door schadelijke invoer door te geven aan een ASP-webpagina. Een aanvaller die dit beveiligingslek weet te misbruiken kan handelingen op de IIS-server verrichten met dezelfde rechten als de werkprocesidentiteit (WPI), die standaard met netwerkservice-accountbevoegdheden wordt geconfigureerd.
  
Zie [CVE-2008-0075](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-0075) als u dit beveiligingslek wilt weergeven als standaardvermelding in de lijst met veelvoorkomende beveiligingslekken en blootstellingen.
  
#### Beperkende factoren voor het beveiligingslek met betrekking tot ASP - CVE-2008-0075
  
Met beperkende factoren wordt verwezen naar een instelling, de algehele configuratie of algemene tips voor een standaardstatus waarmee de ernst van misbruik via een beveiligingslek kan worden verminderd. De volgende beperkende factoren kunnen in uw situatie van pas komen:
  
-   IIS 5.1 is geen onderdeel van de standaardinstallatie van Windows XP Professional Service Pack 2.  
-   Op ondersteunde edities van Windows Server 2003 is IIS niet standaard geïnstalleerd of ingeschakeld.  
-   Als IIS op ondersteunde edities van Windows Server 2003 is ingeschakeld, is klassieke ASP niet standaard geïnstalleerd of ingeschakeld.  
-   Als IIS op ondersteunde edities van Windows Server 2003 is ingeschakeld en klassieke ASP wordt gebruikt, kan een aanvaller die misbruik weet te maken van dit beveiligingslek alleen standaard netwerkservice-accountbevoegdheden krijgen. Standaard hebben gebruikers met een netwerkservice-account dezelfde gebruikersrechten als geverifieerde gebruikers.  
-   Dit beveiligingslek is niet van toepassing op ASP.NET. Klanten die alleen ASP.NET en niet ASP hebbben geïnstalleerd, lopen geen risico door dit beveiligingslek.
  
#### Tijdelijke oplossingen voor het beveiligingslek met betrekking tot ASP - CVE-2008-0075
  
Tijdelijke oplossingen zijn wijzigingen in een instelling of configuratie waardoor het beveiligingslek zelf niet wordt gedicht, maar waarmee aanvalsvectoren wel worden geblokkeerd. Microsoft heeft de volgende methoden en statussen om het probleem te omzeilen getest om uit te zoeken of een tijdelijke oplossing functionaliteit vermindert:
  
-   **Schakel in Windows Server 2003 klassieke ASP uit**
  
    Volg deze stappen om klassieke ASP uit te schakelen:
  
    1. Klik achtereenvolgens op **Start**, **Systeembeheer** en **Internet Information Services (IIS) Manager**.
  
    2. Klik op het **plusteken** (+) naast de naam van de server en klik vervolgens op de map **Web Service Extensions**.
  
    3. Klik om **Active Server Pages** in het rechterdeelvenster te markeren en klik vervolgens op **Verbieden.**
  
    **Effecten van de tijdelijke oplossing.** Gebruikers zullen geen klassieke ASP-pagina's kunnen gebruiken.
  
    **De tijdelijke oplossing ongedaan maken.**Voer de stappen 1 tot en met 3 uit zoals die hierboven staan beschreven als u de tijdelijke oplossing ongedaan wilt maken. Klik vervolgens bij het markeren van **Active Server-pagina's** op **Toestaan**.
  
#### Veelgestelde vragen over het beveiligingslek met betrekking tot ASP - CVE-2008-0075
  
**Wat is de omvang van het beveiligingslek?**  
Dit beveiligingslek heeft betrekking op de uitvoering van externe code. Een aanvaller die dit beveiligingslek weet te misbruiken, kan handelingen op de IIS-server verrichten met dezelfde rechten als de werkprocesidentiteit (WPI), die standaard met netwerkservice-accountbevoegdheden wordt geconfigureerd.
  
**Waardoor wordt het beveiligingslek veroorzaakt?**  
Het beveiligingslek wordt veroorzaakt door de manier waarop Internet Information Services invoer op ASP-pagina's verkeerd verwerkt.
  
**Met welk doel kan een aanvaller het beveiligingslek misbruiken?**  
Een aanvaller die dit beveiligingslek weet te misbruiken, kan vervolgens handelingen op de IIS-server verrichten met dezelfde rechten als de werkprocesidentiteit (WPI), die standaard met netwerkservice-accountbevoegdheden wordt geconfigureerd. Services geconfigureerd met netwerkservice-accountbevoegdheden verkrijgen geverifieerde toegang op gebruikersniveau, niet op beheerniveau. Zie [netwerkservice-account](http://msdn2.microsoft.com/en-us/library/ms684272(vs.85).aspx) voor meer informatie. IIS-servers waarvan de toepassingsgroep is geconfigureerd met een WPI die gebruikmaakt van een account met beheerdersbevoegdheden, lopen mogelijk een groter gevaar dan IIS-servers waarvan de toepassingsgroep is geconfigureerd met de standaard WPI-instellingen.
  
De netwerkservice-account gebruikt de referenties van de computer bij verificatie op afstand. maar heeft veel lagere bevoegdheden op de server zelf en beschikt derhalve niet over lokale beheerdersbevoegdheden. Zie voor meer informatie [netwerkservice-account](http://msdn2.microsoft.com/en-us/library/ms684272(vs.85).aspx), [IIS en geïntegreerde accounts (IIS 6.0)](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/iis/3648346f-e4f5-474b-86c7-5a86e85fa1ff.mspx), [Werkprocesidentiteiten configureren (IIS 6.0)](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/iis/76be66e2-1db1-432e-95be-14ff59c2ec75.mspx) en [Toepassingsgroepen configureren in IIS 6.0 (IIS 6.0)](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/iis/2a231dcb-d786-4b6d-b3ba-bda04061a5e7.mspx).
  
**Hoe kan een aanvaller dit beveiligingslek misbruiken?**  
Een aanvaller zou speciaal vervaardigde invoer naar een ASP-pagina kunnen doorgeven. De ASP-pagina zou reeds kunnen bestaan of zou door een gebruiker kunnen worden geüpload in een gehoste omgeving.
  
**Wat zijn werkprocessen voor IIS?**  
Werkprocessen zijn processen die op de server taken voor een webserver uitvoeren**.** Voor informatie over werkprocessen en hoe deze in IIS 6.0 zijn geïmplementeerd, zie [Werkprocessen (IIS 6.0).](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/iis/29f53968-0115-451f-b26d-5ad58d87b5d1.mspx) De aanvullende informatie over werkprocessen en hoe deze webtoepassingen in isolatie uitvoeren, zie [Werkproces - Isolatiemodus (IIS 6.0).](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/iis/333f3410-b566-49bc-8399-57bcc404db8c.mspx)
  
**Wat is ASP?**  
Microsoft Active Server Pages (ASP) is een scripttechnologie op de server voor het maken van dynamische en interactieve webtoepassingen. Een ASP-pagina is een HTML-pagina met scripts op de server die door de webserver worden verwerkt, voordat deze naar de browser van de gebruiker worden verzonden.
  
**Voor welke systemen vormt dit beveiligingslek het grootste risico?**  
Webservers met Windows Server 2003 en Windows XP Professional lopen een risico als IIS en ASP zijn ingeschakeld. Webhostingproviders die hun klanten toestaan om ASP-pagina's naar hun gehoste websites te uploaden, lopen een verhoogd risico.
  
**Wat doet de update?**  
De update verwijdert het beveiligingslek door de wijze te veranderen waarop Internet Information Services invoer op ASP-pagina's verwerkt.
  
**Was dit beveiligingslek openbaar gemaakt op het moment dat dit beveiligingsbulletin werd uitgegeven?**  
Nee. Microsoft heeft uit discrete bronnen informatie over dit beveiligingslek ontvangen. Microsoft heeft geen informatie ontvangen die erop wees dat dit beveiligingslek openbaar was gemaakt op het moment dat dit beveiligingsbulletin oorspronkelijk werd uitgegeven.
  
**Was aan Microsoft gerapporteerd dat er misbruik was gemaakt van dit beveiligingslek op het moment dat dit beveiligingsbulletin werd uitgegeven?**  
Nee. Microsoft beschikte niet over informatie dat dit beveiligingslek was gebruikt voor aanvallen op klanten en had geen voorbeelden van gepubliceerde conceptcode gezien als bewijs toen dat dit beveiligingsbulletin oorspronkelijk werd uitgegeven.

**Controleren of de update is toegepast**

-   **Microsoft Baseline Security Analyzer**

    Gebruik het hulpprogramma Microsoft Baseline Security Analyzer (MBSA) om te controleren of de beveiligingsupdate is toegepast op het systeem waarvoor het probleem geldt. Zie de sectie Hulpmiddelen en richtlijnen voor detecteren en implementeren eerder in dit bulletin voor meer informatie.

-   **Verificatie van bestandsversie**

    Omdat er verschillende versies zijn van Microsoft Windows, kunnen de volgende stappen voor uw computer enigszins afwijken. Als dit het geval is, raadpleeg dan de productdocumentatie om deze stappen uit te voeren.

    1.  Klik op **Start** en klik vervolgens op **Zoeken**.
    2.  Klik in het venster **Zoekresultaten** op **Alle bestanden en mappen** onder **Zoekassistent**.
    3.  Typ in het vak **De volledige of gedeeltelijke bestandsnaam** een bestandsnaam uit de betreffende bestandsinformatietabel en klik vervolgens op **Zoeken**.
    4.  Klik in de lijst met bestanden met de rechtermuisknop op een bestandsnaam uit de betreffende bestandsinformatietabel en klik vervolgens op **Eigenschappen**.
        **Opmerking** Afhankelijk van de versie van het geïnstalleerde besturingssysteem of de geïnstalleerde programma's worden sommige bestanden in de bestandsinformatietabel mogelijk niet geïnstalleerd.
    5.  Bepaal op het tabblad **Versie** de versie van het bestand dat op uw computer is geïnstalleerd door de versie te vergelijken met de versie in de betreffende bestandsinformatietabel.
        **Opmerking** Andere kenmerken dan de bestandsversie kunnen tijdens de installatie worden veranderd. Vergelijking van andere bestandskenmerken met de gegevens in de bestandsinformatietabel vormt geen ondersteunde methode om te controleren of de update is toegepast. Bovendien kunnen in bepaalde gevallen de namen van bestanden tijdens installatie worden gewijzigd. Als de bestands- of versiegegevens niet aanwezig zijn, gebruikt u een van de andere beschikbare methoden om te controleren of de update is geïnstalleerd.

-   **Verificatie van registersleutel**

    Aan de hand van de registersleutels in de **Referentietabel** in deze sectie kunt u ook controleren welke bestanden met deze beveiligingsupdate zijn geïnstalleerd.

    Deze registersleutels bevatten mogelijk geen volledige lijst met geïnstalleerde bestanden. Bovendien zijn deze registersleutels mogelijk niet correct als een beheerder of een OEM de beveiligingsupdate integreert in de bronbestanden van de Windows-installatie.

#### Informatie over implementatie

**De update installeren**

Wanneer u deze beveiligingspatch installeert, wordt door het installatieprogramma gecontroleerd of een of meerdere bestanden die op de computer worden bijgewerkt, al eerder door een Microsoft-hotfix zijn bijgewerkt.

**Controleren of de update is toegepast**

-   **Microsoft Baseline Security Analyzer**

    Gebruik het hulpprogramma Microsoft Baseline Security Analyzer (MBSA) om te controleren of de beveiligingsupdate is toegepast op het systeem waarvoor het probleem geldt. Zie de sectie Hulpmiddelen en richtlijnen voor detecteren en implementeren eerder in dit bulletin voor meer informatie.

-   **Verificatie van bestandsversie**

    Omdat er verschillende versies zijn van Microsoft Windows, kunnen de volgende stappen voor uw computer enigszins afwijken. Als dit het geval is, raadpleeg dan de productdocumentatie om deze stappen uit te voeren.

    1.  Klik op **Start** en klik vervolgens op **Zoeken**.
    2.  Klik in het venster **Zoekresultaten** op **Alle bestanden en mappen** onder **Zoekassistent**.
    3.  Typ in het vak **De volledige of gedeeltelijke bestandsnaam** een bestandsnaam uit de betreffende bestandsinformatietabel en klik vervolgens op **Zoeken**.
    4.  Klik in de lijst met bestanden met de rechtermuisknop op een bestandsnaam uit de betreffende bestandsinformatietabel en klik vervolgens op **Eigenschappen**.
        **Opmerking** Bepaalde bestanden die in de bestandsinformatietabel staan vermeld, worden mogelijk niet geïnstalleerd. Dit is afhankelijk van de editie van het besturingssysteem en de programma's die op uw systeem zijn geïnstalleerd.
    5.  Bepaal op het tabblad **Versie** de versie van het bestand dat op uw systeem is geïnstalleerd door de versie te vergelijken met de versie in de betreffende bestandsinformatietabel.
        **Opmerking** Andere kenmerken dan de bestandsversie kunnen tijdens de installatie worden veranderd. Vergelijking van andere bestandskenmerken met de gegevens in de bestandsinformatietabel vormt geen ondersteunde methode om te controleren of de update is toegepast. Bovendien kunnen in bepaalde gevallen de namen van bestanden tijdens installatie worden gewijzigd. Als de bestands- of versiegegevens niet aanwezig zijn, gebruikt u een van de andere beschikbare methoden om te controleren of de update is geïnstalleerd.

-   **Verificatie van registersleutel**

    Aan de hand van de registersleutels in de **Referentietabel** in deze sectie kunt u ook controleren welke bestanden met deze beveiligingsupdate zijn geïnstalleerd.

    Deze registersleutels bevatten mogelijk geen volledige lijst met geïnstalleerde bestanden. Bovendien zijn deze registersleutels mogelijk niet correct als een beheerder of een OEM de beveiligingsupdate integreert in de bronbestanden van de Windows-installatie.
