Security Bulletin

Microsoft-beveiligingsbulletin MS08-049 - Belangrijk
====================================================

Door beveiligingslekken in het gebeurtenissysteem kan externe code worden uitgevoerd (950974)
---------------------------------------------------------------------------------------------

Gepubliceerd: dinsdag 12 augustus 2008 | Bijgewerkt: woensdag 10 september 2008

**Versie:** 1.1

### Algemene informatie

#### Samenvatting

Deze update lost twee privé gemelde beveiligingslekken in Microsoft Windows Event System op die uitvoering van externe code kunnen toestaan. Een aanvaller die misbruik weet te maken van het beveiligingslek, kan volledige controle krijgen over een systeem waarvoor dit probleem geldt. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige beheerdersrechten maken.

Het prioriteitsniveau van dit bulletin is Kritiek voor alle ondersteunde versies van Microsoft Windows 2000, Windows XP, Windows Server 2003, Windows Vista en Windows Server 2008. Zie de subsectie **Software waarin dit probleem optreedt en software waarin dit probleem niet optreedt** in deze sectie voor meer informatie.

De beveiligingsupdate lost het beveiligingslek op door de manier te wijzigen waarop abonnementen per gebruiker door het gebeurtenissysteem worden verwerkt. Zie de subsectie Veelgestelde vragen over het specifieke beveiligingslek in de volgende sectie, **Informatie over het beveiligingslek**, voor meer informatie over deze beveiligingslekken.

**Aanbeveling.**  Microsoft raadt klanten aan de update zo spoedig mogelijk te installeren.

**Bekende problemen.** In [Microsoft Knowledge Base-artikel 950974](http://support.microsoft.com/kb/950974) worden bekende problemen beschreven die klanten mogelijk ondervinden bij de installatie van deze beveiligingsupdate. In het artikel worden ook de aanbevolen oplossingen voor deze problemen toegelicht. Zie de subsectie **Veelgestelde vragen met betrekking tot deze beveiligingsupdate** in deze sectie voor meer informatie over het installeren van deze updates.

Beveiligingslek Gebeurtenissysteem - CVE-2008-1457  
--------------------------------------------------
  
<span></span>
Een beveiligingslek met betrekking tot het uitvoeren van externe code bestaat omdat het Microsoft Windows Gebeurtenissysteem aanvragen voor gebruikersabonnementen onjuist valideert wanneer deze worden gemaakt. Door het beveiligingslek kan externe code worden uitgevoerd. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan volledige controle krijgen over een systeem waarin dit probleem optreedt. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige beheerdersrechten maken.
  
Zie [CVE 2008-1457](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1457) als u dit beveiligingslek wilt weergeven als standaardvermelding in de lijst met veelvoorkomende beveiligingslekken en blootstellingen.
  
#### Beperkende Factoren voor beveiligingslek Gebeurtenissysteem - CVE-2008-1457
  
Met beperkende factoren wordt verwezen naar een instelling, de algehele configuratie of algemene tips voor een standaardstatus waarmee de ernst van misbruik via een beveiligingslek kan worden verminderd. De volgende beperkende factoren kunnen in uw situatie van pas komen:
  
-   Een aanvaller moet zijn geverifieerd om misbruik van dit beveiligingslek te kunnen maken.
  
#### Tijdelijke oplossingen voor beveiligingslek Gebeurtenissysteem - CVE-2008-1457
  
Tijdelijke oplossingen zijn wijzigingen in een instelling of configuratie waardoor het beveiligingslek zelf niet wordt gedicht, maar waarmee aanvalsvectoren wel worden geblokkeerd voordat u de update uitvoert. Microsoft heeft de volgende methoden en statussen om het probleem te omzeilen getest om uit te zoeken of een tijdelijke oplossing functionaliteit vermindert:
  
-   **Maak de registratie ongedaan van es.dll**
  
    Voer de volgende stappen uit:
  
    1.  Meld u aan als gebruiker met beheerdersbevoegdheden.  
    2.  Klik achtereenvolgens op **Start** en **Uitvoeren**, typ **cmd** en klik op **OK**.  
    3.  Typ de volgende opdracht op de opdrachtprompt:  
        regsvr32 /-u %WINDIR%\\SYSTEM32\\es.dll
  
    **Gevolgen van de tijdelijke oplossing:** Het Gebeurtenissysteem wordt niet uitgevoerd.
  
    **Zo kunt u tijdelijke oplossingen ongedaan maken.** Registreer es.dll opnieuw. Typ de volgende opdracht op de opdrachtregel:
  
    regsvr 32 %windir%\\system32\\es.dll
  
-   **Stop de gebeurtenissysteemservices System Event Notification en COM+ en schakel die uit**
  
    Het uitschakelen van deze services helpt de kwetsbare systemen te beschermen tegen pogingen om dit beveiligingslek te misbruiken.
  
    -   Interactief
  
        Voer de volgende stappen uit:
  
        1.  Klik op **Start** en klik op **Configuratiescherm**. (Of klik op Start, wijs **Instellingen** aan en klik op **Configuratiescherm**.)  
        2.  Dubbelklik op **Systeembeheer**. Of klik op **Klassieke weergave** en dubbelklik op **Systeembeheer.**  
        3.  Dubbelklik op **Services**.  
        4.  Dubbelklik op **System Event Notification**.  
        5.  Klik in de lijst **Opstarttype** op **Uitgeschakeld**.  
        6.  Klik op **Stoppen** en op **OK**.
  
        U kunt de service **System Event Notification** ook uitschakelen door de volgende opdracht in de opdrachtregel op te geven (beschikbaar in Windows XP en in de Microsoft Windows 2000 Resource Kit):
  
        sc stop sens & sc config sens start= disabled
  
        Herhaal vervolgens de bovenstaande stappen voor **COM + Event System.**
  
        U kunt de service **COM + Event System** ook uitschakelen door de volgende opdracht in de opdrachtregel op te geven (beschikbaar in Windows XP en in de Microsoft Windows 2000 Resource Kit):
  
        sc stop eventsystem & sc config eventsystem start= disabled
  
    -   Via Groepsbeleid:
  
        Schakel deze services uit via de instellingen van Groepsbeleid. U kunt het opstarten van deze service op lokaal niveau, siteniveau of organisatieniveau uitschakelen door in Microsoft Windows 2000-domeinen en Windows Server 2003-domeinen het groepsbeleidsobject te gebruiken.
  
        **Opmerking:&lt;/B2&gt;** Raadpleeg desgewenst de Windows Server 2003 Security Guide. In deze handleiding vindt u informatie over het uitschakelen van services.
  
        Ga naar de volgende websites van Microsoft voor meer informatie over Groepsbeleid:
  
        -   [Verzameling groepsbeleidsregels](http://technet2.microsoft.com/windowsserver/en/library/6d7cb788-b31d-4d17-9f1e-b5ddaa6deecd1033.mspx?mfr=true)  
        -   [Wat is de Groepsbeleidsobjecteditor?](http://technet2.microsoft.com/windowsserver/en/library/47ba1311-6cca-414f-98c9-2d7f99fca8a31033.mspx?mfr=true)  
        -   [Essentiële hulpprogramma's en instellingen voor groepsbeleid](http://technet2.microsoft.com/windowsserver/en/library/e926577a-5619-4912-b5d9-e73d4bdc94911033.mspx?mfr=true)
  
    **Gevolgen van de tijdelijke oplossing:** Programma's die deze diensten vereisen worden niet uitgevoerd.
  
    **Zo kunt u de tijdelijke oplossing ongedaan maken:** Gebruik de stappen hierboven om het **Opstarttype** in te stellen op **Automatisch** en om de service te starten.
  
-   **De toegangsbeheerlijst in es.dll wijzigen**
  
    Ga als volgt te werk om de toegangsbeheerlijst (ACL) in es.dll te wijzigen en strengere restricties in te stellen:
  
    1.  Meld u aan als een gebruiker met beheerdersbevoegdheden.  
    2.  Klik achtereenvolgens op **Start** en **Uitvoeren**, typ **cmd** en klik op **OK**.  
    3.  Noteer de huidige ACL's voor het bestand (met inbegrip van overgenomen instellingen) voor het geval u deze aanpassing later ongedaan wilt maken. Typ het volgende als u de ACL's wilt weergeven:  
        cacls %WINDIR%\\SYSTEM32\\es.DLL  
    4.  Als u de groep "everyone" (iedereen) geen toegang tot het bestand wilt geven, typt u de volgende opdracht op de opdrachtregel:  
        echo y| cacls %WINDIR%\\SYSTEM32\\es.DLL /E /P everyone:N
  
    **Gevolgen van de tijdelijke oplossing:** Programma's die Gebeurtenissysteem vereisen worden niet uitgevoerd.
  
    **Zo kunt u de tijdelijke oplossing ongedaan maken:** Typ de volgende opdracht op de opdrachtprompt:
  
    echo y| cacls %WINDIR%\\SYSTEM32\\es.DLL /E /R everyone  
    Ga als volgt te werk om de toegangsbeheerlijst (ACL) in es.dll te wijzigen en minder strengere restricties in te stellen:
  
    1.  Meld u aan als een gebruiker met beheerdersbevoegdheden.  
    2.  Klik achtereenvolgens op **Start** en **Uitvoeren**, typ **cmd** en klik op **OK**.  
    3.  Noteer de huidige ACL's voor het bestand (met inbegrip van overgenomen instellingen) voor het geval u deze aanpassing later ongedaan wilt maken. Typ het volgende als u de ACL's wilt weergeven:  
        cacls %WINDIR%\\SYSTEM32\\es.DLL  
    4.  Als u de groep "everyone" (iedereen) toegang tot het bestand wilt geven, typt u de volgende opdracht op de opdrachtregel:  
        takeown.exe /f %WINDIR%\\SYSTEM32\\es.DLL  
        icacls.exe %WINDIR%\\SYSTEM32\\es.DLL /save %TEMP%\\es\_ACL.TXT  
        icacls.exe %WINDIR%\\SYSTEM32\\es.DLL /allow everyone:(F)
  
#### Veelgestelde vragen voor beveiligingslek Gebeurtenissysteem - CVE-2008-1457
  
**Wat is de omvang van het beveiligingslek?**    
Een beveiligingslek met betrekking tot het uitvoeren van externe code bestaat in het Microsoft Windows Gebeurtenissysteem. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan volledige controle krijgen over een systeem waarin dit probleem optreedt. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige beheerdersrechten maken.
  
**Waardoor wordt het beveiligingslek veroorzaakt?**    
Het Microsoft Windows Gebeurtenissysteem behandelt per-gebruikerabonnementen onjuist, waardoor willekeurige code kan worden uitgevoerd in de context van het lokale systeem.
  
**Wat is het Microsoft Windows Gebeurtenissysteem?**    
Microsoft Windows Gebeurtenissysteem is een service die methodeaanroepen en gebeurtenisabonnementen beheert tussen Windows en toepassingen op het systeem.
  
**Met welk doel kan een aanvaller het beveiligingslek misbruiken?**    
Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan de volledige controle krijgen over een systeem waarin dit probleem optreedt. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige beheerdersrechten maken.
  
**Hoe kan een aanvaller dit beveiligingslek misbruiken?**    
Een aanvaller kan alleen misbruik van dit beveiligingslek maken als hij of zij is geverifieerd. Een aanvaller kan vervolgens een speciaal vervaardigde gebeurtenisabonnementaanvraag verzenden naar het Microsoft Windows Gebeurtenissysteem waardoor de aanvaller code kan uitvoeren op systeemniveau en de volledige controle kan krijgen over het getroffen systeem.
  
**Voor welke systemen vormt dit beveiligingslek het grootste risico?**

Beveiligingslek Gebeurtenissysteem - CVE-2008-1456  
--------------------------------------------------
  
<span></span>
Een beveiligingslek met betrekking tot het uitvoeren van externe code bestaat omdat het Microsoft Windows Gebeurtenissysteem het bereik van indexen onjuist valideert wanneer een reeks functieverwijzingen wordt aangeroepen. Door het beveiligingslek kan externe code worden uitgevoerd. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan volledige controle krijgen over een systeem waarin dit probleem optreedt. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige beheerdersrechten maken.
  
Zie [CVE 2008-1456](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1456) als u dit beveiligingslek wilt weergeven als standaardvermelding in de lijst met veelvoorkomende beveiligingslekken en blootstellingen.
  
#### Beperkende factoren voor beveiligingslek Gebeurtenissysteem - CVE-2008-1456
  
Met beperkende factoren wordt verwezen naar een instelling, de algehele configuratie of algemene tips voor een standaardstatus waarmee de ernst van misbruik via een beveiligingslek kan worden verminderd. De volgende beperkende factoren kunnen in uw situatie van pas komen:
  
-   Een aanvaller moet zijn geverifieerd om misbruik van dit beveiligingslek te kunnen maken.
  
#### Tijdelijke oplossingen voor beveiligingslek Gebeurtenissysteem - CVE-2008-1456
  
Tijdelijke oplossingen zijn wijzigingen in een instelling of configuratie waardoor het beveiligingslek zelf niet wordt gedicht, maar waarmee aanvalsvectoren wel worden geblokkeerd voordat u de update uitvoert. Microsoft heeft de volgende methoden en statussen om het probleem te omzeilen getest om uit te zoeken of een tijdelijke oplossing functionaliteit vermindert:
  
-   **Maak de registratie ongedaan van es.dll**
  
    Voer de volgende stappen uit:
  
    1.  Meld u aan als gebruiker met beheerdersbevoegdheden.  
    2.  Klik achtereenvolgens op **Start** en **Uitvoeren**, typ **cmd** en klik op **OK**.  
    3.  Typ de volgende opdracht op de opdrachtprompt:  
        regsvr32 /-u %WINDIR%\\SYSTEM32\\es.dll
  
    **Gevolgen van de tijdelijke oplossing:** Het Gebeurtenissysteem wordt niet uitgevoerd.
  
    **Zo kunt u tijdelijke oplossingen ongedaan maken.** Registreer es.dll opnieuw. Typ de volgende opdracht op de opdrachtregel:
  
    regsvr 32 %windir%\\system32\\es.dll
  
-   **Stop de gebeurtenissysteemservices System Event Notification en COM+ en schakel die uit**
  
    Het uitschakelen van deze services helpt de kwetsbare systemen te beschermen tegen pogingen om dit beveiligingslek te misbruiken.
  
    -   Interactief
  
        Voer de volgende stappen uit:
  
        1.  Klik op **Start** en klik op **Configuratiescherm**. (Of klik op Start, wijs **Instellingen** aan en klik op **Configuratiescherm**.)  
        2.  Dubbelklik op **Systeembeheer**. Of klik op **Klassieke weergave** en dubbelklik op **Systeembeheer.**  
        3.  Dubbelklik op **Services**.  
        4.  Dubbelklik op **System Event Notification**.  
        5.  Klik in de lijst **Opstarttype** op **Uitgeschakeld**.

#### Veelgestelde vragen voor beveiligingslek Gebeurtenissysteem - CVE-2008-1456
  
**Wat is de omvang van het beveiligingslek?**    
Een beveiligingslek met betrekking tot het uitvoeren van externe code bestaat in het Microsoft Windows Gebeurtenissysteem. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan volledige controle krijgen over een systeem waarin dit probleem optreedt. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige beheerdersrechten maken.
  
**Waardoor wordt het beveiligingslek veroorzaakt?**    
Het Microsoft Windows Gebeurtenissysteem gebruikt een methodeparameter als index naar een vaste reeks functieverwijzingen. De reeks indexen wordt onjuist gevalideerd, waardoor willekeurige code kan worden uitgevoerd in de context van het lokale systeem.
  
**Wat is het Microsoft Windows Gebeurtenissysteem?**    
Microsoft Windows Gebeurtenissysteem is een service die methodeaanroepen en gebeurtenisabonnementen beheert tussen Windows en toepassingen op het systeem.
  
**Met welk doel kan een aanvaller het beveiligingslek misbruiken?**    
Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan de volledige controle krijgen over een systeem waarin dit probleem optreedt. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige beheerdersrechten maken.
  
**Hoe kan een aanvaller dit beveiligingslek misbruiken?**    
Een aanvaller kan alleen misbruik van dit beveiligingslek maken als hij of zij is geverifieerd. Vervolgens moet de aanvaller het Microsoft Windows Gebeurtenissysteem aanroepen en een speciaal vervaardigde aanvraag doen buiten de grenzen van de reeks. Door een wijzer naar een voorspelbare locatie ten opzichte de reeks te vinden, kan een aanvaller code uitvoeren met verhoogde bevoegdheden en volledige controle krijgen over het getroffen systeem.
  
**Voor welke systemen vormt dit beveiligingslek het grootste risico?**    
Alle systemen die kwetsbare versies van Microsoft Windows Gebeurtenissysteem uitvoeren lopen risico door dit beveiligingslek.
  
**Wat doet de update?**    
De update verwijdert het beveiligingslek door de manier te veranderen waarop het Microsoft Windows Gebeurtenissysteem indexreeksen behandelt en wel zo dat abonnementen geen code uitvoeren in de beveiligingscontext van een gebeurtenisuitgever.

**Controleren of de update is toegepast**

-   **Microsoft Baseline Security Analyzer**

    Gebruik het hulpprogramma Microsoft Baseline Security Analyzer (MBSA) om te controleren of de beveiligingsupdate is toegepast op het systeem waarvoor het probleem geldt. Zie de sectie Hulpmiddelen en richtlijnen voor detecteren en implementeren eerder in dit bulletin voor meer informatie.

-   **Verificatie van bestandsversie**

    Omdat er verschillende versies zijn van Microsoft Windows, kunnen de volgende stappen voor uw computer enigszins afwijken. Als dit het geval is, raadpleeg dan de productdocumentatie om deze stappen uit te voeren.

    1.  Klik op **Start** en klik vervolgens op **Zoeken**.
    2.  Klik in het venster **Zoekresultaten** op **Alle bestanden en mappen** onder **Zoekassistent**.
    3.  Typ in het vak **De volledige of gedeeltelijke bestandsnaam** een bestandsnaam uit de betreffende bestandsinformatietabel en klik vervolgens op **Zoeken**.
    4.  Klik in de lijst met bestanden met de rechtermuisknop op een bestandsnaam uit de betreffende bestandsinformatietabel en klik vervolgens op **Eigenschappen**.
        **Opmerking** Afhankelijk van de versie van het geïnstalleerde besturingssysteem of de geïnstalleerde programma's worden sommige bestanden in de bestandsinformatietabel mogelijk niet geïnstalleerd.
    5.  Bepaal op het tabblad **Versie** de versie van het bestand dat op uw computer is geïnstalleerd door de versie te vergelijken met de versie in de betreffende bestandsinformatietabel.
        **Opmerking** Andere kenmerken dan de bestandsversie kunnen tijdens de installatie worden veranderd. Vergelijking van andere bestandskenmerken met de gegevens in de bestandsinformatietabel vormt geen ondersteunde methode om te controleren of de update is toegepast. Bovendien kunnen in bepaalde gevallen de namen van bestanden tijdens installatie worden gewijzigd. Als de bestands- of versiegegevens niet aanwezig zijn, gebruikt u een van de andere beschikbare methoden om te controleren of de update is geïnstalleerd.

-   **Verificatie van registersleutel**

    Aan de hand van de registersleutels in de **Referentietabel** in deze sectie kunt u ook controleren welke bestanden met deze beveiligingsupdate zijn geïnstalleerd.

    Deze registersleutels bevatten mogelijk geen volledige lijst met geïnstalleerde bestanden. Bovendien zijn deze registersleutels mogelijk niet correct als een beheerder of een OEM de beveiligingsupdate integreert in de bronbestanden van de Windows-installatie.
