Microsoft-beveiligingsbulletin MS09-066 - Belangrijk
====================================================

Beveiligingslek in Active Directory kan leiden tot denial of service (973309)
-----------------------------------------------------------------------------

Gepubliceerd: dinsdag 10 november 2009

**Versie:** 1.0

### Algemene informatie

#### Samenvatting

Met deze beveiligingsupdate wordt een privé gemeld beveiligingslek in de Active Directory-service, Active Directory Application Mode (ADAM) en Active Directory Lightweight Directory Service (AD LDS) opgelost. Het beveiligingslek kan leiden tot denial of service als de stackruimte volledig wordt gebruikt bij het uitvoeren van bepaalde soorten LDAP- of LDAPS-aanvragen. Dit beveiligingslek treft alleen domeincontrollers en systemen die zijn geconfigureerd voor ADAM of AD LDS.

Het prioriteitsniveau van dit bulletin is Belangrijk voor Active Directory, ADAM en AD LDS op alle ondersteunde edities van Microsoft Windows 2000 Server, Windows XP, Windows Server 2003 en Windows Sever 2008. Zie de subsectie **Software waarin dit probleem optreedt en software waarin dit probleem niet optreedt** in deze sectie voor meer informatie.

De beveiligingsupdate lost het beveiligingslek op door de manier te wijzigen waarop Active Directory, ADAM en AD LDS verkeerd ingedeelde LDAP- of LDAPS-aanvragen verwerken. Zie de subsectie Veelgestelde vragen over het specifieke beveiligingslek in de volgende sectie, **Informatie over het beveiligingslek**, voor meer informatie over het beveiligingslek.

**Aanbeveling.**  De meeste klanten hebben automatische updates ingeschakeld en hoeven geen actie te ondernemen omdat deze beveiligingsupdate automatisch wordt gedownload en geïnstalleerd. Klanten die automatische updates niet hebben ingeschakeld, moeten controleren op updates en deze update handmatig installeren. Zie [Microsoft Knowledge Base-artikel 294871](http://support.microsoft.com/kb/294871) voor informatie over specifieke configuratieopties voor het uitvoeren van automatische updates.

Als het gaat om beheerders en bedrijfinstallaties of eindgebruikers die deze beveiligingsupdate handmatig willen installeren, adviseert Microsoft klanten de update zo snel mogelijk toe te passen met behulp van de software voor updatebeheer of door op updates te controleren met de service [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Zie ook de sectie **Hulpmiddelen en richtlijnen voor detecteren en implementeren**, verderop in dit bulletin.

**Bekende problemen.** Geen

Beveiligingslek in LSASS met betrekking tot recursieve stack-overloop – CVE-2009-1928
-------------------------------------------------------------------------------------

<span></span>
Er is een "denial of service"-beveiligingslek aanwezig in implementaties van Active Directory op Microsoft Windows 2000 Server, Windows Server 2003 en Windows Server 2008. Het beveiligingslek is ook aanwezig in implementaties van Active Directory Application Mode (ADAM) indien geïnstalleerd op Windows XP en Windows Server 2003 en Active Directory LightWeight Directory Services (AD LDS) indien geïnstalleerd op Windows Server 2008. Het beveiligingslek wordt veroorzaakt door onvoldoende validatie van speciaal vervaardigde LDAP-aanvragen. Een aanvaller die dit lek weet te misbruiken, kan ervoor zorgen dat het systeem waarin dit probleem optreedt niet meer reageert.

Zie [CVE 2009-1928](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-1928) als u dit beveiligingslek wilt weergeven als standaardvermelding in de lijst met veelvoorkomende beveiligingslekken en blootstellingen.

#### Beperkende factoren voor het beveiligingslek in LSASS met betrekking tot recursieve stack-overloop - CVE-2009-1928

Met beperkende factoren wordt verwezen naar een instelling, de algehele configuratie of algemene tips voor een standaardstatus waarmee de ernst van misbruik via een beveiligingslek kan worden verminderd. De volgende beperkende factoren kunnen in uw situatie van pas komen:

-   Met aanbevolen procedures voor firewalls en standaardfirewallconfiguraties kunt u netwerken beveiligen tegen aanvallen die van buiten het bedrijfsnetwerk komen. Het wordt aanbevolen op de systemen die met internet zijn verbonden zo min mogelijk poorten bloot te stellen aan deze aanvallen.

#### Tijdelijke oplossingen voor het beveiligingslek in LSASS met betrekking tot recursieve stack-overloop - CVE-2009-1928

Tijdelijke oplossingen zijn wijzigingen in een instelling of configuratie waardoor het beveiligingslek zelf niet wordt gedicht, maar waarmee aanvalsvectoren wel worden geblokkeerd voordat u de update uitvoert. Microsoft heeft de volgende methoden en statussen om het probleem te omzeilen getest om uit te zoeken of een tijdelijke oplossing functionaliteit vermindert:

-   **TCP-poorten 389, 636, 3268 en 3269 bij de firewall blokkeren**

    Blokkeer het volgende in de firewall:

    TCP-poorten 389, 636, 3268 en 3269

    Deze poorten worden gebruikt om een verbinding tot stand te brengen met het onderdeel waarin dit probleem optreedt. Door deze poorten voor zowel binnenkomend als uitgaand verkeer op de bedrijfsfirewall te blokkeren, zorgt u ervoor dat op systemen achter deze firewall geen aanvallen kunnen worden uitgevoerd. De poorten 389 en 636 bieden respectievelijk LDAP-services en beveiligde LDAP-services. De poorten 3268 en 3269 worden gebruikt door de Global Catalog-server die ook LDAP-aanvragen verwerkt. In Active Directory Application Mode (ADAM) en Active Directory Lightweight Directory Service (AD LDS) kunnen beheerders niet-standaard LDAP-poorten configureren. Wanneer deze tijdelijke oplossing wordt toegepast, dienen ook die poorten bij de firewall geblokkeerd te worden om misbruik te voorkomen.

    U wordt aanbevolen alle ongewenste inkomende communicatie via internet te blokkeren om aanvallen te voorkomen waarvoor andere poorten worden gebruikt. Ga naar [TCP and UDP Port Assignments](http://go.microsoft.com/fwlink/?linkid=21312) voor meer informatie over poorten. Zie [Windows Firewall op één computer configureren](http://www.microsoft.com/technet/security/smallbusiness/prodtech/windowsxp/cfgfwall.mspx) voor meer informatie over de Windows Firewall.

#### Veelgestelde vragen over het beveiligingslek in LSASS met betrekking tot recursieve stack-overloop - CVE-2009-1928

**Wat is de omvang van het beveiligingslek?**  
Dit beveiligingslek heeft betrekking op een denial of service. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan ervoor zorgen dat het systeem waarin dit probleem optreedt, niet meer reageert en opnieuw moet worden opgestart. Het beveiligingslek betreft een 'denial-of-service'-probleem. Een aanvaller kan geen code uitvoeren of zichzelf gebruikersrechten toekennen, maar kan er wel voor zorgen dat het systeem waarin dit probleem optreedt geen aanvragen meer accepteert.

**Waardoor wordt het beveiligingslek veroorzaakt?**  
De LDAP-service verwerkt specifieke LDAP- of LDAPS-aanvragen op een manier die kan leiden tot volledig gebruik van de stackruimte.

**Wat is Active Directory?**  
Het hoofddoel van Active Directory is het leveren van centrale verificatie- en validatieservices voor Windows-computers.

**Wat is Active Directory Lightweight Directory Service?**  
Active Directory Lightweight Directory Services (AD LDS) is een onafhankelijke modus van Active Directory die speciale directoryservices voor toepassingen levert. AD LDS is beschikbaar in Windows Server 2008 en hoger en vervangt Active Directory Application Mode (ADAM), beschikbaar in Windows XP en Windows Server 2003.

**Wat is LDAP?**  
Lightweight Directory Access Protocol (LDAP) is een open netwerkprotocolstandaard die is ontworpen om toegang tot gedistribueerde directory's te verzorgen.

**Wat is LDAP over SSL (LDAPS)?**  
Standaard worden LDAP-gegevens onbeveiligd verzonden. Het is echter mogelijk om LDAP-gegevens met behulp van de SSL-technologie (Secure Sockets Layer) en TLS-technologie (Transport Layer Security) vertrouwelijk te maken en te beveiligen tegen wijzigingen. Beheerders kunnen LDAP over SSL (LDAPS) inschakelen door volgens de richtlijnen uit [Microsoft Knowledge Base-artikel 321051](http://support.microsoft.com/kb/321051) een goed ingedeeld digitaal certificaat van een Microsoft-certificeringsinstantie (CA) of een certificeringsinstantie van een andere leverancier te installeren.

**Is LDAP over SSL standaard beschikbaar?**  
Nee. Voordat een LDAP-server aan een SSL-sessie kan deelnemen, moet de beheerder een digitaal certificaat aanvragen en op de server installeren. Alleen dan is LDAP over SSL beschikbaar.

**Hoe kan ik controleren of de LDAP-service wordt uitgevoerd op mijn server als ik Active Directory gebruik**?  
U kunt controleren of een server naar LDAP- of LDAPS-aanvragen luistert door de volgende opdrachten vanaf een beheerdersopdrachtregel uit te voeren en te kijken of het systeem op de LDAP-poort (389) of de LDAPS-poort (636) of de Global Catalog-servicepoorten (3268 of 3269) luistert:
netstat –a
LDAP is ingeschakeld als de resultaten de volgende gegevens bevatten:
Proto Local Address Foreign Address State
TCP 0.0.0.0:389 0.0.0.0:0 LISTENING
TCP 0.0.0.0:636 0.0.0.0:0 LISTENING
TCP 0.0.0.0:3268 0.0.0.0:0 LISTENING
TCP 0.0.0.0:3269 0.0.0.0:0 LISTENING

**Met welk doel kan een aanvaller het beveiligingslek misbruiken?**  
Een aanvaller die dit lek weet te misbruiken, kan ervoor zorgen dat het systeem van een gebruiker niet meer reageert en opnieuw moet worden opgestart.

**Hoe kan een aanvaller dit beveiligingslek misbruiken?**  
Een aanvaller kan proberen het beveiligingslek te misbruiken door een speciaal ontworpen LDAP- of LDAPS-pakket naar de ADAM-, AD LDS- of Active Directory-server te verzenden. Op Microsoft Windows 2000 Server kan een anonieme gebruiker met toegang tot het doelnetwerk een speciaal ontworpen netwerkpakket bezorgen bij het systeem waarin dit probleem optreedt om misbruik te maken van dit beveiligingslek. Op alle andere getroffen platforms moet de aanvaller geldige verificatiereferenties hebben om misbruik te kunnen maken van dit beveiligingslek.

**Voor welke systemen vormt dit beveiligingslek het grootste risico?**  
Lightweight Directory Access Protocol (LDAP) is vereist voor Active Directory, ADAM en AD LDS. Daardoor heeft elk systeem dat als ADAM-, AD LDS- of Active Directory-server wordt uitgevoerd last van dit beveiligingslek.

**Wat doet de update?**  
De update lost het beveiligingslek door verkeerd ingedeelde LDAP- of LDAPS-aanvragen op de juiste manier te verwerken.
