Security Bulletin

Microsoft-beveiligingsbulletin MS09-070 - Belangrijk
====================================================

Door beveiligingslekken in Active Directory Federation Services kan externe code worden uitgevoerd (971726)
-----------------------------------------------------------------------------------------------------------

Gepubliceerd: dinsdag 8 december 2009 | Bijgewerkt: woensdag 9 december 2009

**Versie:** 1.1

### Algemene informatie

#### Samenvatting

Met deze beveiligingsupdate worden twee privé gemelde beveiligingslekken in Microsoft Windows opgelost. Door het ernstigste beveiligingslek kan externe code worden uitgevoerd indien een aanvaller een speciaal vervaardigde HTTP-aanvraag verzendt naar een ADFS-webserver. Een aanvaller moet een geverifieerde gebruiker zijn om misbruik te kunnen maken van deze beveiligingslekken.

Het prioriteitsniveau van deze beveiligingsupdate is Belangrijk voor getroffen releases van Windows Server 2003, Windows Server 2003 x64 Edition, Windows Server 2008 en Windows Server 2008 x64 Edition. Zie de subsectie **Software waarin dit probleem optreedt en software waarin dit probleem niet optreedt** in deze sectie voor meer informatie.

De beveiligingsupdate lost de beveiligingslekken op door de validatie te verbeteren die ADFS-webservers toepassen om van een webclient ontvangen headers aan te vragen. Zie de subsectie Veelgestelde vragen over het specifieke beveiligingslek in de volgende sectie, **Informatie over het beveiligingslek**, voor meer informatie over de beveiligingslekken.

**Aanbeveling.**  De meeste klanten hebben automatische updates ingeschakeld en hoeven geen actie te ondernemen omdat deze beveiligingsupdate automatisch wordt gedownload en geïnstalleerd. Klanten die automatische updates niet hebben ingeschakeld, moeten controleren op updates en deze update handmatig installeren. Zie [Microsoft Knowledge Base-artikel 294871](http://support.microsoft.com/kb/294871) voor informatie over specifieke configuratieopties voor het uitvoeren van automatische updates.

Als het gaat om beheerders en bedrijfinstallaties of eindgebruikers die deze beveiligingsupdate handmatig willen installeren, adviseert Microsoft klanten de update zo snel mogelijk toe te passen met behulp van de software voor updatebeheer of door op updates te controleren met de service [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Zie ook de sectie **Hulpmiddelen en richtlijnen voor detecteren en implementeren**, verderop in dit bulletin.

**Bekende problemen.** In [Microsoft Knowledge Base-artikel 971726](http://support.microsoft.com/kb/971726) worden bekende problemen beschreven die klanten mogelijk ondervinden bij de installatie van deze beveiligingsupdate. **Wat is Active Directory Federation Services?**  
Active Directory Federation Services (ADFS) is gebaseerd op de opkomende webservicearchitectuur, die is gedefinieerd in WS-\*-specificaties. ADFS helpt u Single Sign-On (SSO) te gebruiken om gebruikers te verifiëren voor diverse, verwante webtoepassingen tijdens de duur van één online sessie. ADFS doet dit door digitale identiteit en validatie veilig te delen over beveiligings- en bedrijfsgrenzen.

**Hoe kan ik controleren of ADFS op mijn server word geïnstalleerd? **
Als u wilt controleren of ADFS op uw server is geïnstalleerd, klikt u op **Start**, vervolgens op **Configuratiescherm** en op **Software**. Dubbelklik op **Windows-onderdelen** en zoek naar **Federation Services** bij de geïnstalleerde onderdelen.

**Wat zijn de WS-\*-webspecificaties?**

De specificaties voor een webservicearchitectuur die is gebaseerd op industriestandaarden zoals Simple Object Access Protocol (SOAP); XML; Web Service Description Language (WSDL); en Universal Description, Discovery, and Integration (UDDI). WS-\* vormt een basis voor het leveren van volledige, interoperabele bedrijfsoplossingen voor uitgebreide ondernemingen, waaronder de mogelijkheid om gezamenlijke identiteit en beveiliging te beheren.

Het webservicesmodel is ontwikkeld omdat bedrijfssystemen in verschillende talen zijn geschreven, met verschillende programmeermodellen, die op en via verschillende soorten apparatuur worden uitgevoerd. Webservices vormen een manier voor het bouwen van diverse systemen die eenvoudig en efficiënt met elkaar kunnen communiceren via internet, ongeacht in welke taal ze zijn geschreven of op welk platform ze worden uitgevoerd.

Beveiligingslek met betrekking tot Single Sign-On-vervalsing in ADFS - CVE-2009-2508  
------------------------------------------------------------------------------------
  
<span></span>
Door een beveiligingslek met betrekking tot vervalsing in Active Directory Federation Services kan een aanvaller zich voordoen als een bevoegde gebruiker als de aanvaller toegang heeft tot een werkstation en webbrowser die onlangs zijn gebruikt door de aangevallen gebruiker om een website met Single Sign-On te openen.
  
Zie [CVE 2009-2508](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2508) als u dit beveiligingslek wilt weergeven als standaardvermelding in de lijst met veelvoorkomende beveiligingslekken en blootstellingen.
  
#### Beperkende factoren voor het beveiligingslek met betrekking tot Single Sign-On-vervalsing in ADFS - CVE-2009-2508
  
Met beperkende factoren wordt verwezen naar een instelling, de algehele configuratie of algemene tips voor een standaardstatus waarmee de ernst van misbruik via een beveiligingslek kan worden verminderd. De volgende beperkende factoren kunnen in uw situatie van pas komen:
  
-   Een aanvaller kan niet kiezen als wie hij zich voordoet. Een aanvaller kan zich alleen voordoen als iemand van wie de aanvaller het validatietoken heeft bemachtigd, bijvoorbeeld in een internetkiosk.  
-   Een aanval is alleen mogelijk totdat de sessie verloopt op de server. Standaard blijven sessies gedurende 600 minuten geopend op de server. Als een aanvaller na die periode probeert het validatietoken van een slachtoffer te gebruiken, mislukt de aanval.  
-   Een aanvaller moet toegang hebben tot het validatietoken dat door de vorige gebruiker is gebruikt op die computer. In het geval van een HTTPS-server is dat validatietoken beveiligd tijdens de overdracht en moet een aanvaller toegang hebben tot de computer van het slachtoffer (bijvoorbeeld in een internetkiosk) om misbruik te kunnen maken van het beveiligingslek. In een internetkiosk kan de gebruiker zich aanmelden bij een webtoepassing met ADFS Single Sign-On en zich vervolgens afmelden bij de toepassing. Een aanvaller kan vervolgens die computer gebruiken om de webtoepassing te openen als de vorige gebruiker, ook al heeft de gebruiker zich afgemeld.
  
#### Tijdelijke oplossingen voor het beveiligingslek met betrekking tot Single Sign-On-vervalsing in ADFS - CVE-2009-2508
  
Microsoft heeft nog geen oplossingen voor dit beveiligingslek.
  
#### Veelgestelde vragen over het beveiligingslek met betrekking tot Single Sign-On-vervalsing in ADFS - CVE-2009-2508
  
**Wat is de omvang van het beveiligingslek?**    
Dit is een beveiligingslek dat betrekking heeft op spoofing. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan zich voordoen als een andere gebruiker, waardoor de aanvaller als de aangevallen gebruiker toegang krijgt tot een website die gebruikmaakt van de Single Sign-On-functionaliteit van Active Directory Federation Services (ADFS).
  
**Waardoor wordt het beveiligingslek veroorzaakt?**    
Dit beveiligingslek bestaat omdat ADFS sessiebeheer onvoldoende valideert.
  
**Met welk doel kan een aanvaller het beveiligingslek misbruiken?**    
Op systemen met gedeelde toegang, zoals in een internetkiosk, kan een aanvaller die misbruik weet te maken van dit beveiligingslek toegang krijgen tot bronnen van een eerder geverifieerde gebruiker. Deze aanval kan alleen worden uitgevoerd binnen de sessietijd die is geconfigureerd door de ADFS-beheerder. Standaard is deze tijd 600 minuten.
  
**Hoe kan een aanvaller dit beveiligingslek misbruiken?**    
Een aanvaller kan misbruik maken van dit beveiligingslek door specifieke gegevens in de cache van de browser te openen en deze gegevens te gebruiken om zich te valideren bij de webtoepassing met ADFS Single Sign-On. Nadat de sessie is verlopen op de server, schakelt ADFS de gevalideerde sessie uit op de server en kan de aanvaller de gegevens niet langer gebruiken.
  
**Voor welke systemen vormt dit beveiligingslek het grootste risico?**    
Dit beveiligingslek vormt een risico voor ADFS-webservers voor Single Sign-On-doeleinden. Clientsystemen worden niet rechtstreeks getroffen door dit beveiligingslek.
  
**Wat doet de update?**    
De update lost het beveiligingslek op door validatiegegevens op de juiste wijze te wissen wanneer de gebruiker zich afmeldt.
  
**Was dit beveiligingslek openbaar gemaakt op het moment dat dit beveiligingsbulletin werd uitgegeven?**    
Nee. Microsoft heeft uit discrete bronnen informatie over dit beveiligingslek ontvangen.
  
**Was aan Microsoft gerapporteerd dat er misbruik was gemaakt van dit beveiligingslek op het moment dat dit beveiligingsbulletin werd uitgegeven?**    
Nee. Microsoft heeft geen informatie ontvangen die erop wees dat dit beveiligingslek was gebruikt om aanvallen uit te voeren op klanten op het moment dat dit beveiligingsbulletin oorspronkelijk werd uitgegeven.
  
Beveiligingslek met betrekking tot het uitvoeren van externe code in ADFS - CVE-2009-2509  
-----------------------------------------------------------------------------------------
  
<span></span>
Er bestaat een beveiligingslek met betrekking tot het uitvoeren van externe code in implementaties van Active Directory Federation Services (ADFS). Het beveiligingslek wordt veroorzaakt door onjuiste validatie van aanvraagheaders wanneer een geverifieerde gebruiker verbinding maakt met een ADFS-webserver. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan volledige controle krijgen over een systeem waarin dit probleem optreedt.
  
Zie [CVE 2009-2509](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2509) als u dit beveiligingslek wilt weergeven als standaardvermelding in de lijst met veelvoorkomende beveiligingslekken en blootstellingen.
  
#### Beperkende factoren voor het beveiligingslek met betrekking tot het uitvoeren van externe code in ADFS - CVE-2009-2509
  
Met beperkende factoren wordt verwezen naar een instelling, de algehele configuratie of algemene tips voor een standaardstatus waarmee de ernst van misbruik via een beveiligingslek kan worden verminderd. De volgende beperkende factoren kunnen in uw situatie van pas komen:
  
-   De aanvaller moet geldige aanmeldingsgegevens voor de kwetsbare server hebben. Een aanval kan alleen worden uitgevoerd na succesvolle verificatie bij de server die ADFS ondersteunt.
  
#### Tijdelijke oplossingen voor het beveiligingslek met betrekking tot het uitvoeren van externe code in ADFS - CVE-2009-2509
  
Microsoft heeft nog geen oplossingen voor dit beveiligingslek.
  
#### Veelgestelde vragen over het beveiligingslek met betrekking tot het uitvoeren van externe code in ADFS - CVE-2009-2509
  
**Wat is de omvang van het beveiligingslek?**    
Dit beveiligingslek heeft betrekking op de uitvoering van externe code. Een aanvaller die dit beveiligingslek weet te misbruiken, kan handelingen op de IIS-server verrichten met dezelfde rechten als de werkprocesidentiteit (WPI), die standaard met netwerkservice-accountbevoegdheden wordt geconfigureerd.
  
**Waardoor wordt het beveiligingslek veroorzaakt?**    
Dit beveiligingslek wordt veroorzaakt doordat ADFS aanvraagheaders van geverifieerde clients niet juist valideert.
  
**Met welk doel kan een aanvaller het beveiligingslek misbruiken?**    
Een aanvaller die dit beveiligingslek weet te misbruiken, zou vervolgens handelingen op de IIS-server kunnen verrichten met dezelfde rechten als de werkprocesidentiteit (WPI). De WPI wordt standaard met netwerkservice-accountbevoegdheden geconfigureerd. IIS-servers met ASP-pagina's waarvan de toepassingsgroepen zijn geconfigureerd met een WPI die gebruikmaakt van een account met beheerdersbevoegdheden lopen mogelijk een groter gevaar dan IIS-servers waarvan de toepassingsgroep is geconfigureerd met de standaard WPI-instellingen.
  
**Hoe kan een aanvaller dit beveiligingslek misbruiken?**    
Een aanvaller kan proberen misbruik te maken van het beveiligingslek door een speciaal vervaardigde HTTP-aanvraag te verzenden naar een ADFS-webserver. Een aanvaller moet een geverifieerde gebruiker zijn om misbruik te kunnen maken van dit beveiligingslek.
  
**Voor welke systemen vormt dit beveiligingslek het grootste risico?**    
ADFS-webservers lopen risico. Clientsystemen worden niet getroffen door dit beveiligingslek.
  
**Wat doet de update?**    
Deze update verbetert de validatie die ADFS-webservers toepassen op aanvraagheaders van een webclient.
  
**Was dit beveiligingslek openbaar gemaakt op het moment dat dit beveiligingsbulletin werd uitgegeven?**    
Nee. Microsoft heeft uit discrete bronnen informatie over dit beveiligingslek ontvangen.
  
**Was aan Microsoft gerapporteerd dat er misbruik was gemaakt van dit beveiligingslek op het moment dat dit beveiligingsbulletin werd uitgegeven?**

    Omdat er verschillende versies zijn van Microsoft Windows, kunnen de volgende stappen voor uw computer enigszins afwijken. Als dit het geval is, raadpleeg dan de productdocumentatie om deze stappen uit te voeren.

Vergelijking van andere bestandskenmerken met de gegevens in de bestandsinformatietabel vormt geen ondersteunde methode om te controleren of de update is toegepast. Bovendien kunnen in bepaalde gevallen de namen van bestanden tijdens installatie worden gewijzigd. | Schakeloptie      | Beschrijving                                                                                                                                                                        |  
|-------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|  
| **/?, /h, /help** | Geeft informatie over de ondersteunde schakelopties weer.                                                                                                                           |  
| **/quiet**        | Onderdrukt de weergave van status- of foutberichten.                                                                                                                                |  
| **/norestart**    | In combinatie met de schakeloptie **/quiet** wordt het systeem na de installatie niet opnieuw opgestart, zelfs als er opnieuw moet worden opgestart om de installatie af te ronden. |
