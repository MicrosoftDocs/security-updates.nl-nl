Security Bulletin

Microsoft-beveiligingsbulletin MS10-016 - Belangrijk
====================================================

Door een beveiligingslek in Windows Movie Maker kan externe code worden uitgevoerd (975561)
-------------------------------------------------------------------------------------------

Gepubliceerd: vrijdag 3 september 2010 | Bijgewerkt: woensdag 11 augustus 2010

**Versie:** 2.3

### Algemene informatie

#### Samenvatting

Deze beveiligingsupdate lost een privé gemeld beveiligingslek in Windows Movie Maker en Microsoft Producer 2003 op. Dit beveiligingslek is niet van toepassing op Windows Live Movie Maker, eveneens beschikbaar voor Windows Vista en Windows 7. Door het beveiligingslek kan externe code worden uitgevoerd als een aanvaller een speciaal vervaardigd Movie Maker- of Microsoft Producer-projectbestand verzendt en de gebruiker kan overhalen om het speciaal vervaardigde bestand te openen. Gebruikers met accounts waarvoor minder gebruikersrechten op het systeem zijn ingesteld, lopen minder risico dan gebruikers die met beheerdersrechten werken.

Het prioriteitsniveau van deze beveiligingsupdate is Belangrijk voor Windows Movie Maker 2.1, Windows Movie Maker 2.6, Windows Movie Maker 6.0 en Microsoft Producer 2003. Zie de subsectie **Software waarin dit probleem optreedt en software waarin dit probleem niet optreedt** in deze sectie voor meer informatie.

De beveiligingsupdate lost het beveiligingslek op door de manier te wijzigen waarop Windows Movie Maker projectbestanden parseert. Zie de subsectie Veelgestelde vragen over het specifieke beveiligingslek in de volgende sectie, **Informatie over het beveiligingslek**, voor meer informatie over het beveiligingslek.

**Aanbeveling.** De meeste klanten hebben automatische updates ingeschakeld en hoeven geen actie te ondernemen omdat deze beveiligingsupdate automatisch wordt gedownload en geïnstalleerd. Klanten die automatische updates niet hebben ingeschakeld, moeten controleren op updates en deze update handmatig installeren. Zie [Microsoft Knowledge Base-artikel 294871](http://support.microsoft.com/kb/294871) voor informatie over specifieke configuratieopties voor het uitvoeren van automatische updates.

Als het gaat om beheerders en bedrijfinstallaties of eindgebruikers die deze beveiligingsupdate handmatig willen installeren, adviseert Microsoft klanten de update zo snel mogelijk toe te passen met behulp van de software voor updatebeheer of door op updates te controleren met de service [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Zie ook de sectie **Hulpmiddelen en richtlijnen voor detecteren en implementeren**, verderop in dit bulletin.

**Bekende problemen.** Geen

Andere versies of edities hebben het einde van hun ondersteuningscyclus bereikt of hebben geen last van dit beveiligingslek. Ga naar [Microsoft Support Lifecycle](http://go.microsoft.com/fwlink/?linkid=21742) om de ondersteuningscyclus voor uw product en versie te bepalen.

Veelgestelde vragen met betrekking tot deze beveiligingsupdate  
--------------------------------------------------------------
  
<span></span>
**Waarom zijn er op 11 augustus 2010 wijzigingen in dit bulletin aangebracht?**    
Microsoft heeft wijzigingen in dit bulletin aangebracht om Windows Movie Maker 2.6 te verwijderen als een getroffen onderdeel van Windows 7. Windows Movie Maker 2.6 is geen ondersteund onderdeel van dit besturingssysteem.
  
**Waarom zijn er op 3 mei 2010 wijzigingen in dit bulletin aangebracht?**    
Microsoft heeft wijzigingen aangebracht in dit beveiligingsbulletin om de beschikbaarheid van [Microsoft Producer](http://www.microsoft.com/downloads/details.aspx?familyid=1b3c76d5-fc75-4f99-94bc-784919468e73) in het Microsoft Downloadcentrum aan te kondigen. Microsoft raadt gebruikers van Microsoft Producer 2003 aan een upgrade uit te voeren naar Microsoft Producer om het beveiligingslek dat wordt beschreven in dit bulletin op te lossen.
  
**Waar is de update voor Microsoft Producer 2003?**    
Microsoft raadt alle gebruikers van Microsoft Producer 2003 aan een upgrade uit te voeren naar de nieuwste versie van [Microsoft Producer](http://www.microsoft.com/downloads/details.aspx?familyid=1b3c76d5-fc75-4f99-94bc-784919468e73) om het beveiligingslek dat wordt beschreven in dit bulletin op te lossen. Klanten die de upgrade niet kunnen uitvoeren, kunnen de gevolgen voor systemen met Microsoft Producer 2003 beperken door de Microsoft Producer-bestandskoppelingen ongedaan te maken met behulp van de automatische oplossing in [Microsoft Knowledge Base-artikel 975561](http://support.microsoft.com/kb/975561), en door de tijdelijke oplossingen in dit bulletin toe te passen. Microsoft Producer is uitsluitend verkrijgbaar in het Engels.
  
**Wat is Microsoft Producer 2003?**  
Microsoft Producer 2003 is een downloadbare invoegtoepassing voor Microsoft Office PowerPoint 2002 en Microsoft Office PowerPoint 2003. Microsoft Producer 2003 is standaard niet geïnstalleerd. Microsoft Producer 2003 biedt gebruikers diverse functies voor het synchroniseren van audio, video, dia's en afbeeldingen voor uitgebreide presentaties die kunnen worden weergegeven in een browser.
  
Vanaf de revisie van dit bulletin van 3 mei 2010 is Microsoft Producer 2003 niet langer verkrijgbaar van het Microsoft Downloadcentrum en wordt de Microsoft Producer 2003 vervangen door de nieuwe versie, Microsoft Producer.
  
**Hoe weet ik welke versie van Windows Movie Maker op mijn systeem is geïnstalleerd?**    
U kunt uw versie van Movie Maker opzoeken door Movie Maker te starten via het menu **Start**. Klik op **Help** en vervolgens op **Info**.
  
**Kan er meer dan een versie van Windows Movie Maker geïnstalleerd zijn op één computer?**    
Ja, op Microsoft Vista hebt u Movie Maker 6.0, dat bij het besturingssysteem werd geleverd, en mogelijk hebt u ook Windows Movie Maker 2.6 geïnstalleerd, dat afzonderlijk kan worden gedownload. In dit geval worden beide versies van Movie Maker gedetecteerd en wordt de beveiligingsupdate voor beide versies aangeboden.
  
Op Windows Vista is het ook mogelijk Windows Live Movie Maker te installeren, een afzonderlijk product dat deel uitmaakt van Windows Live Essentials. Windows Live Movie Maker wordt niet getroffen en krijgt deze beveiligingsupdate niet aangeboden.

Beveiligingslek met betrekking tot bufferoverloop in Movie Maker en Producer – CVE-2010-0265  
--------------------------------------------------------------------------------------------
  
<span></span>
Er bestaat een beveiligingslek met betrekking tot het uitvoeren van externe code door de manier waarop Windows Movie Maker en Microsoft Producer 2003 speciaal vervaardigde projectbestanden verwerken. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan volledige controle krijgen over een systeem waarin dit probleem optreedt. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige gebruikersrechten maken.
  
Zie [CVE 2010-0265](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0265) als u dit beveiligingslek wilt weergeven als standaardvermelding in de lijst met veelvoorkomende beveiligingslekken en blootstellingen.
  
#### Beperkende factoren voor het beveiligingslek met betrekking tot bufferoverloop in Movie Maker en Producer – CVE-2010-0265
  
Met beperkende factoren wordt verwezen naar een instelling, de algehele configuratie of algemene tips voor een standaardstatus waarmee de ernst van misbruik via een beveiligingslek kan worden verminderd. De volgende beperkende factoren kunnen in uw situatie van pas komen:
  
-   In het geval van een aanval vanaf het web kan de aanvaller beschikken over een website waarop zich een webpagina bevindt waarmee dit beveiligingslek wordt misbruikt. Daarnaast kunnen ook gemanipuleerde websites en websites waarop gegevens of advertenties staan of gegevens of advertenties kunnen worden ingevoerd, schadelijke inhoud bevatten die dit beveiligingslek kunnen misbruiken. Een aanvaller kan echter een gebruiker er niet toe dwingen om naar deze websites te gaan. De aanvaller moet een gebruiker er dus van overtuigen een bezoek te brengen aan de website. In de meeste gevallen doet de aanvaller dat door de gebruiker ertoe te bewegen op een koppeling in een e-mailbericht of een verzoek in een expresbericht te klikken waarmee de gebruiker naar de website van de aanvaller gaat.  
-   Het speciaal vervaardigde bestand kan als een e-mailbijlage worden verzonden, maar om het beveiligingslek te kunnen misbruiken, moet de aanvaller de gebruiker ertoe overhalen de bijlage te openen.  
-   Een aanvaller die erin slaagt misbruik te maken van dit beveiligingslek, kan dezelfde rechten over het systeem krijgen als de lokale gebruiker. Gebruikers met accounts waarvoor minder gebruikersrechten op het systeem zijn ingesteld, lopen minder risico dan gebruikers die met beheerdersrechten werken.
  
#### Tijdelijke oplossingen voor het beveiligingslek met betrekking tot bufferoverloop in Movie Maker en Producer - CVE-2010-0265
  
Tijdelijke oplossingen zijn wijzigingen in een instelling of configuratie waardoor het beveiligingslek zelf niet wordt gedicht, maar waarmee aanvalsvectoren wel worden geblokkeerd voordat u de update uitvoert. Microsoft heeft de volgende methoden en statussen om het probleem te omzeilen getest om uit te zoeken of een tijdelijke oplossing functionaliteit vermindert:
  
-   **De Movie Maker .MSWMM-bestandskoppeling ongedaan maken**
  
    **Waarschuwing** Als u de Register-editor onjuist gebruikt, kan dit tot zeer ernstige problemen leiden waardoor u het besturingssysteem mogelijk opnieuw dient te installeren. Microsoft kan niet garanderen dat problemen die ontstaan als gevolg van een onjuist gebruik van de Register-editor, kunnen worden opgelost. Het gebruik van de Register-editor is voor eigen risico. Zie voor meer informatie over het bewerken van het register het Help-onderwerp "Sleutels en waarden wijzigen" in de Register-editor (regedit.exe) of de Help-onderwerpen "Toevoegen of verwijderen van registerinformatie" en "Registergegevens bewerken" in regedt32.exe.
  
    **Opmerking** Het is aan te raden een back-up te maken van het register voordat u het bewerkt.
  
    Volg deze stappen om de Movie Maker .MSWMM-bestandsassociatie ongedaan te maken:
  
    **Interactieve methode**
  
    1.  Klik op **Start** en vervolgens op **Uitvoeren**. Typ **regedit** en klik vervolgens op **OK**.  
    2.  Vouw **HKEY\_CLASSES\_ROOT** uit, klik op **.MSWMM** en klik vervolgens op het menu **Bestand** en selecteer **Exporteren**.  
    3.  Typ in het dialoogvenster **Registerbestand exporterenMSWMM file association registry backup.reg** en klik op **Opslaan**. Hierdoor wordt er standaard een back-up van deze registersleutel in de map 'Mijn documenten' gemaakt.  
    4.  Druk op **Verwijderen** om de registersleutel te verwijderen. Klik op **Ja** wanneer u wordt gevraagd of u de registerwaarde wilt verwijderen.
  
    **Script voor een beheerde implementatie**
  
    1.  Maak een back-up van de registersleutel met de volgende opdracht:  
        `Regedit.exe /e MSWMM_registry_backup.reg HKEY_CLASSES_ROOT\.MSWMM`  
    2.  Sla een bestand op met de extensie .REG (bijvoorbeeld Delete\_MSWMM\_file\_association.reg):  
        `Windows Registry-editor versie 5.00`  
        `  [-HKEY_CLASSES_ROOT\.MSWMM]`  
    3.  Voer het registerscript dat u in stap 2 hebt gemaakt uit op het doelsysteem met de volgende opdracht:  
        `Regedit.exe /s Delete_MSWMM_file_association.reg`
  
    **Gevolgen van de tijdelijke oplossing.** Met deze tijdelijke oplossing wordt de .MSWMM-bestandsassociatie ongedaan gemaakt. Als u dubbelklikt op een MSWMM-bestand wordt het bestand niet langer geopend in Windows Movie Maker.
  
    **De tijdelijke oplossing ongedaan maken.** Herstel de registersleutel door Regedit te gebruiken om de instellingen in het .REG-bestand te herstellen.
  
-   **Maak de Microsoft Producer 2003-, .MSProducer-, .MSProducerZ- en .MSProducerBF-bestandsassociaties ongedaan**
  
    **Opmerking** Zie [Microsoft Knowledge Base-artikel 975561](http://support.microsoft.com/kb/975561) als u de automatische **Microsoft Fix it**-oplossing wilt gebruiken om deze tijdelijke oplossing in of uit te schakelen.
  
    **Waarschuwing** Als u de Register-editor onjuist gebruikt, kan dit tot zeer ernstige problemen leiden waardoor u het besturingssysteem mogelijk opnieuw dient te installeren. Microsoft kan niet garanderen dat problemen die ontstaan als gevolg van een onjuist gebruik van de Register-editor, kunnen worden opgelost. Het gebruik van de Register-editor is voor eigen risico. Zie voor meer informatie over het bewerken van het register het Help-onderwerp "Sleutels en waarden wijzigen" in de Register-editor (regedit.exe) of de Help-onderwerpen "Toevoegen of verwijderen van registerinformatie" en "Registergegevens bewerken" in regedt32.exe.
  
    **Opmerking** Het is aan te raden een back-up te maken van het register voordat u het bewerkt.
  
    Volg deze stappen om de .MSProducer-, .MSProducerZ en .MSProducerBF-bestandsassociaties ongedaan te maken.
  
    **Interactieve methode**
  
    1.  Klik op **Start** en vervolgens op **Uitvoeren**. Typ **regedit** en klik vervolgens op **OK**.  
    2.  Vouw **HKEY\_CLASSES\_ROOT** uit, klik op **.MSProducer** en klik vervolgens op het menu **Bestand** en selecteer **Exporteren**.  
    3.  Typ in het dialoogvenster **Registerbestand exporterenMSProducerHKCR file association registry backup.reg** en klik op **Opslaan**. Hierdoor wordt er standaard een back-up van deze registersleutel in de map 'Mijn documenten' gemaakt.  
    4.  Druk op **Verwijderen** om de registersleutel te verwijderen. Klik op **Ja** wanneer u wordt gevraagd of u de registerwaarde wilt verwijderen.  
    5.  Herhaal de stappen 2-4 voor de .MSProducerZ- en .MSProducerBF-bestandsassociaties.  
    6.  Vouw **HKEY\_CURRENT\_USER** uit en vervolgens **Software**, dan **Microsoft**, dan **Windows**, dan **CurrentVersion**, dan **Explorer** en dan **FileExts**.  
    7.  Klik op **.MSProducer**, klik op **Bestand** en klik vervolgens op **Exporteren**.  
    8.  Typ in het dialoogvenster **Registerbestand exporterenMSProducer HKCU file association registry backup.reg** en klik op **Opslaan**. Hierdoor wordt er standaard een back-up van deze registersleutel in de map 'Mijn documenten' gemaakt.

#### Veelgestelde vragen over het beveiligingslek met betrekking tot bufferoverloop in Movie Maker en Producer – CVE-2010-0265
  
**Wat is de omvang van het probleem?**    
Dit beveiligingslek heeft betrekking op de uitvoering van externe code. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan op afstand volledige controle krijgen over een systeem waarin dit probleem optreedt. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige gebruikersrechten maken.
  
**Waardoor wordt het beveiligingslek veroorzaakt?**    
Het beveiligingslek ontstaat door de manier waarop Windows Movie Maker en Microsoft Producer 2003 indelingen van projectbestanden parseren. Om misbruik te kunnen maken van dit beveiligingslek, moet een aanvaller een speciaal vervaardigd Movie Maker- of Producer-bestand verzenden en de gebruiker ertoe overhalen het speciaal vervaardigde bestand te openen.
  
**Met welk doel kan een aanvaller het beveiligingslek misbruiken?**    
Een aanvaller die misbruik weet te maken van dit beveiligingslek kan zich voordoen als de aangemelde gebruiker en zo willekeurige programmacode uitvoeren. Als er een gebruiker met beheerdersrechten is aangemeld, kan een aanvaller volledige controle krijgen over het systeem waarin dit probleem optreedt. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige gebruikersrechten maken. Gebruikers met accounts waarvoor minder gebruikersrechten op het systeem zijn ingesteld, lopen minder risico dan gebruikers die met beheerdersrechten werken.
  
In een aanval per e-mail kan een aanvaller misbruik maken van het beveiligingslek door een speciaal gemaakt Movie Maker- of Producer-projectbestand naar de gebruiker te verzenden en de gebruiker ertoe te bewegen dat bestand te openen.
  
In het geval van een aanval vanaf het web moet de aanvaller beschikken over een website waarop zich een speciaal ontworpen Movie Maker- of Producer-projectbestand bevindt waarmee wordt geprobeerd dit beveiligingslek te misbruiken. Daarnaast kunnen ook gemanipuleerde websites en websites waarop gegevens kunnen worden ingevoerd, schadelijke inhoud bevatten die dit beveiligingslek kunnen misbruiken. Een aanvaller kan een gebruiker er niet toe dwingen om naar de speciaal vervaardigde website te gaan. De aanvaller moet een gebruiker er dus van overtuigen een bezoek te brengen aan de website. In de meeste gevallen doet de aanvaller dat door de gebruiker ertoe te bewegen op een koppeling te klikken waarmee de gebruiker naar de website van de aanvaller gaat. Vervolgens moet de aanvaller de gebruiker ertoe overhalen het speciaal vervaardigde Movie Maker- of Producer-projectbestand te openen.
  
**Hoe kan een aanvaller dit beveiligingslek misbruiken?**    
Voor misbruik van dit beveiligingslek is het noodzakelijk dat een gebruiker een speciaal vervaardigd Movie Maker- of Producer-projectbestand opent in een getroffen versie van Window Movie Maker of Microsoft Producer.
  
**Voor welke systemen vormt dit beveiligingslek het grootste risico?**    
Het risico is het grootst voor Windows-systemen waarop Windows Movie Maker of Microsoft Producer 2003 wordt gebruikt.
  
**Wat doet de update?**    
De update voor Movie Maker verwijdert het beveiligingslek door de manier te wijzigen waarop Windows Movie Maker-projectbestanden worden geparseerd in Windows Movie Maker. De download voor Microsoft Producer 2003 voert een upgrade uit voor de installatie van Microsoft Producer 2003 naar de nieuwe versie, Microsoft Producer.
  
**Was dit beveiligingslek openbaar gemaakt op het moment dat dit beveiligingsbulletin werd uitgegeven?**    
Nee. Microsoft heeft uit discrete bronnen informatie over dit beveiligingslek ontvangen.
  
**Was aan Microsoft gerapporteerd dat er misbruik was gemaakt van dit beveiligingslek op het moment dat dit beveiligingsbulletin werd uitgegeven?**    
Nee. Microsoft heeft geen informatie ontvangen die erop wees dat dit beveiligingslek was gebruikt om aanvallen uit te voeren op klanten op het moment dat dit beveiligingsbulletin oorspronkelijk werd uitgegeven.

    Omdat er verschillende versies zijn van Microsoft Windows, kunnen de volgende stappen voor uw computer enigszins afwijken. Als dit het geval is, raadpleeg dan de productdocumentatie om deze stappen uit te voeren.

    1.  Klik op **Start** en klik vervolgens op **Zoeken**.
