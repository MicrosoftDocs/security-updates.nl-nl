Security Bulletin

Microsoft-beveiligingsbulletin MS10-104 - Belangrijk
====================================================

Door een beveiligingslek in Microsoft SharePoint kan externe code worden uitgevoerd (2455005)
---------------------------------------------------------------------------------------------

Gepubliceerd: dinsdag 14 december 2010

**Versie:** 1.0

### Algemene informatie

#### Samenvatting

Met deze beveiligingsupdate wordt een privé gemeld beveiligingslek in Microsoft SharePoint opgelost. Door het beveiligingslek kan externe code worden uitgevoerd in de beveiligingscontext van een gastgebruiker als een aanvaller een speciaal vervaardigde SOAP-aanvraag naar de Document Conversions Launcher Service in een SharePoint-serveromgeving stuurt die de Document Conversions Load Balancer Service gebruikt. Standaard zijn de Document Conversions Load Balancer Service en Document Conversions Launcher Service in Microsoft Office SharePoint Server 2007 niet ingeschakeld.

Dit is een belangrijke beveiligingsupdate voor ondersteunde versies van Microsoft Office SharePoint Server 2007. Zie de subsectie **Software waarin dit probleem optreedt en software waarin dit probleem niet optreedt** in deze sectie voor meer informatie.

De update lost het beveiligingslek op door de manier te wijzigen waarop de Startservice voor documentconversies speciaal vervaardigde SOAP-aanvragen valideert. Zie de subsectie Veelgestelde vragen over het specifieke beveiligingslek in de volgende sectie, **Informatie over het beveiligingslek**, voor meer informatie over het beveiligingslek.

**Aanbeveling.** Microsoft raadt klanten aan de update zo spoedig mogelijk te installeren.

**Bekende problemen.** Geen.

#### Software waarin dit probleem optreedt en software waarin dit probleem niet optreedt

Andere versies of edities hebben het einde van hun ondersteuningscyclus bereikt of hebben geen last van dit beveiligingslek. Ga naar [Microsoft Support Lifecycle](http://go.microsoft.com/fwlink/?linkid=21742) om de ondersteuningscyclus voor uw product en versie te bepalen.

Beveiligingslek met betrekking tot het uitvoeren van code door verkeerd ingedeelde aanvragen - CVE-2010-3964  
------------------------------------------------------------------------------------------------------------
  
<span></span>
Er bestaat een beveiligingslek met betrekking tot het uitvoeren van externe code door de manier waarop de Startservice voor documentconversies SOAP-aanvragen valideert voordat ze worden verwerkt op een SharePoint-server. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan willekeurige code uitvoeren op een getroffen SharePoint-server in de beveiligingscontext van een gastaccount.
  
Zie [CVE 2010-3964](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3964) als u dit beveiligingslek wilt weergeven als standaardvermelding in de lijst met veelvoorkomende beveiligingslekken en blootstellingen.
  
#### Beperkende factoren voor het beveiligingslek met betrekking tot het uitvoeren van code door verkeerd ingedeelde aanvragen - CVE-2010-3964:
  
Met beperkende factoren wordt verwezen naar een instelling, de algehele configuratie of algemene tips voor een standaardstatus waarmee de ernst van misbruik via een beveiligingslek kan worden verminderd. De volgende beperkende factoren kunnen in uw situatie van pas komen:
  
-   Standaard zijn de Document Conversions Load Balancer Service en Document Conversions Launcher Service in Microsoft Office SharePoint Server 2007 niet ingeschakeld.  
-   Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan code uitvoeren als een gastaccount met rechten voor een specifieke tijdelijke directory. De inhoud van deze specifieke tijdelijke directory wordt regelmatig gewist.
  
#### Tijdelijke oplossing voor het beveiligingslek met betrekking tot het uitvoeren van code door verkeerd ingedeelde aanvragen - CVE-2010-3964:
  
Tijdelijke oplossingen zijn wijzigingen in een instelling of configuratie waardoor het beveiligingslek zelf niet wordt gedicht, maar waarmee aanvalsvectoren wel worden geblokkeerd voordat u de update uitvoert. Microsoft heeft de volgende methoden en statussen om het probleem te omzeilen getest om uit te zoeken of een tijdelijke oplossing functionaliteit vermindert:
  
-   **Startservice voor documentconversies in Office stoppen en uitschakelen**
  
    Voer als beheerder de volgende opdracht uit vanaf een opdrachtprompt op elk systeem met de Startservice voor documentconversies:  
    `sc stop dclauncher`  
    `sc config dclauncher start= disabled`
  
    **Gevolgen van de tijdelijke oplossing.** Office-documentconversie is uitgeschakeld.
  
    **De tijdelijke oplossing ongedaan maken.**
  
    Voer als beheerder de volgende opdracht uit vanaf een opdrachtprompt:  
    `sc config dclauncher start= auto`  
    `sc start dclauncher`
  
-   **Blokeer poorten die worden gebruikt door de Startservice voor documentconversies op de firewall**
  
    Blokkeer de volgende poort op de firewall:  
    TCP-poort 8082
  
    Deze poort wordt gebruikt om een verbinding tot stand te brengen met het onderdeel waarin dit probleem optreedt. Door deze poort voor zowel binnenkomend als uitgaand verkeer op de bedrijfsfirewall te blokkeren, zorgt u ervoor dat op systemen achter deze firewall geen aanvallen kunnen worden uitgevoerd. U wordt aanbevolen alle ongewenste inkomende communicatie via internet te blokkeren om aanvallen te voorkomen waarvoor andere poorten worden gebruikt. Zie het TechNet-artikel [TCP and UDP Port Assignments](http://go.microsoft.com/fwlink/?linkid=21312) voor meer informatie over poorten.
  
    **Gevolgen van de tijdelijke oplossing.** Toepassingen en services kunnen de betreffende poort gebruiken. Het blokkeren van de verbinding met de poort kan ertoe leiden dat verschillende toepassingen of services niet werken.
  
    **De tijdelijke oplossing ongedaan maken.**
  
    Deblokkeer TCP-poort 8082 op de firewall. Zie het TechNet-artikel [TCP and UDP Port Assignments](http://go.microsoft.com/fwlink/?linkid=21312) voor meer informatie over poorten.
  
#### Veelgestelde vragen over het beveiligingslek met betrekking tot het uitvoeren van code door verkeerd ingedeelde aanvragen - CVE-2010-3964:
  
**Wat is de omvang van het probleem?**    
Dit beveiligingslek heeft betrekking op de uitvoering van externe code. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan willekeurige code uitvoeren in de beveiligingscontext van een gastaccount op de getroffen SharePoint-server.
  
**Waardoor wordt het beveiligingslek veroorzaakt?**    
Dit beveiligingslek wordt veroorzaakt wanneer de Startservice voor documentconversies speciaal vervaardigde SOAP-aanvragen (Simple Object Access Protocol) niet goed valideert.
  
**Wat is de Startservice voor documentconversies?**    
De Startservice voor documentconversies plant en start de documentconversies op een server. Wanneer Microsoft Office SharePoint Server 2007 een aanvraag voor een documentconversie verzendt naar de service, start de Startservice voor documentconversies het geschikte documentconversieprogramma. In de meeste serverfarmconfiguraties moeten de Startservice voor documentconversies en de conversieprogramma's die door de service worden gestart, op een toepassingsserver worden geïnstalleerd. Op elke toepassingsserver kan slechts één Startservice voor documentconversies worden gebruikt.
  
De Startservice voor documentconversies wordt uitgevoerd onder de account HVU\_*Computernaam*. De account HVU\_*Computernaam* is een lokale gastaccount die wordt gemaakt op de toepassingsserver wanneer de Startservice voor documentconversies wordt ingeschakeld.
  
**Wat is de Gelijkmatige verdeling van de belasting voor documentconversies?**    
De Gelijkmatige verdeling van de belasting voor documentconversies verdeelt aanvragen voor documentconversies. De Gelijkmatige verdeling van de belasting voor documentconversies kan op een zelfstandige server of op een server in een farm worden geconfigureerd. Wanneer deze service een conversieaanvraag ontvangt van Microsoft Office SharePoint Server 2007, verzendt de Gelijkmatige verdeling van de belasting voor documentconversies een URI (Uniform Resource Identifier) die de geschikte Startservice voor documentconversies aanwijst. SharePoint Server 2007 maakt verbinding met de aangewezen startservice via SOAP-aanvragen en vraagt de service het opgegeven document te converteren.
  
Als u de Gelijkmatige verdeling van de belasting voor documentconversies uitschakelt, stopt het documentconversieproces en wordt ook de Startservice voor documentconversies uitgeschakeld.
  
**Met welk doel kan een aanvaller het beveiligingslek misbruiken?**    
Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan willekeurige code uitvoeren in de beveiligingscontext van een gastgebruiker op de getroffen server.
  
**Hoe kan een aanvaller dit beveiligingslek misbruiken?**    
Een aanvaller kan een speciaal vervaardigde SOAP-aanvraag verzenden naar de Startservice voor documentconversies in een SharePoint-serveromgeving die gebruikmaakt van de Gelijkmatige verdeling van de belasting voor documentconversies. Wanneer de Startservice voor documentconversies de speciaal vervaardigde SOAP-aanvraag verwerkt, kan door het beveiligingslek externe code worden uitgevoerd in de beveiligingscontext van een gastaccount op het getroffen systeem.
  
**Voor welke systemen vormt dit beveiligingslek het grootste risico?**    
Dit beveiligingslek vormt het grootste risico voor SharePoint-serveromgevingen die gebruikmaken van de Gelijkmatige verdeling van de belasting voor documentconversies. Door het beveiligingslek kan externe code worden uitgevoerd op servers met de Startservice voor documentconversies in de SharePoint-serveromgeving.
  
**Wat doet de update?**    
Deze update lost het beveiligingslek op door de manier te wijzigen waarop de Startservice voor documentconversies speciaal vervaardigde SOAP-aanvragen valideert.
  
**Was dit beveiligingslek openbaar gemaakt op het moment dat dit beveiligingsbulletin werd uitgegeven?**    
Nee. Microsoft heeft uit betrouwbare bronnen informatie over dit beveiligingslek ontvangen.
  
**Was aan Microsoft gerapporteerd dat er misbruik was gemaakt van dit beveiligingslek op het moment dat dit beveiligingsbulletin werd uitgegeven?**    
Nee. Microsoft heeft geen informatie ontvangen die erop wees dat dit beveiligingslek was gebruikt om aanvallen uit te voeren op klanten op het moment dat dit beveiligingsbulletin oorspronkelijk werd uitgegeven.

Updates schrijven vaak naar de bestanden en registerinstellingen die nodig zijn om uw toepassingen te kunnen uitvoeren. De Application Compatibility Toolkit (ACT) bevat de noodzakelijke hulpprogramma's en documentatie om problemen met de compatibiliteit van toepassingen te evalueren en te verminderen voordat Microsoft Windows Vista, een Windows-update, een Microsoft-beveiligingsupdate of een nieuwe versie van Windows Internet Explorer op uw systeem wordt geïnstalleerd.

Bij de installatie van deze beveiligingsupdate kunnen de volgende schakelopties worden gebruikt.
  
| Schakeloptie                    | Beschrijving                                                                                                                                             |  
|---------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------|  
| **/? **of **/help**             | Geeft informatie over het gebruik van het dialoogvenster.                                                                                                |  
| **/passive**                    | Geeft de passieve modus aan. Hierbij is geen interactie van de gebruiker nodig. U ziet wel de voortgangsvensters maar kunt deze vensters niet annuleren. |  
| **/quiet**                      | Hiermee schakelt u de stille modus in of onderdrukt u vragen aan de gebruiker wanneer bestanden worden uitgepakt.                                        |

| **/forcerestart**               | Start het systeem automatisch opnieuw op nadat de update is geïnstalleerd, ongeacht of het systeem wel of niet voor de update moet worden opgestart.     |  
| **/extract**                    | De bestanden worden uitgepakt maar niet geïnstalleerd. U wordt gevraagd een doelmap op te geven.                                                         |
