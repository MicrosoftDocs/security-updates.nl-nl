Security Bulletin

Microsoft-beveiligingsbulletin MS11-004 - Belangrijk
====================================================

Door een beveiligingslek in de FTP-service van Internet Information Services (IIS) kan externe code worden uitgevoerd (2489256)
-------------------------------------------------------------------------------------------------------------------------------

Gepubliceerd: dinsdag 8 februari 2011 | Bijgewerkt: dinsdag 8 maart 2011

**Versie:** 2.0

### Algemene informatie

#### Samenvatting

Met deze beveiligingsupdate wordt een openbaar gemeld beveiligingslek in de FTP-service van Internet Information Services (IIS) opgelost. Door het beveiligingslek kan externe code worden uitgevoerd als een FTP-server een speciaal vervaardigde FTP-opdracht ontvangt. De FTP-service is niet standaard geïnstalleerd op IIS.

Het prioriteitsniveau van deze beveiligingsupdate is Belangrijk voor Microsoft FTP Service 7.0 voor IIS 7.0 en Microsoft FTP Service 7.5 voor IIS 7.0 indien geïnstalleerd op alle ondersteunde edities van Windows Vista en Windows Server 2008, en voor Microsoft FTP Service 7.5 voor Internet Information Services 7.5 op alle ondersteunde edities van Windows 7 en Windows Server 2008 R2. Zie de subsectie **Software waarin dit probleem optreedt en software waarin dit probleem niet optreedt** in dit onderdeel voor meer informatie.

De beveiligingsupdate lost het beveiligingslek op door de manier te wijzigen waarop de FTP-service van IIS speciaal vervaardigde FTP-opdrachten afhandelt. Zie de subsectie Veelgestelde vragen over het specifieke beveiligingslek in de volgende sectie, **Informatie over het beveiligingslek**, voor meer informatie over het beveiligingslek.

**Aanbeveling.** De meeste klanten hebben automatische updates ingeschakeld en hoeven geen actie te ondernemen omdat deze beveiligingsupdate automatisch wordt gedownload en geïnstalleerd. Klanten die automatische updates niet hebben ingeschakeld, moeten controleren op updates en deze update handmatig installeren. Zie [Microsoft Knowledge Base-artikel 294871](http://support.microsoft.com/kb/294871) voor informatie over specifieke configuratieopties voor het uitvoeren van automatische updates.

Als het gaat om beheerders en bedrijfinstallaties of eindgebruikers die deze beveiligingsupdate handmatig willen installeren, adviseert Microsoft klanten de update zo snel mogelijk toe te passen met behulp van de software voor updatebeheer of door op updates te controleren met de service [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Zie ook de sectie **Hulpmiddelen en richtlijnen voor detecteren en implementeren**, verderop in dit bulletin.

**Bekende problemen.** In [Microsoft Knowledge Base-artikel 2489256](http://support.microsoft.com/kb/2489256) worden bekende problemen beschreven die klanten mogelijk ondervinden bij de installatie van deze beveiligingsupdate. In het artikel worden ook de aanbevolen oplossingen voor deze problemen toegelicht.

Veelgestelde vragen met betrekking tot deze beveiligingsupdate
--------------------------------------------------------------

<span></span>
**Waarom is dit bulletin herzien op 8 maart 2011?**  
Microsoft heeft het onderdeel Software waarin dit probleem optreedt verduidelijkt en aangevuld met Windows 7 voor 32-bits systemen met Service Pack 1, Windows 7 voor x64-systemen met Service Pack 1, Windows Server 2008 R2 voor x64-systemen met Service Pack 1 en Windows Server 2008 R2 voor Itanium-systemen met Service Pack 1.

Klanten van de oorspronkelijke officiële versie van Windows 7 voor 32-bits systemen, Windows 7 voor x64-systemen, Windows Server 2008 R2 voor x64-systemen en Windows Server 2008 R2 voor Itanium-systemen die deze beveiligingsupdate al hebben toegepast, hoeven geen actie te ondernemen. Klanten die deze besturingssystemen hebben bijgewerkt met het bijbehorende Service Pack 1, moeten deze beveiligingsupdate toepassen.

**Ik heb IIS geïnstalleerd maar heb FTP niet ingeschakeld. Moet ik deze update toepassen?**  
Dit beveiligingslek treft alleen systemen waarop de genoemde versies van FTP Service zijn ingeschakeld. Als een getroffen versie van FTP Service wel is geïnstalleerd, maar niet is ingeschakeld, wordt de update wel aangeboden via Automatische updates.

**Hoe kan ik controleren welke versie van de FTP-service op mijn systeem is geïnstalleerd?**  
Als in Windows Vista en Windows Server 2008 ftpsvc.dll voorkomt in de map %SystemRoot%\\system32\\inetsrv, bevat het systeem een versie van FTP Service die door dit beveiligingslek is getroffen (Microsoft FTP Service 7.0 of Microsoft FTP Service 7.5). Als op een van deze systemen het bestand ftpsvc2.dll in deze map voorkomt, wil dit zeggen dat Microsoft FTP Service 6.0 op het systeem is geïnstalleerd. Deze versie is niet getroffen door het beveiligingslek.

**Heeft mijn systeem last van dit beveiligingslek als de FTP-service is ingesteld om anonieme verificatie uit te schakelen?**  
Ja. Dit beveiligingslek treedt op vóór verificatie. Een schadelijke client kan een kwetsbaar systeem dus in gevaar brengen, zelfs als het systeem is geconfigureerd om anonieme verificatie te weigeren.

**Waar is de informatie over de bestandsgegevens?**  
Zie de referentietabellen in de sectie **Implementatie van de beveiligingsupdate** voor de details van de bestandsgegevens.

**Ik gebruik een oudere versie van de software die staat beschreven in dit beveiligingsbulletin. Wat moet ik doen?**  
De software waarin het probleem optreedt en die in dit beveiligingsbulletin wordt vermeld, is getest om te controleren of het probleem bij deze versies optreedt. Andere versies hebben het einde van hun ondersteuningscyclus bereikt. Ga naar de volgende [Microsoft Support Lifecycle](http://go.microsoft.com/fwlink/?linkid=21742)-website voor meer informatie over de levenscyclus van Windows-producten.

Klanten die met deze besturingssysteemversies werken, kunnen er het beste voor kiezen om binnenkort te migreren naar versies waarvoor wel ondersteuning wordt aangeboden om te voorkomen dat potentiële beveiligingslekken een probleem vormen. Zie [Selecteer een product voor lifecycle-informatie](http://go.microsoft.com/fwlink/?linkid=169555) om de ondersteuningscyclus voor uw softwareversie te bepalen. Zie [Lifecycle ondersteunde service packs](http://go.microsoft.com/fwlink/?linkid=89213) voor meer informatie over service packs voor deze softwareversies.

Klanten zonder Alliance-, Premier- of Authorized-contract kunnen contact opnemen met het Microsoft-verkoopkantoor in hun land. Voor contactinformatie gaat u naar de website [Microsoft Worldwide](http://go.microsoft.com/fwlink/?linkid=33329) en selecteert u het land in de lijst Contact Information. Beveiligingslek door heapbufferoverloop in FTP-service van IIS - CVE-2010-3972  
------------------------------------------------------------------------------
  
<span></span>
Er is een beveiligingslek aangetroffen in de FTP-service in Microsoft Internet Information Services (IIS) 7.0 en Microsoft Internet Information Services (IIS) 7.5. Door het beveiligingslek kan externe code worden uitgevoerd.
  
Zie [CVE 2010-3972](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3972) als u dit beveiligingslek wilt weergeven als standaardvermelding in de lijst met veelvoorkomende beveiligingslekken en blootstellingen.
  
#### Beperkende factoren voor het beveiligingslek door heapbufferoverloop in FTP-service van IIS - CVE-2010-3972
  
Met beperkende factoren wordt verwezen naar een instelling, de algehele configuratie of algemene tips voor een standaardstatus waarmee de ernst van misbruik via een beveiligingslek kan worden verminderd. De volgende beperkende factoren kunnen in uw situatie van pas komen:
  
-   FTP Service wordt niet standaard geïnstalleerd op Internet Information Service (IIS).  
-   FTP Service wordt niet automatisch gestart na installatie.  
-   Er wordt niet automatisch een FTP-site gemaakt, zelfs niet wanneer FTP Service is geïnstalleerd en gestart.
  
#### Tijdelijke oplossingen voor het beveiligingslek door heapbufferoverloop in FTP-service van IIS - CVE-2010-3972
  
Tijdelijke oplossingen zijn wijzigingen in een instelling of configuratie waardoor het beveiligingslek zelf niet wordt gedicht, maar waarmee aanvalsvectoren wel worden geblokkeerd voordat u de update uitvoert. Microsoft heeft de volgende methoden en statussen om het probleem te omzeilen getest om uit te zoeken of een tijdelijke oplossing functionaliteit vermindert:
  
-   **FTP Service stoppen en uitschakelen**
  
    Voer als beheerder de volgende opdracht uit vanaf een opdrachtprompt:
  
    `sc stop ftpsvcsc config ftpsvc start= disabled`
  
    **Gevolgen van de tijdelijke oplossing.** Clients kunnen geen verbinding maken met de FTP-server en kunnen geen bestanden uploaden of downloaden met de FTP-service.
  
    **De tijdelijke oplossing ongedaan maken.**
  
    Voer als beheerder de volgende opdracht uit vanaf een opdrachtprompt:
  
    `sc config ftpsvc start= autosc start ftpsvc`
  
-   **Inkomend FTP-verkeer blokkeren bij de edge-firewall**
  
    Blokkeer de volgende poort op de firewall:  
    TCP-poort 21
  
    Deze poort wordt gebruikt om een verbinding tot stand te brengen met het onderdeel waarin dit probleem optreedt. Door deze poort bij de edge-firewall te blokkeren, zorgt u ervoor dat op systemen achter deze firewall geen aanvallen kunnen worden uitgevoerd. U wordt aanbevolen alle ongewenste inkomende communicatie via internet te blokkeren om aanvallen te voorkomen waarvoor andere poorten worden gebruikt. Zie het TechNet-artikel [TCP and UDP Port Assignments](http://go.microsoft.com/fwlink/?linkid=21312) voor meer informatie over poorten.
  
    **Gevolgen van de tijdelijke oplossing.** Het is mogelijk dat de betreffende poort wordt gebruikt door toepassingen en services. Het blokkeren van de verbinding met de poort kan ertoe leiden dat verschillende toepassingen of services niet werken.
  
    **De tijdelijke oplossing ongedaan maken.**
  
    Hef de blokkering van TCP-poort 21 bij de firewall op. Zie het TechNet-artikel [TCP and UDP Port Assignments](http://go.microsoft.com/fwlink/?linkid=21312) voor meer informatie over poorten.
  
#### Veelgestelde vragen over het beveiligingslek door heapbufferoverloop in FTP-service van IIS - CVE-2010-3972
  
**Wat is de omvang van het probleem?**    
Dit beveiligingslek heeft betrekking op de uitvoering van externe code. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan volledige controle krijgen over een systeem waarin dit probleem optreedt. Een aanvaller kan dan programma's installeren, gegevens bekijken, wijzigen of verwijderen, of nieuwe accounts maken in de context van het lokale systeem.
  
**Waardoor wordt het beveiligingslek veroorzaakt?**    
Dit beveiligingslek wordt veroorzaakt door de manier waarop de FTP-service van Microsoft IIS speciaal vervaardigde FTP-opdrachten afhandelt.
  
**Met welk doel kan een aanvaller het beveiligingslek misbruiken?**    
Een aanvaller die misbruik weet te maken van dit beveiligingslek kan ervoor zorgen dat externe code op het doelsysteem wordt uitgevoerd in de context van actieve lokale systeem.
  
**Hoe kan een aanvaller dit beveiligingslek misbruiken?**    
Een aanvaller kan misbruik maken van dit beveiligingslek door een speciaal vervaardigde FTP-opdracht naar de FTP-server te sturen.
  
**Voor welke systemen vormt dit beveiligingslek het grootste risico?**    
Voor systemen waarop Microsoft IIS 7.0 en 7.5 en FTP-services worden gebruikt, is het risico het grootst.
  
**Wat doet de update?**    
Dit beveiligingslek verandert de manier waarop de FTP-service van Microsoft IIS speciaal vervaardigde FTP-opdrachten afhandelt.
  
**Was dit beveiligingslek openbaar gemaakt op het moment dat dit beveiligingsbulletin werd uitgegeven?**    
Ja. Dit beveiligingslek is openbaar gemaakt. Aan dit beveiligingslek is nummer [CVE-2010-3972](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3972) toegewezen.
  
**Was aan Microsoft gerapporteerd dat er misbruik was gemaakt van dit beveiligingslek op het moment dat dit beveiligingsbulletin werd uitgegeven?**    
Nee. Microsoft heeft geen informatie ontvangen die erop wees dat dit beveiligingslek was gebruikt om aanvallen uit te voeren op klanten op het moment dat dit beveiligingsbulletin oorspronkelijk werd uitgegeven.

Wanneer u deze beveiligingspatch installeert, wordt door het installatieprogramma gecontroleerd of een of meerdere bestanden die op de computer worden bijgewerkt, al eerder door een Microsoft-hotfix zijn bijgewerkt.
