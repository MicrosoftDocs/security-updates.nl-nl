Security Bulletin

Microsoft-beveiligingsbulletin MS11-006 - Kritiek
=================================================

Door een beveiligingslek in de grafische verwerking van de Windows-shell kan externe code worden uitgevoerd (2483185)
---------------------------------------------------------------------------------------------------------------------

Gepubliceerd: dinsdag 8 februari 2011 | Bijgewerkt: maandag 14 februari 2011

**Versie:** 1.1

### Algemene informatie

#### Samenvatting

Deze beveiligingsupdate lost een openbaar gemaakt beveiligingslek in de grafische verwerking van de Windows-shell op. Door dit beveiligingslek kan externe code worden uitgevoerd als een gebruiker een speciaal vervaardigde miniatuurafbeelding weergeeft. Een aanvaller die erin slaagt misbruik te maken van dit beveiligingslek, kan dezelfde rechten over het systeem krijgen als de lokale gebruiker. Gebruikers met accounts waarvoor minder gebruikersrechten op het systeem zijn ingesteld, lopen minder risico dan gebruikers die met beheerdersrechten werken.

Het prioriteitsniveau van deze beveiligingsupdate is Kritiek voor alle ondersteunde edities van Windows XP, Windows Server 2003, Windows Vista en Windows Server 2008. Ondersteunde edities van Windows 7 en Windows Server 2008 R2 worden niet door het beveiligingslek getroffen. Zie de subsectie **Software waarin dit probleem optreedt en software waarin dit probleem niet optreedt** in deze sectie voor meer informatie.

De beveiligingsupdate lost het beveiligingslek op doordat de manier wordt verbeterd waarop de grafische verwerking van de Windows-shell miniatuurafbeeldingen parseert. Zie de subsectie Veelgestelde vragen over het specifieke beveiligingslek in de volgende sectie, **Informatie over het beveiligingslek**, voor meer informatie over het beveiligingslek.

Deze beveiligingsupdate verwijdert ook het beveiligingslek dat voor het eerst werd beschreven in [Microsoft-beveiligingsadvies 2490606](http://technet.microsoft.com/security/advisory/2490606).

**Aanbeveling.** De meeste klanten hebben automatische updates ingeschakeld en hoeven geen actie te ondernemen omdat deze beveiligingsupdate automatisch wordt gedownload en geïnstalleerd. Klanten die automatische updates niet hebben ingeschakeld, moeten controleren op updates en deze update handmatig installeren. Zie [Microsoft Knowledge Base-artikel 294871](http://support.microsoft.com/kb/294871) voor informatie over specifieke configuratieopties voor het uitvoeren van automatische updates.

Als het gaat om beheerders en bedrijfinstallaties of eindgebruikers die deze beveiligingsupdate handmatig willen installeren, adviseert Microsoft klanten de update onmiddellijk toe te passen met behulp van de software voor updatebeheer of door op updates te controleren met de service [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Zie ook de sectie **Hulpmiddelen en richtlijnen voor detecteren en implementeren**, verderop in dit bulletin.

Beveiligingslek met betrekking tot overloop in de grafische verwerking van de Windows-shell - CVE-2010-3970  
-----------------------------------------------------------------------------------------------------------
  
<span></span>
Er bestaat een beveiligingslek met betrekking tot het uitvoeren van externe code door de manier waarop de grafische verwerking van de Windows-shell speciaal vervaardigde miniatuurafbeeldingen verwerkt. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan willekeurige code uitvoeren in de beveiligingscontext van de huidige aangemelde gebruiker. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige gebruikersrechten maken. Gebruikers met accounts waarvoor minder gebruikersrechten op het systeem zijn ingesteld, lopen minder risico dan gebruikers die met beheerdersrechten werken.
  
Zie [CVE-2010-3970](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3970) als u dit beveiligingslek wilt weergeven als standaardvermelding in de lijst met veelvoorkomende beveiligingslekken en blootstellingen.
  
#### Beperkende factoren voor het beveiligingslek met betrekking tot overloop in de grafische verwerking van de Windows-shell - CVE-2010-3970
  
Met beperkende factoren wordt verwezen naar een instelling, de algehele configuratie of algemene tips voor een standaardstatus waarmee de ernst van misbruik via een beveiligingslek kan worden verminderd. De volgende beperkende factoren kunnen in uw situatie van pas komen:
  
-   Het beveiligingslek kan niet automatisch worden misbruikt via e-mail.  
-   Een aanvaller die erin slaagt misbruik te maken van dit beveiligingslek, kan dezelfde rechten over het systeem krijgen als de lokale gebruiker. Gebruikers met accounts waarvoor minder gebruikersrechten op het systeem zijn ingesteld, lopen minder risico dan gebruikers die met beheerdersrechten werken.
  
#### Tijdelijke oplossingen voor het beveiligingslek met betrekking tot overloop in de grafische verwerking van de Windows-shell - CVE-2010-3970
  
Tijdelijke oplossingen zijn wijzigingen in een instelling of configuratie waardoor het beveiligingslek zelf niet wordt gedicht, maar waarmee aanvalsvectoren wel worden geblokkeerd voordat u de update uitvoert. Microsoft heeft de volgende methoden en statussen om het probleem te omzeilen getest om uit te zoeken of een tijdelijke oplossing functionaliteit vermindert:
  
-   **De toegangsbeheerlijst (ACL) voor shimgvw.DLL op Windows XP- en Windows Server 2003-systemen wijzigen**
  
    **Opmerking** Zie [Microsoft Knowledge Base-artikel 2483185](http://support.microsoft.com/kb/2483185) als u de automatische **Microsoft Fix it**-oplossing wilt gebruiken om deze tijdelijke oplossing in of uit te schakelen.
  
    Voer de volgende opdrachten uit vanaf een opdrachtprompt met beheerdersbevoegdheden om strengere beperkingen op te leggen voor de toegangsbeheerlijst voor shimgvw.dll:
  
    **Voor 32-bits versies van Windows XP en Windows Server 2003:**
  
    `Echo y| cacls %WINDIR%\SYSTEM32\shimgvw.dll /E /P everyone:N`
  
    **Voor 64-bits versies van Windows XP en Windows Server 2003:**
  
    `Echo y| cacls %WINDIR%\SYSTEM32\shimgvw.dll /E /P everyone:NEcho y| cacls %WINDIR%\SYSWOW64\shimgvw.dll /E /P everyone:N`
  
    **Gevolgen van de tijdelijke oplossing.** Mediabestanden die normaal worden verwerkt door de grafische verwerking van de Windows-shell, worden niet correct weergegeven. Voordat u de beveiligingsupdate van MS11-006 installeert, moet u deze tijdelijke oplossing ongedaan maken op systemen waarop u deze hebt toegepast.
  
    **De tijdelijke oplossing ongedaan maken.**
  
    Voer als beheerder de volgende opdrachten uit vanaf een opdrachtprompt:
  
    Voor 32-bits versies van Windows XP en Windows Server 2003:
  
    `cacls %WINDIR%\SYSTEM32\shimgvw.dll /E /R everyone`
  
    Voor 64-bits versies van Windows XP en Windows Server 2003:
  
    `cacls %WINDIR%\SYSTEM32\shimgvw.dll /E /R everyonecacls %WINDIR%\SYSWOW64\shimgvw.dll /E /R everyone`
  
-   **De weergave van miniaturen in Windows Verkenner uitschakelen op Windows Vista- en Windows Server 2008-systemen**
  
    Ga als volgt te werk als u de weergave van miniaturen in Windows Verkenner op Windows Vista- en Windows Server 2008-systemen wilt uitschakelen:
  
    1.  Open **Configuratiescherm** en selecteer **Vormgeving aan persoonlijke voorkeur aanpassen**.  
    2.  Selecteer **Mapopties** in het menu.  
    3.  Ga naar het tabblad **Weergave** en zorg ervoor dat de optie **Altijd pictogrammen weergeven, nooit miniatuurweergaven** is geselecteerd. Klik op **OK** om de instellingen op te slaan.  
    4.  Sluit alle geopende vensters van Windows Verkenner om de wijziging door te voeren.
  
    **Gevolgen van de tijdelijke oplossing.** Windows Verkenner geeft geen miniatuurafbeeldingen meer weer.
  
    **De tijdelijke oplossing ongedaan maken.**
  
    Ga als volgt te werk als u de weergave van miniatuurafbeeldingen in Windows Vista en Windows Server 2008 weer wilt inschakelen:
  
    1.  Open **Configuratiescherm** en selecteer **Vormgeving aan persoonlijke voorkeur aanpassen**.  
    2.  Selecteer **Mapopties** in het menu.  
    3.  Ga naar het tabblad **Weergave** en zorg ervoor dat de optie **Altijd pictogrammen weergeven, nooit miniatuurweergaven** niet is geselecteerd. Klik op **OK** om de instellingen op te slaan.  
    4.  Sluit alle geopende vensters van Windows Verkenner om de wijziging door te voeren.
  
#### Veelgestelde vragen voor het beveiligingslek met betrekking tot overloop in de grafische verwerking van de Windows-shell - CVE-2010-3970
  
**Wat is de omvang van het probleem?**    
Dit beveiligingslek heeft betrekking op de uitvoering van externe code. Een aanvaller die misbruik weet te maken van dit beveiligingslek, kan op afstand volledige controle krijgen over een systeem waarin dit probleem optreedt. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige gebruikersrechten maken.
  
**Waardoor wordt het beveiligingslek veroorzaakt?**    
Het beveiligingslek wordt veroorzaakt wanneer de grafische verwerking van de Windows-shell op onjuiste wijze een speciaal vervaardigde miniatuurafbeelding parseert, waardoor een stackoverloop ontstaat.
  
**Wat is de Windows-shell?**    
De Windows-gebruikersinterface (UI) biedt gebruikers toegang tot zeer veel verschillende objecten die nodig zijn voor het uitvoeren van toepassingen en het beheren van het besturingssysteem. Het grootste en bekendste van deze typen objecten zijn de mappen en bestanden op de schijfstations. Er is ook een aantal virtuele objecten waarmee de gebruiker taken kan uitvoeren zoals het verzenden van bestanden naar netwerkprinters of toegang tot de prullenbak. De shell structureert deze objecten in een hiërarchische naamruimte en biedt gebruikers en toepassingen een consistente en efficiënte manier om toegang te krijgen tot objecten en deze te beheren.
  
**Met welk doel kan een aanvaller het beveiligingslek misbruiken?**    
Als er een gebruiker met beheerdersrechten is aangemeld, kan een aanvaller die misbruik weet te maken van dit beveiligingslek, volledige controle krijgen over het systeem waarin dit probleem optreedt. De aanvaller kan vervolgens programma's installeren, gegevens bekijken, wijzigen of wissen, of nieuwe accounts met volledige gebruikersrechten maken. Gebruikers met accounts waarvoor minder gebruikersrechten op het systeem zijn ingesteld, lopen minder risico dan gebruikers die met beheerdersrechten werken.
  
**Hoe kan een aanvaller dit beveiligingslek misbruiken?**    
Voor dit beveiligingslek moet een gebruiker een speciaal vervaardigde miniatuurafbeelding weergeven.
  
In een aanval via het netwerk kan een aanvaller een speciaal vervaardigde miniatuurafbeelding of een bestand met daarin een speciaal vervaardigde miniatuurafbeelding op een netwerkshare plaatsen, zoals op een UNC- of WebDAV-locatie, en de gebruiker vervolgens overhalen om in Windows Verkenner naar die locatie te bladeren. Wanneer de gebruiker naar de share gaat, wordt het pad van het getroffen besturingselement geactiveerd via de grafische verwerking van de Windows-shell. De speciaal vervaardigde miniatuurafbeelding kan vervolgens misbruik maken van het beveiligingslek en code uitvoeren in de beveiligingscontext van de aangemelde gebruiker. Een aanvaller kan gebruikers nooit dwingen om een netwerkshare, UNC- of WebDAV-locatie te bezoeken. De aanvaller moet gebruikers er dus toe overhalen de share te bezoeken. In de meeste gevallen gebeurt dit door hen ertoe te bewegen op een koppeling in een e-mailbericht of Instant Messenger-bericht te klikken, die ze naar de netwerkshare, UNC- of WebDAV-locatie brengt.
  
**Wat is Uniform Naming Convention (UNC)? **  
Uniform Naming Convention (UNC) is een methode om bronnen zoals sharenamen of bestanden in een netwerk te identificeren. Een normale UNC-naam begint met twee backslashes, gevolgd een servernaam, bijvoorbeeld \\\\server\\share\\submap\\bestandsnaam.
  
**Voor welke systemen vormt dit beveiligingslek het grootste risico?**    
Voor werkstations en terminalservers is het risico het grootst. Servers zouden eventueel meer gevaar kunnen lopen als beheerders toestaan dat gebruikers zich op servers kunnen aanmelden en programma's kunnen uitvoeren. Dit is echter volkomen in strijd met de aanbevolen richtlijnen voor gebruik.
  
**Wat doet de update?**    
De beveiligingsupdate lost het beveiligingslek op doordat de manier wordt verbeterd waarop de grafische verwerking van de Windows-shell miniatuurafbeeldingen parseert.
  
**Was dit beveiligingslek openbaar gemaakt op het moment dat dit beveiligingsbulletin werd uitgegeven?**    
Ja. Dit beveiligingslek is openbaar gemaakt. Aan dit beveiligingslek is nummer [CVE-2010-3970](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3970) toegewezen. Dit beveiligingslek is voor het eerst beschreven in [Microsoft-beveiligingsadvies 2490606](http://technet.microsoft.com/security/advisory/2490606).
  
**Was aan Microsoft gerapporteerd dat er misbruik was gemaakt van dit beveiligingslek op het moment dat dit beveiligingsbulletin werd uitgegeven?**    
Nee. Microsoft heeft wel voorbeelden van gepubliceerde conceptcode gezien als bewijs maar beschikte niet over rechtstreekse meldingen dat dit beveiligingslek werd gebruikt voor aanvallen op klanten op het moment dat dit beveiligingsbulletin oorspronkelijk werd uitgegeven.
  
**Helpt de toepassing van deze beveiligingsupdate klanten om zich te beschermen tegen de openbaar gemaakte code die is gericht op misbruik van dit beveiligingslek?**    
Ja. Deze beveiligingsupdate dicht het beveiligingslek dat mogelijk kon worden misbruikt via het gepubliceerde voorbeeld van conceptcode. Het beveiligingslek is opgelost en hieraan is het nummer [CVE-2010-3970](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3970) toegewezen.
